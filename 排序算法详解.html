冒泡排序（Bubble Sort）

稳定 · O(n²) · O(1)空间

核心思想：像气泡一样，每轮把最大的元素"浮"到最后面。相邻两两比较，大的往后换。就像气泡从水底冒到水面。

过程：第1轮比较n-1次，最大的沉到最后；第2轮比较n-2次，次大的沉到倒数第二…直到全部有序。

初始: [6, 3, 1, 4, 5, 2] 比较6和3 → 6>3交换 第1轮 (把最大6冒到最后): 比较6和3 → 交换 → [3, 6, 1, 4, 5, 2] 比较6和1 → 交换 → [3, 1, 6, 4, 5, 2] 比较6和4 → 交换 → [3, 1, 4, 6, 5, 2] 比较6和5 → 交换 → [3, 1, 4, 5, 6, 2] 比较6和2 → 交换 → [3, 1, 4, 5, 2, 6✓] ← 6归位 第2轮 (把5冒到倒数第二): 比较3和1 → 交换 → [1, 3, 4, 5, 2, 6] 比较3和4 → 不换 → [1, 3, 4, 5, 2, 6] 比较4和5 → 不换 → [1, 3, 4, 5, 2, 6] 比较5和2 → 交换 → [1, 3, 4, 2, 5✓, 6] ← 5归位 第3轮 → [1,3,2,4✓,5,6] 第4轮 → [1,2,3✓,4,5,6] 第5轮 → [1,2✓,3,4,5,6] 最终: [1, 2, 3, 4, 5, 6]

稳定 最好O(n)已有序 平均O(n²) 空间O(1)

一趟结果判定：一趟冒泡后，最大元素一定在最后位置。如果一趟没有发生任何交换→已经排好序了，可以提前结束。

插入排序（Insertion Sort）

稳定 · O(n²) · O(1)空间 · 像整理扑克牌

核心思想：像抓扑克牌一样，每次把一张新牌插入到已经排好序的牌中正确位置。从左到右，左边始终是有序的。

初始: [6 | 3, 1, 4, 5, 2] ← 竖线左边"已排序区"，右边"待插入区" 第1趟 插入3: 3<6 → 6右移一位，3放到6原来的位置 结果: [3, 6 | 1, 4, 5, 2] ← 前2个有序 第2趟 插入1: 1<6 → 6右移；1<3 → 3右移；1放最前 结果: [1, 3, 6 | 4, 5, 2] ← 前3个有序 第3趟 插入4: 4<6 → 6右移；4>3 → 停止，4放3后面 结果: [1, 3, 4, 6 | 5, 2] 第4趟 插入5: 5<6 → 6右移；5>4 → 停止 结果: [1, 3, 4, 5, 6 | 2] 第5趟 插入2: 2<6,2<5,2<4,2<3,2>1 → 1后面 结果: [1, 2, 3, 4, 5, 6] ← 全部有序！

稳定 最好O(n) 平均O(n²) 空间O(1)

关键特点：第i趟后，前i+1个元素有序。适合"基本有序"的序列（接近O(n)）。

一趟结果示例：序列6,3,1,4,5,2 → 第1趟后3,6,1,4,5,2 → 第2趟后1,3,6,4,5,2

选择排序（Selection Sort）

不稳定 · O(n²) · O(1)空间

核心思想：每轮从未排序区选出最小的那个，放到已排序区的末尾。像一个挑剔的人，每次都先挑最小的。

初始: [12, 33, 25, 7, 19] 第1趟: 在[12,33,25,7,19]中找最小 → 7（位置4） 7和12交换 → [7, 33, 25, 12, 19] ← 7归位到位置1 第2趟: 在[33,25,12,19]中找最小 → 12（位置4） 12和33交换 → [7, 12, 25, 33, 19] ← 12归位到位置2 第3趟: 在[25,33,19]中找最小 → 19（位置5） 19和25交换 → [7, 12, 19, 33, 25] 第4趟: 在[33,25]中找最小 → 25（位置5） 25和33交换 → [7, 12, 19, 25, 33] ← 全部有序！

不稳定 最好/平均/最坏 都是O(n²) 空间O(1)

为什么不稳定？跨距离交换可能打乱相同值的顺序。例如 [5₁, 8, 5₂, 2]→选最小2与5₁交换→[2,8,5₂,5₁]两个5顺序反了！

一趟结果示例：12,33,25,7,19 → 第2趟后=7,12,25,33,19

希尔排序（Shell Sort）

不稳定 · O(n¹·³)平均 · O(1)空间 · 插入排序的升级版

核心思想：插入排序的升级版——先"大步"跳着排序（让元素快速接近最终位置），然后逐步缩小步长，最后一趟步长=1就是普通插入排序。此时序列已"基本有序"，插入排序接近O(n)。

序列: [24, 13, 9, 64, 37, 86, 32, 41, 72] 增量序列: 7, 3, 1 ━━━ 增量d=7 ━━━ 间隔7的元素一组（共2组）： 组①: 24(0)──32(7) → [24,13,9,64,37,86,32,41,72] 组②: 13(1)──41(8) → [13,41,...] 其余: 各成一组，组内插入排序后基本无变化 结果: [24, 13, 9, 64, 37, 86, 32, 41, 72] (无交换) ━━━ 增量d=3 ━━━ 间隔3的元素一组（共3组）： 组①: 24(0), 64(3), 86(6), 72(8) → 已有序 组②: 13(1), 37(4), 32(7) 组内排序: 13,32,37 → 位置调整 组③: 9(2), 86(5), 41(8) 组内排序: 9,41,86 → 位置调整 结果: [24, 13, 9, 64, 32, 41, 86, 37, 72] → [24, 13, 9, 32, 37, 72, 64, 41, 86] ━━━ 增量d=1 ━━━ 普通插入排序（此时序列已基本有序，很快完成） 结果: [9, 13, 24, 32, 37, 41, 64, 72, 86]

不稳定 平均O(n1.3) 最坏O(n²) 空间O(1)

关键：增量序列的选择很重要。常用：... → 7 → 3 → 1（最后一趟必须是1）。

一趟结果示例：增量7趟=24,13,9,64,37,86,32,41,72（无交换）；增量3趟=24,13,9,32,37,72,64,41,86

快速排序（Quick Sort）

不稳定 · O(nlogn)平均 · 分治思想 · 最重要！

核心思想：选一个基准（pivot），把比它小的放左边，比它大的放右边，基准放到中间正确位置。然后递归处理左右两部分。分而治之！

序列: [6, 1, 8, 3, 5, 2, 7, 4] 选最后一个元素4为基准 ━━━ 一趟 partition ━━━ i=-1, j从0到6: j=0: 6>4 → 不动 j=1: 1<4 → i=0, 交换6和1 → [1, 6, 8, 3, 5, 2, 7, 4] j=2: 8>4 → 不动 j=3: 3<4 → i=1, 交换6和3 → [1, 3, 8, 6, 5, 2, 7, 4] j=4: 5>4 → 不动 j=5: 2<4 → i=2, 交换8和2 → [1, 3, 2, 6, 5, 8, 7, 4] j=6: 7>4 → 不动 最后基准4和i+1位置(6)交换 → [1, 3, 2, 4, 5, 8, 7, 6] ←左边都≤4 右边都≥4→ 一趟结果: [1, 3, 2, 4, 5, 8, 7, 6] ← 4已归位(排序后的正确位置)

不稳定 平均O(nlogn)⚡ 最坏O(n²)（完全有序时） 栈空间O(logn)

一趟判定（选择题必考）：一趟快排后，存在一个元素，左边全≤它，右边全≥它，且它在排序后的最终位置。逐个位置假设检验。

经典例题：下面哪个不可能是一趟快排结果？A、18,10,2,88,71 B、3,14,8,27,31 C、8,20,15,1,23 D、23,19,28,37,50 → 答案A（18不能是基准因为它在开头但右边有88>18也有10<18，任何位置都不满足条件）

另一例题：27,13,2,5,19 → 不可能（27最大却在第一位，没归位）

堆排序（Heap Sort）

不稳定 · O(nlogn) · O(1)空间 · 原地排序

核心思想：先建堆（最大堆的堆顶=最大值），然后反复"输出堆顶→用最后一个元素替补→下沉调整"。就像锦标赛，每次决出冠军后淘汰他，再选新冠军。

序列: [6, 3, 1, 4, 5, 2] ━━━ ① 建最大堆（从最后一个非叶结点开始调整）━━━ 6 6 6 / \ / \ / \ 3 1 → 5 1 → 5 2 / \ / / \ / / \ / 4 5 2 4 3 2 4 3 1 堆数组（层序）: [6, 5, 2, 4, 3, 1] ← 最大堆建成！ ━━━ ② 堆排序（每次输出堆顶+调整）━━━ 第1趟: 输出6，用最后元素1替顶 → [1,5,2,4,3] 1下沉: 1<5换 → [5,1,2,4,3]; 1<4换 → [5,4,2,1,3] 已输出: [6] 第2趟: 输出5，用最后元素3替顶 → [3,4,2,1] 3下沉: 3<4换 → [4,3,2,1] 已输出: [6, 5] 最终: [1,2,3,4,5,6] ← 反复输出即得有序序列

不稳定（！） 建堆O(n) + n×O(logn) = O(nlogn) 空间O(1)

选择题陷阱：堆排序空间复杂度是O(1)（原地排序），不像归并需要O(n)！平均时间O(nlogn)，但不稳定（建堆和调整会跨距离交换）。

一趟结果示例：建堆6,5,2,4,3,1 → 第1趟后5,4,2,1,3 → 第2趟后4,3,2,1

归并排序（Merge Sort）

稳定 · O(nlogn) · O(n)空间 · 分治思想

核心思想：分治法——先把序列递归地分成两半直到每段只有1个元素（天然有序），然后合并（merge）两个有序段。合并时用双指针，谁小取谁。

序列: [38, 27, 43, 3, 9, 82, 10] ━━━ 递归分解 ━━━ [38,27,43,3,9,82,10] / \ [38,27,43,3] [9,82,10] / \ / \ [38,27] [43,3] [9,82] [10] / \ / \ / \ | [38] [27] [43] [3] [9] [82] [10] ━━━ 递归合并（Merge，双指针归并）━━━ [38]←→[27] → 27<38: [27,38] [43]←→[3] → 3<43: [3,43] [27,38]←→[3,43] → 3<27:3, 27<43:27, 38<43:38 → [3,27,38,43] [9]←→[82] → 9<82: [9,82] [9,82]←→[10] → 9<10:9, 10<82:10, 82 → [9,10,82] [3,27,38,43]←→[9,10,82] 3<9→3, 9<27→9, 10<27→10, 27<82→27, 38<82→38, 43<82→43, 82 → [3, 9, 10, 27, 38, 43, 82] ← 完全有序！

稳定 最好/平均/最坏 都是O(nlogn) 空间O(n)（需要辅助数组）

选择题常考点：归并排序是唯一空间O(n)的O(nlogn)算法。快排O(logn)栈空间，堆排O(1)空间。

适用场景：适合外部排序（数据太大放磁盘，分段排序后归并）。稳定所以适合对象排序。

基数排序（Radix Sort）

稳定 · O(d(n+r)) · 非比较排序

核心思想：不比较元素大小！而是按个位→十位→百位的顺序，依次按每位数字分配+收集。多关键字排序的思想。

序列: [170, 45, 75, 90, 802, 24, 2, 66] ━━━ 第1趟：按个位分配 ━━━ 桶0: 170, 90 桶2: 802, 2 桶4: 24 桶5: 45, 75 桶6: 66 收集: [170, 90, 802, 2, 24, 45, 75, 66] ━━━ 第2趟：按十位分配 ━━━ 桶0: 802, 2 桶2: 24 桶4: 45 桶6: 66 桶7: 170, 75 桶9: 90 收集: [802, 2, 24, 45, 66, 170, 75, 90] ━━━ 第3趟：按百位分配 ━━━ 桶0: 2, 24, 45, 66, 75, 90 桶1: 170 桶8: 802 收集: [2, 24, 45, 66, 75, 90, 170, 802] ← 有序！

稳定 O(d(n+r))（d=位数，r=基数，n=元素数）空间O(n+r)

关键特点：不基于比较的排序！突破O(nlogn)下限。但只能排整数/字符串等可分解为"位"的数据。

排序方向：LSD（最低位优先，从个位开始）最常用。每趟必须稳定（桶内保持原顺序）。

🔥 考前速记口诀

• 不稳定的有4个：选（选择）希（希尔）快（快排）堆（堆排序）—— "选希快堆"谐音"选西快堆"

• 空间O(1)的O(nlogn)：只有堆排序！快排要O(logn)栈，归并要O(n)辅助

• 时间复杂度不受输入影响（最好=最坏）：选择O(n²)、归并O(nlogn)、堆O(nlogn)、基数O(d(n+r))

• 最好O(n)的：冒泡（已有序一趟无交换）、插入（已有序每趟只比一次）

• 快排最坏O(n²)：每次基准选到最值（如已有序序列选第一个/最后一个做基准）

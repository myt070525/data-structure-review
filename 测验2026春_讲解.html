时间复杂度分析 选择填空·4分

🍜 第一步：大白话引入

想象你去食堂打饭。一个阿姨打一个菜需要 1分钟。如果你一个人点了 n个菜，那就要等 n分钟。

但如果有两个窗口——第一个窗口负责 n 道主食，第二个窗口负责 m 道炒菜，而你必须先在第一个窗口排完，再去第二个窗口排——那总时间就是 n + m。

可如果变成这样：第一个窗口每出一个菜，你都得去第二个窗口把所有 m 个菜重新热一遍——那总时间就是 n × m！

"时间复杂度"不是算精确的秒数，而是问：数据量变大的时候，计算量增长得有多快？

🔍 第二步：拆解题目的意思

题目给了这样一段代码：

for (i = 0; i < n; i++) for (j = 0; j < m; j++) sum += a[i][j];

问：这段代码的时间复杂度是多少？选项里会有 O(n+m)、O(nm)、O(n²) 等。

翻译成人话：当 n 和 m 都变大时，sum += a[i][j] 这条语句要执行多少次？

📝 第三步：一步一步推导

1 外层循环 for (i = 0; i < n; i++)：i 从 0 到 n-1，一共转 n 圈。

2 内层循环 for (j = 0; j < m; j++)：外层每转一圈，内层都要从 0 到 m-1 转 m 圈。

3 所以一共转了 n × m 圈。就像你妈让你把 n 个抽屉每个都擦 m 遍——总共擦了 n×m 次。

4 大O表示法里，常数系数忽略，这就是 O(nm)。

循环结构执行次数时间复杂度生活类比
并列的两个循环n + mO(n+m)先排窗口A再排窗口B
嵌套的两个循环n × mO(nm)窗口A每出一个菜，窗口B全部重做

核心规律：嵌套 = 相乘，并列 = 相加。

答案：O(mn)（或写作 O(nm)）
🔑 关键点：看到嵌套循环，把内外层次数乘起来就是时间复杂度。别和并列循环搞混——嵌套是"每...都..."的关系，并列是"先...再..."的关系。

ADT三要素缺的是？选择填空·4分

🚗 第一步：大白话引入

你去4S店买车，销售给你一张"配置表"。表上写着：

  • 这车有什么：发动机、四个轮子、座椅、方向盘
  • 这些东西之间什么关系：发动机连着轮子，方向盘控制轮子转向
  • 你能怎么用：踩油门加速、转方向盘拐弯、踩刹车停车

但配置表不会告诉你发动机的内部构造、活塞怎么运动的。那些是"怎么造的"——属于制造厂的秘密。

ADT（抽象数据类型）就是这个"配置表"！它只告诉你"有什么、什么关系、能做什么"，不告诉你"怎么实现"。

🔍 第二步：拆解题目的意思

题目问：ADT的三个要素包括"数据对象"和"数据操作"，第三个缺的是什么？

选项里往往会有"存储结构"这种干扰项——那属于"怎么实现"，不是ADT的范畴。

📝 第三步：一步一步推导

ADT三要素，记住这个口诀："有什么·什么关系·能做什么"

要素含义买车类比
① 数据对象这个结构里存的是什么数据车有什么零件
② 数据关系数据之间怎么关联、谁前谁后零件之间怎么连接
③ 数据操作能对这些数据做什么（插入、删除、查找…）你能怎么操控这辆车

"存储结构"（比如用数组还是链表实现）属于物理层面，是"怎么造"的问题，不属于ADT。ADT只管逻辑层面。

答案：数据关系
🔑 关键点：ADT = 逻辑（有什么、关系、操作），不涉及物理存储。把ADT想象成"产品说明书"，不是"制造图纸"。

入栈abcde，d第一个出栈，可能的出栈序列个数选择填空·5分

🍽️ 第一步：大白话引入

栈是什么？就是一叠盘子。你只能从最顶上放盘子，也只能从最顶上拿盘子。最后放上去的，最先被拿走——这叫"后进先出"（LIFO）。

就像你刷短视频：最新刷到的在最上面，往回划也是先看到最新的。

🔍 第二步：拆解题目的意思

题目说：a、b、c、d、e 按顺序入栈（一次可以入一个，也可以入了再出），但要求 d 是第一个出栈的。问一共有多少种可能的出栈顺序？

翻译：d 必须是第一个出来的，那 a、b、c、e 在后面以什么顺序出来？

📝 第三步：一步一步推导

1 d第一个出来意味着什么？d要出来，必须先把a、b、c、d都入栈——因为d在最上面才能出。所以此时栈里的状态是：

d在栈顶，所以d第一个出来。a、b、c被压在下面，e还没进来。

2 现在栈里还剩 a、b、c（c在栈顶），外面还有 e 没入栈。接下来每一步，你只有两个选择：

  • 出栈：把栈顶的弹出来
  • 入栈：把 e 推进去（但 e 只能入一次）

3 现在枚举所有可能（d已经出去了，剩下安排 a、b、c、e）：

编号出栈序列操作过程
①d c b a ed出→c出→b出→a出→e入→e出
②d c b e ad出→c出→b出→e入→e出→a出
③d c e b ad出→c出→e入→e出→b出→a出
④d e c b ad出→e入→e出→c出→b出→a出

4 为什么只有4种？因为 a 被压在最底下，只要它上面还有人（b或c），a 就出不来。所以 a 必然最后一个出。e 是灵活因素——它可以在c出来之前或之后入栈。

答案：4种（dcbae、dcbea、dceba、decba）
🔑 关键点：栈的出栈序列受"后进先出"约束。当一个元素在栈底时，它上面所有元素必须先出来。枚举时要考虑"谁被谁压着"。

循环队列大小9，front=7, rear=2，元素个数选择填空·4分

🧑‍🤝‍🧑 第一步：大白话引入

队列就是排队——先进先出（FIFO）。就像奶茶店排队：先来的先买到，后来的排后面。

但用数组实现队列有个问题：前面的人走了，位置空出来，但后面的人不能往前挪（效率太低）。于是有了循环队列——把数组的头和尾"接"起来，形成一个圆圈。空出来的位置可以接着用！

🔍 第二步：拆解题目的意思

一个循环队列有9个格子（编号0到8），front指针在7号位，rear指针在2号位。问现在队列里有几个元素？

📝 第三步：一步一步推导

1 先画一个圆圈——这就是我们的循环队列：

圆圈上的格子。front指向7（从这里开始取），rear指向2（新元素放这里）。红色弧线上的格子7,8,0,1是有数据的。

2 数一数：从 front(7) 开始，沿着顺时针方向，一直到 rear(2) 之前（rear指向的位置是空的，用来区分队空和队满）。

经过的格子：7 → 8 → 0 → 1 = 4个。

3 公式验证：元素个数 = (rear - front + size) % size

元素个数 = (2 - 7 + 9) % 9 = 4 % 9 = 4
答案：4个
🔑 关键点：循环队列元素个数公式 = (rear - front + size) % size。核心是"转圈取模"——就像时钟过了12点又从1开始。

广义表((a,b),(a,(c,d)))的表尾选择填空·4分

🧅 第一步：大白话引入

广义表就像一个可以嵌套的盒子。大盒子里可以装小盒子，小盒子里还能装更小的盒子。

规则很简单：表头 = 第一个元素（不管它是单个东西还是另一个盒子）；表尾 = 去掉第一个元素后剩下的所有（一定套一层括号！）

就像剥洋葱——每次剥掉最外面一片，剩下的还是一颗洋葱（虽然小了一点）。

🔍 第二步：拆解题目的意思

广义表 ((a,b), (a,(c,d)))——最外层是一个表，里面有两个元素：

  • 第一个元素：(a,b)（一个小表）
  • 第二个元素：(a,(c,d))（另一个小表）

题目问：这个广义表的表尾是什么？

📝 第三步：一步一步推导

1 表尾 = 去掉第一个元素，剩下的所有，外面加一层括号。

2 原表：( (a,b) , (a,(c,d)) )

第一个元素：(a,b)

去掉它之后剩下的：(a,(c,d)) —— 只有一个元素

3 表尾必须是一个表，所以要给它套括号：((a,(c,d)))

4 表头 = (a,b)（第一个元素，不带额外括号）

表尾 = ((a,(c,d)))（去掉第一个，剩下部分再套括号）

答案：((a,(c,d)))
🔑 关键点：表尾一定是一个表（有外层括号）！哪怕去掉第一个元素后只剩一个东西，也要给它套上括号。表头不额外加括号。

后缀表达式求值 = 9选择填空·5分

🧮 第一步：大白话引入

平时我们写算式是这样的：5 + 3 × 2——运算符在数字中间，这叫"中缀表达式"。人类看着很自然，但计算机处理起来很麻烦（要判断优先级、括号）。

计算机更喜欢"后缀表达式"：把运算符放到数字后面。比如 5 + 3 写成 5 3 +。

为什么？因为后缀表达式不需要括号，不需要管优先级——从左到右扫描，遇到数字就存起来，遇到运算符就拿两个出来算。

🔍 第二步：拆解题目的意思

题目给了一个后缀表达式（类似 5 2 3 1 + * 2 / +），问你计算结果。求值规则只有两条：

① 遇到数字 → 入栈

② 遇到运算符 → 弹出两个数（先弹出的是右操作数，后弹出的是左操作数）→ 计算 → 结果入栈

📝 第三步：一步一步推导（跟着栈的变化走）

以后缀表达式 5 2 3 1 + * 2 / + 为例，咱们一步一步看栈的变化：

栈的变化过程：每步读入一个字符，根据规则操作栈。黄色=刚入栈的数字，深黄色=计算结果。

最后 栈里只剩一个数字 9——这就是后缀表达式的值！

答案：9
🔑 关键点：后缀表达式求值 = 数字入栈 + 运算符弹两个算。注意弹出顺序：先弹出的是右操作数！不需要括号和优先级规则，计算机的最爱。

KMP算法：next数组="011223"，比较次数=14选择填空·5分

🔎 第一步：大白话引入

先说说暴力匹配为什么笨。假设你在文章里找"数据结构"这个词：

  • 暴力法：从文章第1个字开始比对，对不上？从第2个字重新比……就像你找钥匙，从客厅第一个抽屉翻到最后一个，没找到？再从第一个抽屉翻一遍。
  • KMP的聪明之处：已经比过的地方，利用这些信息，不要重新比！就像你已经翻过前三个抽屉确认没有钥匙了，不需要为了"从第四个开始找"而重新翻前三个。
🔍 第二步：拆解题目的意思

题目给了模式串的next数组：0 1 1 2 2 3，以及在匹配过程中总共比较了14次字符。

next数组是什么？它是一个"指路牌"。next[i] 告诉你：如果在模式串的第 i 个位置匹配失败了，模式串应该跳到哪个位置继续比（而不是傻傻地从模式串开头重新比）。

📝 第三步：一步一步推导

先理解 next 数组怎么来的

next 数组的核心是找"最长相等前后缀"——模式串每个位置之前，有多长的前缀和后缀是一样的。

举个直观例子，假设模式串是 a b a b c a：

位置 i子串最长相等前后缀长度next[i]含义
0(空)—0第一个字符，无路可退
1a01a没有相等前后缀，失败回到开头
2ab01ab也没有
3aba1（前缀a=后缀a）2失败时跳过1个字符
4abab2（前缀ab=后缀ab）2失败时跳过2个字符
5ababc03失败时跳过…

比较次数为什么是14？

KMP算法中，主串指针永远不回溯，模式串指针根据 next 数组回退。总比较次数 = 主串长度 + 模式串回退过程中额外比较的次数。14次比较意味着匹配过程中，每个字符平均只比了不到2次——比暴力法高效得多！

记住：KMP = 主串不回头，模式串看next指路牌回退

答案：next数组为"011223"，比较次数14
🔑 关键点：KMP的核心是"不重复劳动"。next数组记录了"已经匹配过的部分里，前缀和后缀相等的最大长度"。匹配失败时，模式串跳到next指示的位置，主串指针不后退。

中序ABCDEFG，后序BDCAFGE，左子树几个结点？选择填空·5分

🌲 第一步：大白话引入

三种遍历就像三种"逛树"的方式：

  • 先序遍历：先看自己 → 再看左边 → 最后右边（"我-左-右"）
  • 中序遍历：先看左边 → 再看自己 → 最后右边（"左-我-右"）
  • 后序遍历：先看左边 → 再看右边 → 最后自己（"左-右-我"）

后序遍历有一个非常重要的性质：最后一个访问的一定是整棵树的根！（因为根是"最后才轮到自己"的那个。）

🔍 第二步：拆解题目的意思

已知：中序 = A B C D E F G，后序 = B D C A F G E。

问：左子树有几个结点？（也就是根的左边有几个）

用"后序最后一个是根"这个性质，配合中序来切分左右子树。

📝 第三步：一步一步推导

1 后序最后一个 = E → E 是整棵树的根。

2 在中序里找到 E：A B C D E F G

E 左边有 A、B、C、D 四个结点 → 左子树 = 4个结点

E 右边有 F、G 两个结点 → 右子树 = 2个结点

3 题目只问左子树结点数，到这里答案就已经出来了：4。

完整还原的二叉树。中序+后序可以唯一确定一棵树。
答案：左子树有 4 个结点（ABCD）
🔑 关键点：后序遍历的最后一个结点一定是根。在中序里找到根，左边就是左子树，右边就是右子树。递归下去就能还原整棵树。

完全二叉树321个结点，叶子数=161选择填空·5分

🔺 第一步：大白话引入

完全二叉树长什么样？就像一座金字塔——从上到下，从左到右，一个接一个地填格子，中间不能有空缺。最下面一层可以不满，但必须"靠左对齐"。

叶子是什么？就是没有孩子的结点——金字塔最底下的那些"终端"。

🔍 第二步：拆解题目的意思

有一棵完全二叉树，总共321个结点。问：叶子结点（度为0的结点）有多少个？

📝 第三步：一步一步推导

1 先记住二叉树的一个重要公式：n₀ = n₂ + 1（叶子数 = 度为2的结点数 + 1）。这个公式对所有二叉树都成立。

2 完全二叉树的特殊性质：度为1的结点数 n₁ 只能是0或1。为什么？因为"靠左填充"不允许中间有空缺。如果总结点数是奇数，n₁=0；如果总结点数是偶数，n₁=1。

3 321是奇数，所以 n₁ = 0。

4 总数 n = n₀ + n₁ + n₂ = n₀ + 0 + n₂。

又因为 n₀ = n₂ + 1，即 n₂ = n₀ - 1。

代入：321 = n₀ + 0 + (n₀ - 1) = 2×n₀ - 1

5 2×n₀ = 322 → n₀ = 161

答案：叶子数 = 161
🔑 关键点：完全二叉树 n 为奇数时，n₀ = (n+1)/2；n 为偶数时，n₀ = n/2。背后的原理：n₀ = n₂ + 1，而完全二叉树中 n₁ 只能是0或1。

1

二叉树遍历（先序·中序·后序）简答·10分

🚶 第一步：大白话引入

三种遍历就像三种逛公园的路线：

  • 先序：到了景点先拍照（自己），再逛左边，最后右边。顺序：我→左→右
  • 中序：先逛完左边，回来拍照（自己），再逛右边。顺序：左→我→右
  • 后序：先逛左边，再逛右边，最后回来拍照（自己）。顺序：左→右→我

注意！这三种都是递归的——对每个子树，都用同样的规则。

🔍 第二步：拆解题目的意思

给定一棵树：a是根，b是a的左孩子，c是a的右孩子；b的左孩子是d；d的左孩子是g；c的左孩子是e，右孩子是f。要求写出三种遍历的结果。

📝 第三步：先画出这棵树，再一步一步遍历

先序遍历（我→左→右）：

从根a开始：a → 去左边(b) → b → 去b的左边(d) → d → 去d的左边(g) → g → g的左边没了右边没了，回d → d的右边没了，回b → b的右边没了，回a → 去a的右边(c) → c → 去c的左边(e) → e → 左右都没了 → 去c的右边(f) → f → 没了

先序：a b d g c e f

中序遍历（左→我→右）：

从根a开始：先不过a → 去左边(b) → 先不过b → 去b的左边(d) → 先不过d → 去d的左边(g) → g（没孩子了，访问g）→ 回d（访问d）→ d的右边没了 → 回b（访问b）→ b的右边没了 → 回a（访问a）→ 去a的右边(c) → 先不过c → 去c的左边(e) → e（访问e）→ 回c（访问c）→ 去c的右边(f) → f（访问f）

中序：g d b a e c f

后序遍历（左→右→我）：

从根a开始：先不过a → 去左边 → 去左边 → 去左边(g) → g（访问g）→ 回d → d的右边没了 → d（访问d）→ 回b → b的右边没了 → b（访问b）→ 回a → 去右边(c) → 先不过c → 去c的左边(e) → e（访问e）→ 回c → 去c的右边(f) → f（访问f）→ c（访问c）→ a（访问a）

后序：g d b e f c a

先序：a b d g c e f  |  中序：g d b a e c f  |  后序：g d b e f c a
🔑 关键点：三种遍历的区别只在于"访问自己的时机"。先序=进去就访问，中序=左边回来再访问，后序=两边都回来再访问。都是递归的！

哈夫曼树构造 & WPL=136简答·10分

📦 第一步：大白话引入

假设你要给一篇文章里的每个字母编码（用0和1）。出现次数多的字母——比如字母'e'——你希望给它一个短编码（比如"01"），因为要省空间；出现次数少的字母——比如'z'——可以给长一点的编码（比如"11010"）。

哈夫曼树就是用来做这件事的！它的核心思想：出现次数越多，离根越近，编码越短。

🔍 第二步：拆解题目的意思

题目给了一组权值（比如 5, 10, 15, 20, 25, 30 之类的——每个权值代表一个字符的出现频率），要求：构造哈夫曼树，并计算 WPL（带权路径长度）。

📝 第三步：一步一步建树 + 算WPL

哈夫曼树的构造规则：

① 把所有权值看成独立的"小树"（只有一个结点）。

② 每次选权值最小的两棵树，合并成一棵新树（新树的权值 = 两者之和）。

③ 重复第②步，直到只剩一棵树。

哈夫曼树：叶子是原始权值，内部节点是合并后的权值。WPL = 每个叶子权值 × 到根的边数，全部加起来。

WPL计算 题目给出的权值经过上述构造，WPL = 136。这意味着用哈夫曼编码后，平均每个字符需要的二进制位数是最优的。

🔑 关键点：哈夫曼树 = 贪心策略：每次选最小的两个合并。WPL衡量编码效率，越小越好。这是最优前缀编码方案。

双亲表示法 → 树 → 孩子兄弟表示法简答·10分

👨‍👩‍👧 第一步：大白话引入

存储一棵树有三种常见方式，它们就像是三个不同的"花名册"：

  • 双亲表示法：每个结点只记"我爸是谁"。就像派出所登记——每个人只记父母信息。
  • 孩子表示法：每个结点记"我的孩子有哪些"。就像老师点名册——每个学生名下列出他的小组。
  • 孩子兄弟表示法：每个结点有两个指针——左指针指向第一个孩子，右指针指向下一个兄弟。这是最常用的存储方式！
🔍 第二步：拆解题目的意思

题目给了一个双亲表示法的数组，例如：

下标0123456
结点ABCDEFG
双亲1-144100

要求：画出树的结构，然后转换成孩子兄弟表示法。

📝 第三步：一步一步推导

1 双亲表示法解读：B的双亲是-1 → B是根！A的爸是B，E的爸是B，C的爸是E，D的爸是E，F的爸是A，G的爸是A。

2 孩子兄弟表示法的转换规则：

  • 左孩子指针 → 指向该结点的第一个孩子（如果它有孩子的话）
  • 右兄弟指针 → 指向该结点的下一个兄弟（同父母的比自己小的）

这样一来，任何树都可以用二叉树的形式存储！这就是孩子兄弟表示法的妙处。

见上图：双亲数组→树结构→孩子兄弟链表
🔑 关键点：双亲表示法=只记爸爸；孩子兄弟表示法=左孩子右兄弟。任何树都可以通过孩子兄弟表示法变成一棵二叉树，这是树和二叉树之间的桥梁。

双向链表删除头结点编程·10分

🔗 第一步：大白话引入

双向链表就像一排人手拉手——每个人有两只手，左手拉着左边的人，右手拉着右边的人。如果要让第一个人离队：

  • 先让第二个人松开左手（不再拉第一个人）
  • 然后第一个人就可以走了

每个结点有两个指针：prior（前驱，指向前一个）和 next（后继，指向后一个）。

🔍 第二步：拆解题目的意思

题目：写一个函数，删除双向链表的头结点（第一个数据结点，不是头哨兵）。要考虑链表为空的情况。

📝 第三步：一步一步推导
删除头结点A：让头指针H直接指向B，B的prior设为NULL（或指向H），然后释放A。

删除步骤：

1 判断是否为空：如果链表为空（head == NULL 或 head->next == NULL），直接返回。

2 记录要删的结点：p = head（或 p = head->next，取决于有没有头哨兵）。

3 修改指针：

  • 让 head 指向第二个结点：head = p->next
  • 如果第二个结点存在，把它的 prior 设为 NULL（或指向头哨兵）

4 释放内存：free(p)

// 删除双向链表的第一个数据结点 void deleteFirst(DuLinkList *head) { if (*head == NULL) return; // 空链表，啥也不做 DuNode *p = *head; // 记下要删的结点 *head = p->next; // 头指针指向第二个 if (*head != NULL) // 如果第二个存在 (*head)->prior = NULL; // 它的前驱置空 free(p); // 释放 }
核心操作：head = head→next; if(head) head→prior = NULL; free(p);
🔑 关键点：删除链表结点时，一定要先"接好新线"再"剪旧线"——即先修改指针，再释放内存。还要考虑边界：空链表、只有一个结点的情况。

合并两个有序顺序表编程·10分

🤝 第一步：大白话引入

想象你有两叠从小到大排好序的扑克牌。现在要把它们合并成一叠，依然从小到大。

你会怎么做？肯定是：左手拿第一叠的第一张，右手拿第二叠的第一张，比较大小，谁小就把谁放出去。然后从小的那叠再拿下一张，继续比较……

这就是归并（Merge）的核心思想！

🔍 第二步：拆解题目的意思

两个顺序表 L1 和 L2，元素都是从小到大排好的。要求把它们合并成一个新的顺序表 L3，依然保持有序。

📝 第三步：一步一步推导

核心算法 两个指针 i 和 j：

  • 当 L1[i] ≤ L2[j]：把 L1[i] 放进 L3，i++
  • 当 L1[i] > L2[j]：把 L2[j] 放进 L3，j++
  • 当某一方先走完：把另一方剩下的全部复制到 L3
void Merge(SqList L1, SqList L2, SqList *L3) { int i = 0, j = 0, k = 0; while (i < L1.length && j < L2.length) { if (L1.data[i] <= L2.data[j]) L3->data[k++] = L1.data[i++]; // L1的更小，放L1的 else L3->data[k++] = L2.data[j++]; // L2的更小，放L2的 } while (i < L1.length) L3->data[k++] = L1.data[i++]; // L1剩的 while (j < L2.length) L3->data[k++] = L2.data[j++]; // L2剩的 L3->length = k; }
归并算法如上。时间复杂度 O(m+n)，空间复杂度 O(m+n)。
🔑 关键点：归并=双指针+比较+复制。谁小放谁，一方耗尽则全量复制。这是归并排序的核心操作，也是很多合并问题的通用模板。

后序遍历销毁二叉树编程·10分

🧹 第一步：大白话引入

你要拆掉一栋楼。能先拆一楼吗？不行！一楼上面还压着二楼三楼呢，先拆一楼整栋楼就塌了。

正确的拆法：先拆顶层（叶子），一层一层往下拆，最后拆地基（根）。

销毁二叉树也是一样：必须用后序遍历——先销毁左子树，再销毁右子树，最后销毁根结点。因为如果先销毁根，你就找不到它的孩子们了！

🔍 第二步：拆解题目的意思

写一个函数销毁整棵二叉树，释放所有结点的内存。必须用后序遍历，否则会丢失孩子结点的引用。

📝 第三步：一步一步推导
销毁顺序：①A → ②B → ③左子树销毁完 → ④"左"结点销毁 → ⑤C → ⑥"右"结点销毁 → ⑦根销毁。后序保证：孩子全销毁了才销毁父母。

✗ 为什么不能先序？ 先序是 根→左→右。如果先销毁了根，你就没有根了，找不到它的左右孩子！除非你先存下孩子的地址——但那多此一举。

✗ 为什么不能中序？ 中序是 左→根→右。销毁左子树后销毁根，然后……右子树找不到了！因为要通过根才能访问右孩子。

// 后序遍历销毁二叉树 void destroyTree(BiTree *T) { if (*T == NULL) return; // 空树，啥也不做 destroyTree(&((*T)->lchild)); // ① 先销毁左子树 destroyTree(&((*T)->rchild)); // ② 再销毁右子树 free(*T); // ③ 最后销毁根结点 *T = NULL; // 把指针置空（好习惯） }
后序遍历销毁：先递归销毁左右子树，再释放根结点。
🔑 关键点：销毁树必须用后序遍历——先孩子后父母。就像一个楼，从顶层拆到底层。递归函数中，free根结点之前，必须确保它的孩子们已经被释放了。

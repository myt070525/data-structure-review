线性表的插入位置

在长度为 n 的线性表的第 i 个位置插入元素，i 的合法值应为？
✓ 正确答案
C　1 ≤ i ≤ n+1

🧠 为什么是 n+1 而不是 n？

想象你有 n = 5 个元素的线性表。元素编号从 1 到 5。现在你想插入一个新元素，一共有多少种插法？

❌ 错误想法
只能在 1~5 之间插入
5 个位置
✅ 正确思路
可以在头、尾、任意中间插入
n+1 = 6 个位置

关键看清：第 n+1 个位置 = 紧贴表尾后面，也就是追加一个元素。插入位置是从最前面（第1位之前不能插吗？可以！）到最后面（第 n+1 位）。

👁️ 可视化：n=5 时的 6 个插入位置

原始表（5个元素）：

位置 → 12345
数据 → ABCDE

↕ 可插入位置（▼ 箭头所指）：

i=1 — 插在最前面（A之前）

▼ABCDE

i=2, 3, 4, 5 — 插在中间（i=2 即 A 之后、B 之前）

A▼BC…E

i=6 = n+1 — 插在最后面（E之后，追加）

ABCDE▼

🔍 逐选项辨析

A 1 ≤ i ≤ n ❌ 缺少表尾插入位置 不能追加到末尾
B 0 ≤ i ≤ n ❌ 多出 i=0 缺少 n+1 编号通常从1开始，且仍缺表尾
C 1 ≤ i ≤ n+1 ✅ 正确！ 包含所有合法插入位置
D 1 ≤ i ≤ n-1 ❌ 缺少最后两个位置 缺得最多，最不可能

📐 插入 vs 删除 —— 容易混淆！

插入
1 ≤ i ≤ n+1
可以追加在末尾之后 → 多一个位置
删除
1 ≤ i ≤ n
只能删已有的 → 正好 n 个
💡 记忆口诀：插比删多一
插入 = 站队伍任意位置或队伍最末端（n+1 个缝隙）
删除 = 只能踢队伍中某个已有的人（n 个目标）

💻 代码验证 —— 顺序表插入

典型顺序表插入代码，注意对 i 的合法性检查：

int ListInsert(SeqList *L, int i, int x) {
    if (i < 1 || i > L->length + 1) // ← 检查：i 必须在 1..n+1 之间
        return 0; // 非法位置，返回失败
    // ... 后移元素，写入 x，长度+1
}

注意关键条件：i > L->length + 1 即 i > n+1 才非法 → 合法的是 i ≤ n+1

🎯 一句话总结：n 个元素之间有 n+1 个"缝隙"可以插入——最前面、每两个之间、最后面，所以插入位置从 1 到 n+1。

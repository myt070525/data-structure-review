1出现3次时间复杂度分析
测试题填空1 for(i=0;i<1000;i++) for(j=0;j<i;j++) sum+=i+j → O(1)
测验选择1 for(i=0;i<n;i++) for(j=0;j<m;j++) sum+=a[i][j] → O(mn)
考试选择1 int x=1; while(n>=x*x) x*=2 → O(log n)
  • 单层循环n次 → O(n)
  • 双层嵌套 → 相乘 O(n×m)。不是相加！
  • 并列循环 → 相加 O(n+m)。不是相乘！
  • x翻倍/翻倍增长 → O(log n)。"每次×2"就是对数级
  • 循环次数是常数（如1000次，与n无关）→ O(1)
  • while(n>=x²) x翻倍 → x≈log₂(√n)=½log₂n → O(log n)
先看有没有嵌套→相乘；再看是不是常数→O(1)；最后看增长方式→翻倍=log，线性=n。不要死记，理解"执行次数与n的关系"。
2出现3次出栈序列 / 栈的特性
测试题选择4 入栈ABCD，C开头出栈序列个数 → 3个（CBA?不，是CBA/CBDA/CDBA?等。C开头=CBAD?）→ 具体枚举
测验选择3 入栈abcde，d开头出栈序列个数 → 4个（dcbae/dcbea/dceba/decba）
考试选择4 入栈ABCD，不可能出栈 → BDAC（B出→A在栈底；D出→C在栈中；要A出但C在栈顶挡住）
  • 栈 = 后进先出 LIFO。只能操作栈顶。
  • 出栈序列判定方法：模拟！用一个栈实际推演。遇到"不可能的序列"→找矛盾：后面的小元素压在栈底被大元素挡住了。
  • "X第一个出栈"题型：X要第一个出来→比X早入栈的元素全压在栈里，X之后入栈的元素还在外面。枚举时栈里的必须按"后进先出"顺序弹出。
  • n个元素全部出栈序列数 = 卡特兰数 C(2n,n)/(n+1)。n=4时为14种。但带限制条件（如"X开头"）需要枚举。
枚举d开头（abcde入栈）的4种序列： d先出→栈中[a,b,c]栈顶c，外[e] ① d→c→b→a→e入出 → dcbae ② d→c→b→e入出→a → dcbea ③ d→c→e入出→b→a → dceba ④ d→e入出→c→b→a → decba
"X第一个出"=比X先入的都被压在栈里。按后进先出顺序弹，比X后入的可以穿插弹。枚举时画一个栈帧逐列推演。
3出现3次循环队列判空/元素个数
测试题选择5 循环队列判空 → Front == Rear
测验选择4 size=9, front=7, rear=2 → 元素个数 4
考试选择5 队空时front==rear，位置 → 在同一位置，任意下标都有可能
  • 循环队列 = 把数组头尾"接"成一个圈。目的：复用出队后空出的位置。
  • 判空：front == rear
  • 判满：(rear + 1) % size == front（牺牲一个位置）
  • 元素个数：(rear - front + size) % size
  • 重点：空队时front和rear重合，但不一定在0号位置！它们在哪儿取决于操作历史。
公式验证：(2-7+9)%9 = 4%9 = 4。记住"rear减front，负数加size，最后取模"。
4出现3次后缀表达式求值 / 中缀转后缀
测试题填空2 8*(5-2)/(7+3) → 后缀 852-*73+/；扫描到+时栈中 3个元素
测验选择6 5 2 3 1 + * 2 / + → 9
考试填空3 23*7-64+* → 处理完+时栈中 10,-1；最终值 -10
  • 后缀求值规则：遇数字→入栈；遇运算符→弹出两个数（先弹出的是右操作数！）→计算→结果入栈。
  • 中缀转后缀规则：数字直接输出；运算符比较优先级——当前运算符≤栈顶时，栈顶出栈；否则入栈。左括号直接入栈，遇右括号弹到左括号。
  • 优先级：* / > + -；同级左结合（栈顶优先级≥当前就弹出栈顶）
  • 易错：后缀求值时，先弹出的是第二个操作数！"a-b"→后缀"ab-"，求值：弹b，弹a，算a-b。
后缀 5 2 3 1 + * 2 / + 求值过程： [5]→[5,2]→[5,2,3]→[5,2,3,1]→+弹出1和3→3+1=4→[5,2,4] →*弹出4和2→2*4=8→[5,8]→[5,8,2]→/弹出2和8→8/2=4→[5,4] →+弹出4和5→5+4=9→[9] 答案=9
后缀求值=栈模拟。每次运算符弹出两个，第二个弹出的是左操作数（减法和除法不能弄反！）
5出现3次KMP算法 · next数组 + 比较次数
测试题填空3 T="aabaa" → next=01212；S="acbacbaaabaab" → 比较15次
测验选择7 T="abaaba" → next=011223（或-100112）；S="abacaabaabaa" → 比较14次
考试填空4 T="aaaba" → next=01231；S="aabacbaaabaab" → 比较14次
  • next[j]含义：模式串第j位失配时，模式串跳到哪个位置继续比较（主串指针不动）。
  • 手算next（下标从1开始，next[1]=0）：
    next[j] = "前j-1个字符构成的子串的最长相等前后缀长度 + 1"
  • KMP匹配过程：i(主串)和j(模式)开始比较。相等→都+1。不等→j=next[j]；若j=0→i++,j=1。j超过模式串长度→匹配成功。
  • 比较次数：每次i和j指向字符的比较算1次。KMP的优势是i不回退，总比较次数≈O(n+m)。
手算 "abaaba" 的next（下标从1开始）： j=1: next[1]=0（约定） j=2: 前1个"a"，无相等前后缀→next[2]=1 j=3: 前2个"ab"，前缀{a}≠后缀{b}→next[3]=1 j=4: 前3个"aba"，前缀{a,ab} 后缀{ba,a}→匹配"a"长度1→next[4]=2 j=5: 前4个"abaa"，前缀{a,ab,aba} 后缀{baa,aa,a}→匹配"a"长度1→next[5]=2 j=6: 前5个"abaab"，前缀{a,ab,aba,abaa} 后缀{baab,aab,ab,b}→匹配"ab"长度2→next[6]=3 结果：0 1 1 2 2 3（有的教材写-1 0 0 1 1 2，差1而已）
next[j]手算公式：看前j-1个字符的最长相等前后缀长度L，next[j]=L+1。考试时直接按此公式手算即可。
6出现3次快排一趟结果判定
测试题选择8 不可能 → A: 27,13,2,5,19（27最大却在第一位→没归位）
考试选择9 不可能 → A: 18,10,2,88,71（全都不满足"左边≤它≤右边"）
测验无直接 但知识点高频！
  • 一趟快排后：存在一个基准（pivot），它左边所有元素≤它，右边所有元素≥它，且它在排序后的最终位置。
  • 判定方法：对序列中每个位置假设它是基准，检查是否满足条件。如果所有位置都不满足→这个序列不可能是一趟快排结果。
  • 关键：基准必须归位！最大值却在开头=没归位；最小值却在末尾=没归位。
例：27,13,2,5,19 为什么不可能？ 假设27是基准：27在位置1，排序后27最大应在位置5→没归位✗ 假设19是基准：19在位置5，左边27>19→不满足"左边≤基准"✗ 任何位置都不满足→不可能！ 例：5,26,28,47,15 为什么可能？ 假设5是基准：5最小在位置1，排序后在位置1→已归位，右边都≥5✓
先找全局最大/最小值的位置——如果最大在开头或中间、最小在末尾或中间，直接不可能（因为一趟后最大必在最后，最小必在最前）。
7出现2+次排序算法对比（稳定性/空间/时间）
测试题选择10 空间少+平均快 → 堆排序（O(1)空间+O(nlogn)时间）
考试选择10 不稳定的是 → 堆排序（选择/希尔/快排也不稳定）
测试题填空8 冒泡排序平均时间复杂度 → O(n²)
算法平均时间最坏空间稳定记忆点
冒泡O(n²)O(n²)O(1)✅最大"浮"到最后
插入O(n²)O(n²)O(1)✅抓扑克牌
选择O(n²)O(n²)O(1)❌每趟选最小
希尔O(n1.3)O(n²)O(1)❌分组插入
快速O(nlogn)O(n²)O(logn)❌基准归位
堆O(nlogn)O(nlogn)O(1)❌建堆+反复删顶
归并O(nlogn)O(nlogn)O(n)✅分治+双指针合并
基数O(d(n+r))—O(n+r)✅按位分配收集
  • 不稳定四兄弟：选（选择）希（希尔）快（快排）堆（堆排序）——谐音"选西快堆"
  • 空间对比：堆O(1)＜快排O(logn)栈＜归并O(n)辅助
  • 时间复杂度不受输入影响的（最好=最坏）：选择O(n²)、归并O(nlogn)、堆O(nlogn)
选择题常常这样问："平均O(nlogn)且空间O(1)"→一定是堆排序；"稳定且O(nlogn)"→归并；"空间最少"→O(1)的那几个（冒泡/插入/选择/希尔/堆）。
8出现3次ADT / 逻辑结构 / 存储结构
测试题选择1 ADT不包括 → 数据的存储结构
测验选择2 ADT三要素缺 → 数据关系
考试选择2 不属于基本逻辑结构 → 哈希结构
  • ADT三要素：数据对象 + 数据关系 + 基本操作集。不含存储结构！
  • 四类逻辑结构：集合（无关系）、线性（1对1）、树（1对多）、图（多对多）。哈希结构是存储结构不是逻辑结构！
  • 两类物理（存储）结构：顺序存储（数组）、链式存储（链表）。
  • 算法五特性：有穷性（填空高频！）、确定性、可行性、输入（≥0）、输出（≥1）。
ADT就像"说明书"——只告诉你"能干嘛"，不告诉你"怎么造的"。存储结构是"怎么造的"，不属于ADT范畴。逻辑结构=数据之间的关系模式，物理结构=在内存里怎么放。
9出现3次完全二叉树叶子 / 结点数
测验选择9 321个结点 → 叶子 161
考试填空5 126个叶子 → 总结点 251（或252）
测试题填空4 哈夫曼树125结点 → 叶子 63
  • n₀ = n₂ + 1（任何二叉树：叶子数 = 度为2的结点数+1）
  • 完全二叉树的n₁只能是0或1。（因为结点从左到右填满，最多只有一个结点只有左孩子）
  • n = n₀ + n₁ + n₂ = 2n₀ - 1 + n₁
  • 已知叶子数求总结点：n = 2n₀ - 1 + n₁。n₁=0时n=2n₀-1；n₁=1时n=2n₀。两种可能都要考虑！
  • 已知总结点求叶子：n是奇数→n₁=0→n₀=(n+1)/2；n是偶数→n₁=1→n₀=n/2
  • 哈夫曼树：只有度0和度2的结点(n₁=0) → n₀=(n+1)/2
完全二叉树公式链：n₀=n₂+1 → n=2n₀-1+n₁ → 看n的奇偶性定n₁。n奇数则n₁=0，n偶数则n₁=1。
10出现2次广义表 Head / Tail 运算
测试题选择7 A=((a,b),(c,(d),((e,f),g))) → Head(Tail(Head(Tail(A))))=(d)
测验选择5 ((a,b),(a,(c,d)))表尾 → ((a,(c,d)))
  • Head(表) = 取第一个元素。如果第一个元素是原子→返回原子；是子表→返回子表（保持原样）。
  • Tail(表) = 去掉第一个元素后剩下的，一定加一层括号。即使只剩一个元素也必须加括号！
  • 运算顺序：从最内层括号开始，一层层往外算。
A = ((a,b), (c,(d),((e,f),g))) ① Tail(A) = ((c,(d),((e,f),g))) //去掉(a,b)，加括号 ② Head(①) = (c,(d),((e,f),g)) //取第一个元素 ③ Tail(②) = ((d),((e,f),g)) //去掉c，加括号 ④ Head(③) = (d) //取第一个元素 → 答案
表尾=去首加括号。逐层运算，从里到外。每一步只做一个操作，不要跳步。
11出现2次哈夫曼树 WPL / 结点关系
测试题填空4 哈夫曼树125个结点 → 叶结点 63
测验简答2 WPL=(3+7)×3+(15+16+22)×2=136
  • 哈夫曼树性质：只有度0和度2的结点（n₁=0）。n = n₀ + n₂ = n₀ + (n₀-1) = 2n₀-1 → n₀ = (n+1)/2
  • WPL = Σ(叶结点权值 × 从根到该叶的路径长度)
  • 构造方法：每次选权值最小的两棵树合并，新树权值=两树之和。重复直到只剩一棵树。
  • 带权路径长度WPL：就是所有叶子的权值×它到根的边数，然后求和。
  • 注意：合并时左小右大（有的教材不要求），但WPL结果是一样的。
哈夫曼树结点公式：n=2n₀-1。125个结点→n₀=63。WPL计算时一定要逐层：第k层的权值×k，然后加起来。
12出现2次哈希表 · 装填因子 · 线性探测
考试填空6 哈希表查找效率三个因素 → 哈希函数、冲突处理方法、装填因子
测试题简答4 H(key)=key%11, 线性探测 → ASL=10/7
  • 装填因子 α = 记录数 / 表长。α越大→表越满→冲突越多→查找越慢。
  • 哈希表冲突处理：开放地址法（线性探测：顺位找下一个空位）、链地址法（每个槽挂链表）。
  • ASL成功 = Σ(每个记录的查找次数) / 记录总数
  • 线性探测：H(key)冲突→试H(key)+1→再冲突→H(key)+2→...直到找到空位
  • 注意：线性探测容易产生"聚集"（clustering）——连续被占导致探测越来越长。
装填因子是填空题高频考点。三个因素：哈希函数好坏、冲突处理方式、装填因子（第三个最常被挖空）。
13出现2次线性表：插入/删除/判空
测试题选择2 第i位插入，i合法范围 → 1 ≤ i ≤ n+1
考试填空2 删除第i个元素需移动 → n-i 个元素
测试题选择3 带头结点循环链表判空 → L->Next == L
  • 插入位置范围：1到n+1（n+1=队尾追加）
  • 删除第i个需移动 n-i 个元素（后面的全部前移）
  • 插入第i个需移动 n-i+1 个元素
  • 循环链表空：头结点的Next指向自己→L->Next == L
  • 头结点的作用：统一空表和非空表的操作，避免对第一个结点的特殊处理。
  • 链表适合：频繁插入删除（O(1)只改指针）。顺序表适合：频繁随机访问（O(1)直接下标）。
n个人排队，可插位置=n个"两个人之间"+1个"第一个人之前"+1个"最后一个人之后"? 不对——是n+1个：第1位前、第1-2之间、...、第n位后。
14出现2次森林与树的边/结点关系
测试题填空5 森林15条边、25个结点 → 10棵树
测验选择8 中序ABCDEFG+后序BDCAFGE → 左子树4个结点
  • 每棵树：边数 = 结点数 - 1
  • k棵树组成的森林：总边数 = 总结点数 - k → k = 总结点数 - 总边数
  • 25个结点、15条边 → k = 25 - 15 = 10棵树
  • 二叉树重建：后序最后=根，中序中根左边=左子树，右边=右子树。递归分割。
森林题公式：树数 = 结点数 - 边数。每多一棵树就比"一棵大树"少一条边。
15出现2次图的存储 & 连通性判断
测试题选择9 邻接矩阵 → 便于判断顶点间是否有边（O(1)直接查）
考试选择8 不能判断无向图连通性 → 拓扑排序
  • 邻接矩阵：edge[i][j]直接查→O(1)判边。空间O(n²)。适合稠密图。加删顶点O(n²)。
  • 邻接表：每个顶点挂链表。空间O(n+e)。适合稀疏图。判边需遍历链表O(度)。
  • 连通性判断：DFS或BFS遍历后看是否所有顶点都被访问。
  • 拓扑排序只适用于有向无环图(DAG)！无向图根本没有"方向"，不能做拓扑排序。
  • DFS用栈（递归），BFS用队列。
邻接矩阵的强项是"O(1)判边"，弱项是"加删顶点O(n²)"和"稀疏图浪费空间"。拓扑排序=选无前驱的→删，只对DAG有效。
16出现2次最短路径算法（Dijkstra / Floyd）
测试题填空6 单源最短路径 → Dijkstra算法
考试简答6 Floyd算法 → 所有顶点对最短路径
  • Dijkstra：单源最短路径。一个起点到其他所有顶点的最短路径。贪心思想。不能处理负权边。
  • Floyd：所有顶点对最短路径。三重循环动态规划。可以处理负权边（但不能有负权回路）。
  • Prim/Kruskal：最小生成树！不是最短路径！别搞混。
  • 填空题常考：单源最短路径用Dijkstra（或迪杰斯特拉）。
记名字：Dijkstra=单源，Floyd=全源。Prim/Kruskal=最小生成树（所有顶点连通的最小总权值和），和最短路径是两回事。
17出现1-2次双亲表示法 · 树高
考试选择7 双亲数组[7,0,-1,2,3,0,0,3] → 树高5
测验简答3 给定双亲表格 → 画树 & 孩子兄弟表示
  • 双亲表示法：每个结点只存父结点下标。-1=该结点是根。
  • 求树高：对每个结点追溯其到根的路径长度，最大路径长+1=树高。
  • 孩子兄弟表示法（二叉表示）：每个结点：左指针→第一个孩子，右指针→下一个兄弟。
[7,0,-1,2,3,0,0,3] 解读： 下标2= -1 → 根节点 下标0→父7, 下标1→父0, 下标3→父2, 下标4→父3, 下标5→父0, 下标6→父0, 下标7→父3 追溯最深路径：2→3→7→? 等等，重新追溯每条路径找最长。
双亲表示法求树高：对每个结点while(parent[i]!=-1)一直往上追溯到根，计数。最大计数值+1=树高。
18出现1次BST插入序列种数
测试题填空7 BST树（见图片：root=3,左1右4,1的右=2,4的右=5）→ 可能输入序列 6种
  • BST插入序列的约束：祖先必须在子孙之前出现（因为你不能插入一个还不存在的结点的孩子）。
  • 计算方法：根必须在第一个。左右子树各自内部有序（小的在大的前面），但左右子树之间的元素可以交错排列。
  • 这道题：3固定第一个。左子树{1,2}顺序固定(1→2)，右子树{4,5}顺序固定(4→5)。两组交错排列 = C(4,2) = 6种。
BST插入序种数=左右子树元素交错排列的组合数。左子树m个元素，右子树n个元素→C(m+n,m)种。

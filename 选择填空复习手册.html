01

第一章 · 绪论

📌 数据、数据元素、数据项
定义 数据项（最小不可分单元）→ 组成 数据元素（基本单位）→ 组成 数据（集合）。
易错 数据元素是数据的基本单位，数据项是最小标识单位。选择题常混淆二者！
📌 四类逻辑结构 vs 两类物理结构
四类逻辑结构
① 集合 — 数据元素间无关系
② 线性结构 — 一对一（如线性表、栈、队列）
③ 树形结构 — 一对多
④ 图形结构 — 多对多
两类物理结构
① 顺序存储 — 连续内存，数组
② 链式存储 — 不连续，指针链接
注意 哈希结构是存储结构不是逻辑结构！选择题常考！
📌 算法五大特性
必背 ① 有穷性 — 有限步后结束（不是"死循环"）
② 确定性 — 每步明确无歧义
③ 可行性 — 可以执行
④ 输入 — 0个或多个
⑤ 输出 — 至少1个
填空题高频 常挖空"有穷性"！
📌 时间复杂度
核心公式
• 单层循环n次 → O(n)
• 双层嵌套n×m → O(nm)
• 并列循环 → O(n+m)（不是相乘！）
• x翻倍while(x<n) x*=2 → O(log n)
• 常数次循环for(i=0;i<1000) → O(1)（与n无关！）
📌 ADT（抽象数据类型）三要素
必背 数据对象 + 数据关系 + 基本操作集。
陷阱 存储结构不属于ADT！ADT只关心"能做什么"，不关心"怎么存储"。选择题高频！

第二章 · 线性表

📌 顺序表 vs 链表（必考对比）
比较维度顺序表（数组）链表
存储方式连续内存不连续，指针链接
随机访问O(1) ⚡O(n) 🐢
插入/删除O(n) 🐢（移动元素）O(1) ⚡（改指针）
空间开销0（纯数据）每结点多一个指针
缓存友好✅ 连续好❌ 跳跃差
结论 频繁查找/随机访问→顺序表；频繁插入/删除→链表。选择题必考！
📌 顺序表关键公式
公式 • 第i个元素地址 = 基地址 + (i-1) × 元素大小（逻辑相邻=物理相邻）
• 插入第i位需移动 n-i+1 个元素
• 删除第i位需移动 n-i 个元素
• i的合法范围：1 ≤ i ≤ n+1（插入可在队尾！）
📌 头结点 & 循环链表判空
头结点作用 统一空表和非空表的操作（避免特殊判断），不存数据。
判空公式
• 带头结点单链表空：L->Next == NULL
• 带头结点循环链表空：L->Next == L（自己指自己！）

第三章 · 数组和广义表

📌 矩阵压缩存储
三种可压缩矩阵 对称矩阵（只存上/下三角）、三角矩阵（只存非零三角）、对角矩阵（带状存储）。
公式 n阶三角矩阵需 n(n+1)/2 个存储单元。
注意 普通稀疏矩阵用三元组或十字链表存储，不是压缩成一维数组！
📌 广义表
定义 可以嵌套的表，元素可以是原子或子表。
Head/Tail规则
• Head = 取第一个元素（可能是原子或表）
• Tail = 去掉第一个元素后剩下的，一定加一层括号（即使只剩一个）
陷阱 表尾一定是表（有括号）！表头不一定。
📌 串 & KMP
BF算法 暴力匹配，O(nm)，每次失配主串回退。
KMP核心 主串指针不回退，利用next数组跳转。O(n+m)。
next数组 表示"当前字符失配时，模式串应跳到哪里继续比较"。手算方法：找最长相等前后缀长度+1。

第四章 · 栈和队列

📌 栈（Stack）— 后进先出 LIFO
核心规则
• 只能在栈顶操作
• push（入栈）/ pop（出栈）
• 出栈序列计数：卡特兰数 C(2n,n)/(n+1)
• 栈空：top == -1（顺序栈）
出栈序列判定
方法：用栈模拟！
"d第一个出"→a,b,c已压在栈底，d出→再考虑后续。
枚举法：必须满足"大不能压小"。
📌 队列（Queue）— 先进先出 FIFO
循环队列
• 目的：复用数组空间
• 队空：front == rear
• 队满：(rear+1)%size == front（牺牲一个位置）
• 元素个数 = (rear-front+size)%size
重要
• 空队时front和rear不一定在下标0
• 它们可以在任意相同位置（取决于之前的操作历史）
• BFS用队列，DFS用栈（或递归）
📌 表达式转换（必考）
三种表达式
• 中缀：a+b（人读的）
• 前缀：+ab（运算符在前）
• 后缀：ab+（运算符在后，最常用）
求值方法 遇到数字→入栈；遇到运算符→弹两个→运算→结果入栈。
转换方法 中缀→后缀：数字直接输出，运算符比较优先级入栈/出栈。

第五章 · 树和二叉树

📌 树的基本性质
必背公式
• 结点数 = 总度数 + 1
• n个结点的树有 n-1 条边
• 度为m的树：第i层最多 mi-1 个结点
• 深度为k的二叉树最多 2k-1 个结点
📌 二叉树性质（高频）
核心三公式
① n₀ = n₂ + 1（叶子数 = 度为2的结点数+1）
② 满二叉树：每层都满，深度k有2k-1个结点
③ 完全二叉树：从左上到右下填满，n₁只能是0或1
常见题 已知叶子数求总结点数：n = n₀+n₂+n₁ = n₀+(n₀-1)+n₁ = 2n₀-1+n₁（n₁=0或1）
📌 三种遍历（必须会）
遍历方式顺序口诀
先序（Preorder）根 → 左 → 右"自己先来"
中序（Inorder）左 → 根 → 右"自己在中间"
后序（Postorder）左 → 右 → 根"自己最后"
重建二叉树 已知先序+中序或后序+中序可唯一确定。先序/后序找根，中序分左右。
注意 仅先序+后序不能唯一确定二叉树！
📌 存储结构 & 树与二叉树转换
三种存储
① 双亲表示法：每个结点只记父结点下标（-1=根）
② 孩子表示法：每个结点记所有孩子
③ 孩子兄弟表示法：左孩子+右兄弟 = 二叉表示
树→二叉树
"左孩子右兄弟"规则：
• 每个结点的第一个孩子→左子树
• 每个结点的下一个兄弟→右子树
📌 哈夫曼树（Huffman）
构造 每次选权值最小的两棵树合并，新树权值=两子之和。重复直到只剩一棵。
WPL = Σ(叶结点权值 × 从根到叶的路径长度)
性质 哈夫曼树只有度0和度2的结点：n = 2n₀ - 1 → n₀ = (n+1)/2

第六章 · 图

📌 图的存储方式
邻接矩阵
• 判断边 O(1) ⚡
• 空间 O(n²)，适合稠密图
• 加删顶点 O(n²) 🐢
• 无向图矩阵对称
邻接表
• 找邻接点 O(度) ⚡
• 空间 O(n+e)，适合稀疏图
• 加删顶点灵活
• 判断边需遍历链表 🐢
📌 遍历 & 连通性
DFS 深度优先 → 用栈（或递归）
BFS 广度优先 → 用队列
连通性判断：DFS/BFS遍历后看是否所有顶点都被访问。
注意 拓扑排序只适用于有向无环图(DAG)，不能用来判断无向图连通性！选择题陷阱！

第七章 · 查找

📌 查找算法对比
方法适用ASL成功时间复杂度
顺序查找无序/有序均可(n+1)/2O(n)
折半查找仅有序顺序表≈log₂(n+1)-1O(log n)
分块查找块间有序介于两者之间O(√n)~O(log n)
注意 折半查找只能用顺序存储（需要随机访问），链表不行！
📌 折半查找判定树
构造 每次取中间元素为根。n个元素→判定树深度 ≈ ⌊log₂n⌋+1。
ASL = Σ(每层结点数 × 该层查找次数) / n
特点 判定树是平衡二叉搜索树，但不一定是完全二叉树。
📌 哈希表（Hash）
三要素 ①哈希函数 ②处理冲突方法 ③装填因子α（=记录数/表长）。α越大冲突越多。
冲突处理
• 开放地址法：线性探测(顺位找下一个)、二次探测(±1²,±2²...)
• 链地址法：每个槽位挂一个链表
ASL = Σ(每个记录的查找次数) / 记录总数
填空题 查找效率取决于：哈希函数、冲突处理方法、装填因子。
📌 B-树 vs B+树
对比B-树B+树
数据存储所有结点都存数据仅叶子存数据
叶子链接不链接叶子串成链表（范围查询快）
关键字每个关键字只出现一次非叶关键字会重复出现在叶

第八章 · 排序

📌 排序算法全对比（必考！）
算法最好平均最坏空间稳定？
冒泡排序O(n)O(n²)O(n²)O(1)✅ 稳定
插入排序O(n)O(n²)O(n²)O(1)✅ 稳定
选择排序O(n²)O(n²)O(n²)O(1)❌ 不稳定
希尔排序O(n)O(n1.3)O(n²)O(1)❌ 不稳定
快速排序O(nlogn)O(nlogn)O(n²)O(logn)❌ 不稳定
堆排序O(nlogn)O(nlogn)O(nlogn)O(1)❌ 不稳定
归并排序O(nlogn)O(nlogn)O(nlogn)O(n)✅ 稳定
基数排序O(d(n+r))O(d(n+r))O(d(n+r))O(n+r)✅ 稳定
📌 稳定性是什么？
定义 相等关键字的记录，排序后相对顺序不变。
稳定 冒泡、插入、归并、基数
不稳定 选择、希尔、快排、堆排序（难点！）
原因 堆排序在建堆和调整时可能跨距离交换，破坏相同值的相对顺序。
📌 快排一趟判定（常考选择题）
判定方法 一趟快排后，存在一个基准元素，左边全≤它，右边全≥它，且它在最终排序位置。
解题 逐个位置假设它是基准，检查条件。如果所有位置都不满足→这个序列不可能是一趟快排结果。
📌 堆 & 堆排序
大顶堆 每个父结点 ≥ 所有孩子。堆顶=最大值。
小顶堆 每个父结点 ≤ 所有孩子。堆顶=最小值。
特点 建堆O(n)，每次删除O(logn)。堆排序=建堆+反复删除堆顶。
选择题 堆排序不稳定！空间复杂度O(1)！平均时间O(nlogn)！

1ADT 不包括什么？

题目原文：ADT（抽象数据类型）不包括什么？（ ）
A. 数据对象 B. 数据结构 C. 数据操作 D. 存储结构

🎯 生活类比：手机说明书 vs 手机芯片

想象你去买手机。柜台上有本说明书，上面写着："可以打电话、发微信、拍照、上网"。你会拿这台手机做什么，说明书全告诉你。但说明书不会告诉你里面用的是骁龙8还是A16芯片，也不会告诉你内存颗粒是三星还是镁光。

ADT（抽象数据类型）就是这本说明书。它只描述"这玩意儿能干嘛"，不描述"它内部怎么造的"。而"怎么造的"——用什么存储结构、内存怎么布局——属于物理实现，不在说明书范围里。

🔍 拆解题意

ADT三要素 ① 数据对象（Data Object）—— 这结构存的是什么东西？整数？人？
② 数据关系（Data Relationship）—— 这些东西之间是什么逻辑关系？线性的？树形的？
③ 基本操作（Basic Operations）—— 能对它做什么？插入？删除？查找？
⚠ 注意：存储结构不在三要素里！那是物理实现层面的事。

✍️ 逐步推导

第1步 题目问：ADT不包括什么？
第2步 A. 数据对象 → ❌ 是三要素之一
第3步 B. 数据结构（此处指数据关系）→ ❌ 是三要素之一
第4步 C. 数据操作 → ❌ 是三要素之一
第5步 D. 存储结构 → ✅ 不在三要素中，属于实现细节
答案：D
📝 ADT = 说明书，只管"能干嘛"；存储结构 = 零件表，管"怎么造"——两者不是一回事。

2线性表在第 i 位插入，i 的合法范围是？

题目原文：线性表在第 i 个位置插入新元素，i 的合法范围是？（ ）
A. 1≤i≤n B. 1≤i≤n-1 C. 1≤i≤n+1 D. 0≤i≤n

🎯 生活类比：插队

想象你在食堂排队，队伍已经有 n 个人了。你想插队——你能插在哪些位置？

🔍 拆解题意

线性表有 n 个元素，位置编号从 1 到 n。插入一个新元素，你可以：

• 插在最前面（原来第1个变成第2个）→ i = 1
• 插在任意两个之间 → i = 2, 3, ..., n
• 插在最后面（追加在队尾）→ i = n + 1
所以合法 i 的范围是：1 ≤ i ≤ n + 1
答案：C（1 ≤ i ≤ n+1）
📝 n 个人的队伍有 n+1 个可插入位置——你可以在任何人前面插，也可以在队尾追加。

3带头结点循环链表判空的条件是？

题目原文：带头结点的循环链表 L 判空的条件是？（ ）
A. L->Next==NULL B. L->Next==L C. L->Next==Head D. L->Next!=L

🎯 生活类比：一个人自己牵自己的手

想象一群小朋友手拉手围成圈做游戏。如果只有一个小朋友，他怎么围成圈？——他左手牵右手，自己牵自己，形成一个圈！

带头结点的循环链表也一样：头结点是那个"领队"。当链表为空（没有真正的小朋友），头结点只好"自己指回自己"——L->Next == L。

✍️ 逐个选项分析

A L->Next == NULL → ❌ 循环链表没有NULL，最后一个结点指向头结点
B L->Next == L → ✅ 头结点的Next指向自己，说明没有其他结点
C L->Next == Head → ❌ Head和L是同一个东西，表述重复
D L->Next != L → ❌ 这恰恰是非空的标志
答案：B
📝 空循环链表 = 头结点自己指自己，就像一个小朋友左手牵右手围成圈。

4入栈顺序 A→B→C→D，C 第一个出栈，不同出栈序列有几个？

题目原文：入栈顺序为 A→B→C→D，若 C 第一个出栈，则不同的出栈序列有几个？（ ）
A. 2 B. 3 C. 4 D. 5

🎯 生活类比：叠盘子

你把 A、B、C、D 四个盘子依次叠上去（A在最底下，D在最上面）。拿盘子只能从最上面拿——这就是栈。

现在规定：C 必须第一个被拿走。这意味着什么？

🔍 拆解题意

关键推理 C 第一个出栈 → C 拿的时候，A 和 B 还在栈里压着，而且 A 在 B 下面。
所以 A 和 B 的出栈顺序只能是 B 先出、A 后出（因为 B 压在 A 上面，不拿 B 就拿不到 A）。
而 D 呢？D 是最后入栈的——它可以在 C 出栈之前入栈然后立刻出，也可以在 C 出栈之后再入栈。

✍️ 枚举所有合法序列

序列① C → D → B → A
入A,入B,入C,出C,入D,出D,出B,出A
序列② C → B → D → A
入A,入B,入C,出C,出B,入D,出D,出A
序列③ C → B → A → D
入A,入B,入C,出C,出B,出A,入D,出D
为什么 C, A, B, D 不行？—— C 出栈后，栈里还剩 A(底) 和 B(顶)。必须先拿 B 才能拿 A，所以 A 不可能在 B 前面出。
答案：C（3种）
📝 C 先出 → A、B 被压在栈底，只能 B→A 顺序出；D 可在任意时机出入，共 3 种合法序列。

5循环队列判空的条件是？

题目原文：循环队列判空的条件是？（ ）
A. Front==Rear B. Front==(Rear+1)%Size C. Rear==(Front+1)%Size D. Front==0

🎯 生活类比：环形跑道上的两个人

想象一个环形跑道上，有两个人：Front（队头）和Rear（队尾）。队列为空时，两人站在同一个位置——队头和队尾重合。

✍️ 关键对比

判空 Front == Rear → 队头和队尾重合，没有元素
判满 (Rear + 1) % Size == Front → 故意牺牲一个位置来区分空和满
（如果不牺牲一个位置，满的时候 Front==Rear 也和空一样，无法区分）
答案：A（Front == Rear）
📝 循环队列空 → 头尾重合；满 → 尾+1追上头（牺牲一格做标记）。

6三角矩阵压缩存储需要多少空间？

题目原文：n 阶下三角矩阵压缩存储需要多少存储单元？（ ）
A. n² B. n(n-1)/2 C. n(n+1)/2 D. n²/2

🎯 生活类比：阶梯座位

想象教室里有 n 排座位，第1排1个座位，第2排2个，……，第n排n个。总共几个座位？

这就是下三角矩阵——主对角线以上全是0（或相同值），不需要存。只存下半部分，就像只数阶梯座位。

✍️ 等差数列求和

第1行：1个元素 → 第2行：2个 → ... → 第n行：n个
总数 = 1 + 2 + 3 + ... + n = n(n+1)/2
这是小学数学就学过的高斯求和公式。
答案：C（n(n+1)/2）
📝 下三角矩阵 = 阶梯座位，第 i 排 i 个座，等差数列求和得 n(n+1)/2。

7广义表 Head(Tail(Head(Tail(A)))) 的结果是？

题目原文：广义表 A = ((a,b), (c,(d),((e,f),g)))，那么 Head(Tail(Head(Tail(A)))) 的值为（ ）。
A、() B、(e,f) C、(c) D、(d)

🎯 生活类比：剥洋葱——一层一层来

广义表就像一颗可以嵌套的洋葱。大盒子里有小盒子，小盒子里还有更小的盒子。两个核心操作：

Head（表头） = 取第一个元素，保持它原本的样子
Tail（表尾） = 去掉第一个元素后剩下的所有，外面一定加一层括号

✍️ 逐层手算——跟着走，别跳过！

先看清 A 的结构
A = ( (a,b) , (c,(d),((e,f),g)) )
最外层括号里有两个元素：
  ① (a,b) —— 一个小表
  ② (c,(d),((e,f),g)) —— 另一个小表（里面有三个元素：c、(d)、((e,f),g)）
第①步：Tail(A)
= 去掉第一个元素 (a,b)，剩下的加括号
= ( (c,(d),((e,f),g)) )
Tail 一定加括号！即使只剩一个元素。
第②步：Head(Tail(A))
= 取 ( (c,(d),((e,f),g)) ) 的第一个元素
= (c, (d), ((e,f),g))
这是有三个元素的表：c、(d) 和 ((e,f),g)
第③步：Tail(Head(Tail(A)))
= 对 (c, (d), ((e,f),g)) 去掉第一个元素 c
剩下 (d) 和 ((e,f),g)，外面加括号
= ( (d), ((e,f),g) )
第④步：Head(Tail(Head(Tail(A))))
= 取 ( (d), ((e,f),g) ) 的第一个元素
= (d) ← 这就是答案！
答案：D（(d)）
📝 Head = 取首元素，Tail = 去首加括号。从最外层开始一层层剥，每步只做一个操作，别跳步！

8哪个不可能是快速排序一趟的结果？

题目原文：下面哪个序列不可能是一趟快速排序之后的结果（ ）。
A、27, 13, 2, 5, 19  B、3, 14, 38, 17, 32  C、8, 52, 15, 1, 71  D、5, 26, 28, 47, 15

🎯 生活类比：老师按身高排座位

老师选一个同学做"基准"，比他矮的坐左边，比他高的坐右边。一趟结束后，基准同学坐在了正确的位置——左边全比他矮，右边全比他高。这个位置以后不会再变。

🔍 快排一趟的核心特征

快排一趟后：存在一个基准元素，它左边所有元素都≤它，右边所有元素都≥它。
这个基准元素已经"归位"——它在最终有序数组中的位置就确定了。
⚠ 关键：一趟后基准必须归位，左边全≤它，右边全≥它。

✍️ 逐个选项检查

选项A 27, 13, 2, 5, 19
• 若 27 是基准在位置1：右边 13,2,5,19 都 ≤ 27 ✓。但问题是 27 是最大值（排序后应在最后位置），一趟后却在第一位 → 27 没有归位！✗
• 若 19 是基准在最后：左边 27,13,2,5 中 27 > 19 ✗。27 不应该出现在19左边！
• 若 13 是基准：左边 27 > 13 ✗。
找不到任何一个元素能成为合法基准 → A 不可能！
选项B 3, 14, 38, 17, 32
3 最小在第一位 → 可选 3 做基准：右边 14,38,17,32 都 ≥ 3 ✓ → 可能。
选项C 8, 52, 15, 1, 71
71 最大在最后 → 可选 71 做基准：左边 8,52,15,1 都 ≤ 71 ✓ → 可能。
选项D 5, 26, 28, 47, 15
5 最小在第一位 → 可选 5 做基准：右边 26,28,47,15 都 ≥ 5 ✓。
注意 15 虽然在 5 右边但 5 是基准时右边元素只要 ≥5 就行，15 > 5 ✓ → 可能。
答案：A
📝 快排一趟后基准归位，左边全≤它，右边全≥它。A 中 27 是最大值却在第一位（没归位），19 在最后但前面有 27>19 —— 没有任何元素能当合法基准！

9邻接矩阵便于判断顶点间是否有边，因为？

题目原文：邻接矩阵便于判断两个顶点之间是否有边，因为？（ ）
A. 空间省 B. 容易遍历 C. 可直接取值 O(1) D. 容易增删顶点

🎯 生活类比：通讯录查好友

你想知道张三和李四是不是好友。邻接矩阵就像一张表格——查第 i 行第 j 列，是 1 就有边，是 0 就无边。一眼就能看到，这就是 O(1)。

而邻接表呢？你得翻张三的"好友列表"一条条找有没有李四——可能找很久。

✍️ 邻接矩阵 vs 邻接表

操作邻接矩阵邻接表
判断 i→j 是否有边O(1) 直接查 edge[i][j]O(度) 需遍历链表
增加/删除顶点O(n²) 需重建矩阵O(1) 或 O(n)
空间（稀疏图）O(n²) 浪费O(n+e) 省空间
找所有邻接点O(n) 扫一整行O(度) 只遍历有边的
答案：C
📝 邻接矩阵 edge[i][j] 直接取值 O(1) 判边，像查表格一样；但加删顶点要重建矩阵成本高。

10空间少 + 平均快 → 选哪个排序？

题目原文：要求空间复杂度尽量低且平均速度较快，应选择哪种排序？（ ）
A. 快速排序 B. 归并排序 C. 插入排序 D. 堆排序

🎯 生活类比：选搬家工具

你要搬家，东西很多：

• 快速排序 = 请搬家公司的车，但需要额外仓库（递归栈 O(logn)）
• 归并排序 = 需要租一个等大的仓库放东西（O(n) 辅助空间）
• 插入排序 = 自己一趟趟搬，慢（O(n²)）
• 堆排序 = 原地操作，不需要额外仓库，而且速度稳定 O(nlogn)

✍️ 四个选项逐一比较

排序时间复杂度空间复杂度评价
A. 快速排序O(nlogn) 平均O(logn) 递归栈空间不是最优
B. 归并排序O(nlogn)O(n) 辅助数组空间太大
C. 插入排序O(n²)O(1)空间小但太慢
D. 堆排序O(nlogn)O(1)✅ 空间少+平均快
答案：D
📝 堆排序 = 原地 O(1) 空间 + O(nlogn) 时间，是"空间少且平均快"的唯一选择。

1时间复杂度分析

题目原文：分析下面代码的时间复杂度。
int func(int n) { int i, j, sum = 0; for (i = 1; i <= 1000; i++) // ← 注意！i只到1000 for (j = 1; j <= i; j++) sum++; return sum; }

🎯 生活类比：数楼层

假设你爬楼梯，不管楼有多高（n多大），你只爬到第1000层就停。那爬的层数是固定的，和楼的总高 n 无关——这就是 O(1)。

但如果把 1000 改成 n——你就得爬完整栋楼，层数随 n 增长——那就是 O(n²)。

✍️ 推导

外层循环：i 从 1 到 1000（常数，不随 n 变化）
内层循环：j 从 1 到 i，最多 1000 次
总执行次数 ≈ 1+2+...+1000 = 1000×1001/2 ≈ 500,500
这是一个常数！与输入 n 没有任何关系。
如果把 1000 改成 n → 执行次数 = n(n+1)/2 = O(n²)
答案：O(1)
📝 循环上限是常数 1000 → 执行次数固定 50 万，与 n 无关 → O(1)。若 i<n 才是 O(n²)。

2中缀 "8-(5+2)*(7-3)/2" → 后缀，扫描到 + 时栈中有几个元素？

题目原文：将中缀表达式 8-(5+2)*(7-3)/2 转换为后缀表达式，当扫描到 '+' 时，运算符栈中有几个元素？

🎯 生活类比：排队进地铁

数字是乘客，直接进站（输出）。运算符是安检员——优先级高的先进站。遇到括号就像来了 VIP——括号内的先处理。

后缀表达式（逆波兰式）就是把运算符放到操作数后面：8 5 2 + 7 3 - * 2 / -

✍️ 逐步转换

初始 栈为空
扫描 '8' → 数字直接输出：8，栈：[]
扫描 '-' → 运算符，栈空直接入栈，栈：[-]
扫描 '(' → 左括号入栈，栈：[-, (]
扫描 '8' → 输出：8 8
扫描 '-' → 运算符，栈顶是 '(' 不弹出，入栈，栈：[-, (, -]
扫描 '5' → 输出：8 8 5
扫描 '+' → 此时栈中有 3 个元素：[-, (, -]
栈顶 '-' 优先级 ≥ '+' 且栈顶不是 '(' → 弹出 '-' 输出 → 输出：8 8 5 -
现在栈：[-, (]，栈顶 '(' → '+' 直接入栈 → 栈：[-, (, +]
扫描到 + 时，栈中有 3 个元素（弹出前）→ 答案：3
继续 扫描 '2' → 输出 8 8 5 - 2
扫描 ')' → 弹出直到 '('：弹出 '+' → 输出 +；弹出 '(' 丢弃
扫描 '*' → 栈顶 '-' 优先级低于 '*' → '*' 入栈
扫描 '(' → 入栈
扫描 '7','-','3' → 输出 7 3 -
扫描 ')' → 弹出 '-'
扫描 '/' → 栈顶 '*' 优先级 ≥ '/' → 弹出 '*'；'/' 入栈
扫描 '2' → 输出 2
最终后缀：8 5 2 + 7 3 - * 2 / -
答案：3
📝 扫描到 '+' 时栈中有 '-'、'('、'-' 三个元素，之后 '-' 弹出，'+' 在 '(' 之上入栈。
填空题 · 第 3 题

3模式串 "aaaab" 的 next 数组 & KMP 比较次数

题目原文：模式串 "aaaab"，求 next 数组以及 KMP 匹配过程中的比较次数。

🎯 生活类比：拼图对齐

KMP 算法就像拼拼图——当你发现当前块对不上时，你不会从头开始，而是利用已经匹配的部分，把拼图滑动到合适的位置继续。next 数组告诉你"滑动多少"。

✍️ 手算 next 数组

模式串：a a a a b（位置从 1 开始编号）

next[1] = 0（第一个字符，约定为 0）
next[2] = 1（"a" 没有相等前后缀）
next[3] = 2（"aa"：前缀 "a" = 后缀 "a"，最长匹配长度=1 → next=2）
next[4] = 3（"aaa"：前缀 "aa" = 后缀 "aa"，最长匹配长度=2 → next=3）
next[5] = 4（"aaaa"：前缀 "aaa" = 后缀 "aaa"，匹配长度=3 → next=4）
但注意：题目说答案是 "01212"，说明用的是另一种 next 定义。
不同教材对 next 的定义略有不同。一种常见定义：next[j] = 第 j 位失配时模式串应跳到第 next[j] 位继续比较。

2Insert_sort —— 带头结点链表按序插入（5个空）

题目原文：补全下面的函数，实现线性表的按序插入。采用带头结点的链表存储。
struct Node { ElementType Data; struct Node *Next; };

🎯 生活类比：按身高插队

一队人已按身高排好，头结点L是"领队标志"。新来X，p从L开始往后找：只要"p后面那人"比X矮就继续走，走到"p后面那人≥X"停下，把X插在p和他后面的人之间。

✍️ 5个空的推导（原题代码结构如下）

void Insert_sort(struct Node *L, ElementType X) { struct Node *p, *q; p = L; // p指向头结点 while (___________ && p->Next->Data < X) // 空① ___________; // 空②：p指向下一结点 q = malloc(sizeof(struct Node)); ___________; // 空③：q的数据部分赋值为X ___________; // 空④：q的下一个节点为p的下一个节点 ___________; // 空⑤：p的下一个节点为q }
空① p后面有结点 → p->Next != NULL
空② p后移 → p = p->Next;
空③ 给新结点赋值 → q->Data = X;
空④ q先接后面 → q->Next = p->Next;（⚠必须先做！否则p->Next一改就丢了原来的链）
空⑤ p再接q → p->Next = q;
📝 p从头结点往后找→while(p->Next存在且Data先q->Next=p->Next，再p->Next=q（顺序绝对不能反！反了链表就断了）

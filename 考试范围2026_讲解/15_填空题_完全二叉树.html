📝 填空第5题 · 完全二叉树结点数

126个叶子 → 251（或252）个结点 | n₀ = n₂ + 1 推导 | n₁ = 0 或 1

总结点数 = 251 或 252
n₀=126（叶子数），n₂ = n₀−1 = 125。n₁ 可能为 0 或 1 → n = 126+125+0=251 或 126+125+1=252

一、二叉树结点分类与基本关系

二叉树中，按度（子女个数）将结点分为三类：

  • n₀：度为0的结点（叶子 / 终端结点）
  • n₁：度为1的结点（只有一个孩子）
  • n₂：度为2的结点（有两个孩子）

总结点数：n = n₀ + n₁ + n₂

二、核心公式：n₀ = n₂ + 1（推导过程）

🔑 从两个角度计算总边数

角度一（从结点出发）：树中除根结点外，每个结点都有一条边连接到其父结点。

边数 B = n − 1（n个结点的树有n−1条边）

角度二（从度出发）：度为2的结点贡献2条边（连向两个孩子），度为1的结点贡献1条边，度为0的结点贡献0条边。

边数 B = 2·n₂ + 1·n₁ + 0·n₀

联立两式：

n − 1 = 2·n₂ + n₁

代入 n = n₀ + n₁ + n₂：

(n₀ + n₁ + n₂) − 1 = 2·n₂ + n₁
n₀ + n₁ + n₂ − 1 = 2n₂ + n₁
n₀ − 1 = n₂
★ n₀ = n₂ + 1
💡 直觉理解：每个度为2的结点"生出"了两个孩子，但只有n₀个"终端"。因为每个非根结点只有一个父结点，所以 n₂ = n₀ − 1。叶子比二度结点永远多一个。

三、完全二叉树的特殊性质

完全二叉树（Complete Binary Tree）的定义

深度为k的完全二叉树，前k−1层是满的，第k层的结点从左到右连续排列，没有"空洞"。

关键特征：在完全二叉树中，度为1的结点个数 n₁ 只能是 0 或 1。不可能出现 n₁ ≥ 2 的情况。

n₁ = 0 的完全二叉树： n₁ = 1 的完全二叉树： ○ ○ / \ / \ ○ ○ ○ ○ / \ / \ / \ / ○ ○ ○ ○ ○ ○ ○ （最后一层满，没有1度结点） （最后一层缺一个右孩子，出现1个1度结点）

为什么 n₁ 只能是 0 或 1？因为完全二叉树的结点是从左到右连续排列的。如果某个结点只有一个孩子，那它只能有左孩子（因为如果有右孩子而无左孩子就出现了"空洞"），并且这样的1度结点最多出现1个（在最后一层的最右边位置）。

四、代入本题：126个叶子 → 两种可能答案

📐 计算过程

1 n₀ = 126（已知叶子数）
2 由 n₀ = n₂ + 1 → n₂ = n₀ − 1 = 125
3 n₁ 可能为 0（完全二叉树最后一层恰好满）
→ n = n₀ + n₁ + n₂ = 126 + 0 + 125 = 251
4 n₁ 也可能为 1（最后一层缺一个最右边的叶子）
→ n = n₀ + n₁ + n₂ = 126 + 1 + 125 = 252

五、二叉树结点关系公式总结

公式含义
n = n₀ + n₁ + n₂总结点数 = 各度结点之和
n = B + 1结点数 = 边数 + 1
B = n₁ + 2n₂边数由度计算
n₀ = n₂ + 1叶子数 = 二度结点数 + 1（二叉树通用）
n₁ ∈ {0, 1}完全二叉树中，1度结点只能是0或1
⚠️ 常考陷阱：公式 n₀ = n₂ + 1 适用于任何二叉树（不限于完全二叉树）。但 n₁∈{0,1} 是完全二叉树特有的性质。答题时不要混淆。

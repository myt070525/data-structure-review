📋 编程题：Merge合并顺序表（填空补全）

第28题 · 合并两个顺序表为一个新顺序表
✅ 填空答案：
① malloc(sizeof(struct SqList))    ② L1->MaxSize + L2->MaxSize
③ L1->N + L2->N    ④ L3

一、题目理解：我们要做什么？

1.1 问题描述

已知两个顺序表 L1 和 L2，每个顺序表都存有若干元素。要求编写函数 Merge， 将L1和L2中的所有元素依次复制到一个新创建的顺序表L3中（先L1的所有元素，再L2的所有元素），并返回L3的指针。 题目给出了完整代码框架，留了4个空需要填写。

1.2 顺序表结构定义

typedef struct { ElementType *Data; // 指向动态分配的数组空间 int MaxSize; // 数组的最大容量（已分配的槽位数） int N; // 当前实际存储的元素个数（N ≤ MaxSize） } SqList;

1.3 合并策略

这道题的合并方式是最简单的拼接：先把L1的所有元素依次放入L3，再把L2的所有元素依次放入L3。不需要排序，不需要去重。

二、逐空推导：四个空分别考什么？

填空①：L3 = malloc(sizeof(struct SqList));

考察点动态内存分配 —— 为结构体本身分配空间
推导过程
  1. L3是一个指针（struct SqList *L3），它需要指向一个有效的SqList结构体。
  2. 目前L3未初始化（野指针），直接使用L3->xxx会崩溃。
  3. 所以必须先用malloc为SqList结构体本身分配堆内存。
  4. sizeof(struct SqList) 获取SqList结构体的大小（通常是三个字段：一个指针+两个int = 16或24字节，取决于平台）。
易错点 ❌ 写成 malloc(sizeof(ElementType) * N) —— 那是给Data数组分配空间，不是给结构体本身分配。
❌ 写成 malloc(sizeof(SqList *)) —— 分配的是指针大小（8字节），不够存结构体。
✅ 必须是 sizeof(struct SqList) 或 sizeof(SqList)。

填空②：L3->MaxSize = L1->MaxSize + L2->MaxSize;

考察点容量计算 —— 新顺序表需要多大空间？
推导过程
  1. L3要容纳L1和L2的全部元素。
  2. 最坏情况：L1满了（N1 = MaxSize1），L2也满了（N2 = MaxSize2），合并后需要 MaxSize1 + MaxSize2 个槽位。
  3. 因此L3的最大容量 = L1的最大容量 + L2的最大容量。
  4. 后续用这个MaxSize去malloc Data数组：L3->Data = malloc(sizeof(ElementType) * L3->MaxSize)。
易错点 ❌ 写成 L1->N + L2->N —— N是当前元素数，但MaxSize表示最大容量。如果后续还有插入操作，用N可能不够。
❌ 写成 L1->MaxSize > L2->MaxSize ? L1->MaxSize : L2->MaxSize —— 这是取最大值，但合并需要的是和而非最大值。
✅ L1->MaxSize + L2->MaxSize。

填空③：L3->N = L1->N + L2->N;

考察点元素个数统计 —— 合并后有多少个元素？
推导过程
  1. 前面的两个for循环已经把L1的N₁个元素和L2的N₂个元素都复制到了L3->Data中。
  2. 所以L3中实际存储的元素个数 = L1的元素个数 + L2的元素个数。
  3. k的值最后就是L1->N + L2->N（因为k从0开始，每次k++）。
  4. 用L3->N记录这个数，后续遍历L3时才知道有效元素的范围。
易错点 ❌ 写成 k —— 虽然k的值恰好等于L1->N + L2->N（功能上等价），但考试填空要看上下文，代码中k是局部变量，写成k可能也被接受。不过从"语义清晰"角度，写N的和更好。
❌ 写成 L3->MaxSize —— MaxSize是容量，可能大于实际元素数。
✅ L1->N + L2->N。

填空④：return L3;

考察点函数返回值 —— 返回什么？
推导过程
  1. 函数签名是 struct SqList *Merge(...)，返回值类型是指向SqList的指针。
  2. L3是我们新创建并填充好的顺序表指针。
  3. 调用者通过这个返回值获取合并后的顺序表，后续可以遍历L3->Data[0] 到 L3->Data[L3->N-1]。
  4. 所以返回L3即可。
易错点 ❌ 写成 *L3 —— 解引用后是结构体本身，类型不匹配（函数返回指针）。
❌ 写成 &L3 —— 取指针的地址，变成二级指针了。
❌ 写成 L3->Data —— 返回的是内部数组指针，丢失了MaxSize和N信息。
✅ L3。

三、完整代码 & 结构示意图

3.1 完整Merge函数

struct SqList *Merge(struct SqList *L1, struct SqList *L2) { struct SqList *L3; int i, k = 0; // ─── 空①：为L3结构体本身分配内存 ─── L3 = malloc(sizeof(struct SqList)); // ─── 空②：设置L3的最大容量 ─── L3->MaxSize = L1->MaxSize + L2->MaxSize; // 根据MaxSize为Data数组分配空间 L3->Data = malloc(sizeof(ElementType) * L3->MaxSize); L3->N = 0; // 初始元素个数为0 // 复制L1的所有元素 for (i = 0; i < L1->N; i++) L3->Data[k++] = L1->Data[i]; // 复制L2的所有元素 for (i = 0; i < L2->N; i++) L3->Data[k++] = L2->Data[i]; // ─── 空③：设置L3的实际元素个数 ─── L3->N = L1->N + L2->N; // ─── 空④：返回新顺序表指针 ─── return L3; }

3.2 调用示例

// 假设已创建 L1 = [10, 20, 30] (N=3, MaxSize=10) // L2 = [40, 50] (N=2, MaxSize=8) struct SqList *result = Merge(L1, L2); // result->Data = [10, 20, 30, 40, 50] // result->N = 5 // result->MaxSize = 18

五、内存布局详解：malloc了两次什么？

这段代码一共调用了两次malloc，分配的是不同层级的内存：

malloc调用分配对象大小目的
malloc(sizeof(struct SqList)) SqList结构体本身 ≈16～24字节 存放MaxSize、N、Data指针这三个字段。如果没有这个结构体，Data指针就没地方放。
malloc(sizeof(ElementType) * L3->MaxSize) Data数组 = sizeof(ElementType) × 18 存放实际的元素数据。这是一个独立的连续内存块。
堆内存中的两层结构： ① SqList结构体 ② Data数组（独立内存块） ┌────────────────┐ ┌──────────────────────────┐ │ MaxSize = 18 │ │[10][20][30][40][50]...[ ]│ │ N = 5 │ └──────────────────────────┘ │ Data ──────────┼──────→ (18个ElementType的连续空间) └────────────────┘ 注意：Data是一个指针，它存储的是数组的首地址。 L3->Data[i] 等价于 *(L3->Data + i)。
⚠️ 关键理解：为什么需要两次malloc？因为SqList结构体和它内部的数组是两个独立的内存块。如果只malloc结构体而不malloc数组，Data就是野指针；如果只malloc数组而没有结构体，MaxSize和N就没地方存。

六、边界情况分析

情况1：L1为空（N=0），L2有元素

L1: N=0, MaxSize=10, Data=[空] L2: N=3, MaxSize=8, Data=[40,50,60] 结果：L3->Data = [40,50,60] L3->N = 0+3 = 3 ✅ 第一个for循环i<0不执行，k保持0。 第二个for循环正常复制L2的元素。

情况2：两个表都为空

L1: N=0, MaxSize=10 L2: N=0, MaxSize=8 结果：L3->N = 0+0 = 0 ✅ L3->MaxSize = 10+8 = 18（虽然暂时没用到） 两个for循环都不执行，k保持0。 返回的L3是一个空的顺序表，Data数组有18个空位。

情况3：L1的MaxSize远大于N

L1: N=2, MaxSize=100, Data=[1,2] L2: N=3, MaxSize=5, Data=[3,4,5] L3->MaxSize = 100 + 5 = 105 L3->N = 2 + 3 = 5 Data数组有105个槽位，但只用了前5个。✅ 虽然有100个空位有点浪费，但题目不要求压缩。
💡 所有边界情况的处理都是自然的，无需特殊if判断——因为循环条件是 i < N，N=0时循环体直接跳过。

七、常见错误 & 考点总结

🔴 致命错误1：混淆两次malloc的用途！
空①是malloc结构体（sizeof(struct SqList)），下一行是malloc数组（sizeof(ElementType) * MaxSize）。
两者不能互换：不能用malloc数组的代码填到空①，也不能用malloc结构体的代码填到空②的位置。
记忆："先有壳（结构体），再有肉（数组）。"
⚠️ 错误2：空②写成N的和而非MaxSize的和。
L1->N + L2->N 虽然"当前够用"，但MaxSize表示的是最大容量。
如果后续还要向L3插入新元素，用N的和作为MaxSize可能不够。题目框架用MaxSize作为数组分配大小的依据，自然应该用MaxSize的和。
当然，在当前代码逻辑下（只合并不插入），N的和也是够用的。但从变量语义上，MaxSize应对应MaxSize。
⚠️ 错误3：空③写成k或L3->MaxSize。
写k：虽然k的值恰好等于L1->N+L2->N，但这不是"推导"出来的。考试更看重语义：L3的元素个数 = L1的元素个数 + L2的元素个数。
写MaxSize：MaxSize=18，但实际只有5个元素，N=18会导致后续遍历访问到垃圾数据。
⚠️ 错误4：空④返回L3->Data或*L3。
返回Data：调用者只能拿到数组，丢失了N和MaxSize信息，无法知道有多少个有效元素。
返回*L3：类型错误——函数返回指针，不是结构体值。
⚠️ 错误5：忘记初始化L3->N = 0。
虽然题目代码里已经写了，但如果自己写类似函数，忘记初始化N会导致空③之前N是垃圾值。
💡 考试心法：
这道题的4个空恰好覆盖了"创建新顺序表"的完整流程——
① 分配结构体（malloc结构体）
② 规划容量（计算MaxSize）
③ 统计个数（计算N）
④ 返回指针（return）
每个空都可以从"这个变量代表什么"以及"它应该等于什么"两个角度推导出来。
💡 记忆口诀："结构体先malloc开天地，MaxSize求和定容量。N数相加标边界，返回L3指针就完毕。"

八、拓展对比：顺序表 Merge vs 链表 Merge

维度顺序表Merge（本题）有序链表Merge
合并策略 直接拼接（先L1后L2），O(n₁+n₂) 归并（比较大小依次取），O(n₁+n₂)
结果有序性 不保证有序 保证有序
空间 需要新数组，O(n₁+n₂)额外空间 可以原地（改指针），O(1)额外空间
内存分配 需要malloc结构体+数组 通常复用原结点（不新建）
典型考点 malloc用法、MaxSize vs N 归并逻辑、指针操作

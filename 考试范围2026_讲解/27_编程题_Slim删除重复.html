🗑️ 编程题：Slim删除重复结点

第27题 · 删除带头结点非递减单链表中的重复元素
✅ 完整代码 · 遍历一次 O(n) · 返回删除个数

一、题目理解：我们要做什么？

1.1 问题描述

给定一个带头结点的单链表，链表中的元素按非递减（即升序，允许相等）排列。 编写函数 Slim，删除所有值重复的结点（保留第一个出现的值，删除后续重复的）， 返回被删除的结点个数。

1.2 核心任务

  1. 遍历链表：从头到尾扫描一遍。
  2. 识别重复：如果当前结点和下一个结点的值相等，则下一个结点是重复的。
  3. 删除结点：跳过重复结点，释放其内存，计数器+1。
  4. 返回结果：返回总共删除了多少个重复结点。

1.3 链表结构定义

typedef struct Node { int Data; struct Node *Next; } Node; // 带头结点：头结点L的Data无效，L->Next指向第一个数据结点

二、核心原理：为什么"非递减 = 重复元素必定相邻"？

2.1 什么是"非递减排列"？

非递减（non-decreasing）的意思是：随着下标增加，元素值不会变小，即：

对于任意相邻结点 A 和 B（A在前，B在后），满足：A.Data ≤ B.Data 简单理解：非递减 ≈ 升序（允许相等） 即 "从小到大排列，但允许相邻元素相等"

2.2 为什么重复元素一定相邻？——反证法

假设链表非递减排列：a₁ ≤ a₂ ≤ a₃ ≤ ... ≤ aₙ。

如果值为 X 的元素出现了多次，设它们出现在位置 i 和 j（i < j），即 ai = aj = X。

由于序列非递减：ai ≤ ai+1 ≤ ... ≤ aj

又因为 ai = aj = X，中间的传递性强制：ai+1 = ai+2 = ... = aj-1 = X。

也就是说，所有值为X的元素必然连续排列在一起，中间不可能插入其他值。

2.3 直观对比

排列类型示例链表重复元素是否相邻？
非递减（本题） [1, 2, 2, 2, 3, 5, 5, 8] ✅ 是！所有重复都紧挨着
无序 [2, 5, 2, 3, 5, 1] ❌ 否！2和5分散在各处
💡 关键结论：正因为非递减保证重复元素相邻，我们只需比较相邻结点（p和p->Next），一次遍历就能删除所有重复。如果链表无序，则需要O(n²)或哈希表辅助。

三、完整代码 & 逐行解析

3.1 Slim函数完整实现

int Slim(struct Node *L) { int n = 0; // 计数器：已删除的重复结点数 struct Node *p, *q; // p: 当前扫描指针 q: 临时保存待删除结点 p = L->Next; // p从第一个数据结点开始（跳过头结点） while (p != NULL && p->Next != NULL) { // 条件：p和p的下一个都存在 if (p->Data == p->Next->Data) { // 发现重复！p和后继值相等 q = p->Next; // ① q指向待删除的重复结点 p->Next = q->Next; // ② 跳过q：p直接连到q的后继 free(q); // ③ 释放重复结点的内存 n++; // ④ 计数器+1 } else { p = p->Next; // 不重复：p正常后移 } } return n; // 返回删除总数 }

3.2 逐行详细解释

代码行解释易错点
int n = 0; 初始化计数器为0。每删除一个重复结点就n++。 别忘了初始化，否则返回的是垃圾值。
p = L->Next; p从第一个数据结点开始，跳过头结点L。 不要写成 p = L，头结点不存数据，不需要参与比较。
while (p != NULL && p->Next != NULL) 循环继续条件：①p存在 且 ②p有后继。
两个条件缺一不可。如果只剩最后一个结点（p->Next==NULL），它不可能有重复，循环结束。 		⚠️ 短路求值！必须先判断p!=NULL，再判断p->Next。如果反过来写，p==NULL时p->Next会空指针崩溃。
if (p->Data == p->Next->Data) 比较相邻两个结点：当前p和下一个。由于非递减，这是判断重复的唯一条件。 不需要考虑"隔一个"的情况——非递减保证重复一定相邻。
q = p->Next; 用q暂存待删除结点的地址，以便后续free。 不暂存的话，p->Next被改后就丢失了待删除结点的地址→内存泄漏。
p->Next = q->Next; "跳过"重复结点：p的Next直接指向q的后继。 这是链表删除的核心一步。
free(q); 释放被删除结点的内存。 不要忘记free，否则内存泄漏。
n++; 每成功删除一个，计数器加1。 -
else p = p->Next; 关键：只有当p和p->Next不相等时，p才后移。
如果相等并删除了p->Next，p不动——因为新的p->Next可能还是重复的！ 		⚠️ 易错！删除后不要立即p=p->Next。比如 [2,2,2,3]，p指向第一个2，删除第二个2后，p->Next变成了第三个2（值也是2），需要继续比较。如果p后移就漏删了。

四、可视化演示：链表 [1, 2, 2, 2, 3, 5, 5, 8] 的Slim过程

4.1 初始状态

L ──→ 1 ──→ 2 ──→ 2 ──→ 2 ──→ 3 ──→ 5 ──→ 5 ──→ 8 ──→ NULL ↑ p (初始指向第一个数据结点1) n = 0

4.2 第一轮：p=结点1，p->Data=1，p->Next->Data=2

p ↓ 1 ──→ 2 ──→ 2 ──→ 2 ──→ 3 ──→ 5 ──→ 5 ──→ 8 ──→ NULL 1 ≠ 2 → 不重复！p后移。 n = 0

4.3 第二轮：p=结点2(第一个)，p->Data=2，p->Next->Data=2

p ↓ 1 ──→ 2 ──→ 2 ──→ 2 ──→ 3 ──→ 5 ──→ 5 ──→ 8 ──→ NULL ↑ q = p->Next（q指向第二个2） 2 == 2 → 重复！执行删除： ① q = p->Next （q指向第二个2） ② p->Next = q->Next （p跳过q，指向第三个2） ③ free(q) （释放第二个2） ④ n++ （n = 1） ⚠️ p不动！因为新的p->Next（第三个2）值也是2，还需继续检查。

4.4 第三轮：p仍在结点2(第一个)，p->Data=2，p->Next->Data=2

p ↓ 1 ──→ 2 ──→ 2 ──→ 3 ──→ 5 ──→ 5 ──→ 8 ──→ NULL ↑ q = p->Next（q指向第三个2，即原来的第三个2） 2 == 2 → 重复！再次执行删除： ① q = p->Next ② p->Next = q->Next （p跳过q，指向3） ③ free(q) （释放第三个2） ④ n++ （n = 2） ⚠️ p仍不动！

4.5 第四轮：p仍在结点2(第一个)，p->Data=2，p->Next->Data=3

p ↓ 1 ──→ 2 ──→ 3 ──→ 5 ──→ 5 ──→ 8 ──→ NULL 2 ≠ 3 → 不重复！p后移。 n = 2

4.6 第五轮～第六轮：继续扫描剩余部分

1 ──→ 2 ──→ 3 ──→ 5 ──→ 5 ──→ 8 ──→ NULL ↑ ↑ p (3≠5,p后移) p (5==5,删除第二个5,n=3) 最后：p指向8，p->Next==NULL → 循环结束！

4.7 最终结果

最终链表： L ──→ 1 ──→ 2 ──→ 3 ──→ 5 ──→ 8 ──→ NULL 删除总数 n = 3（删除了两个多余的2和一个多余的5） 每个值只保留第一次出现的那一个！

六、边界情况分析

情况1：空链表（只有头结点，L->Next == NULL）

L ──→ NULL p = L->Next = NULL → while(p != NULL) 直接不满足 → 返回 0。✅

情况2：只有一个数据结点

L ──→ 5 ──→ NULL p = 结点5，p->Next == NULL → while(p->Next != NULL) 不满足 → 返回 0。✅ （单个结点不可能有重复）

情况3：所有元素都不重复

L ──→ 1 ──→ 3 ──→ 5 ──→ 8 ──→ NULL p一路后移，每次if都不成立，最终返回 0。✅ 链表不变。

情况4：所有元素都相同

初始：L ──→ 5 ──→ 5 ──→ 5 ──→ 5 ──→ NULL ↑p 第1轮：删除第2个5，n=1，p不动 第2轮：删除第3个5，n=2，p不动 第3轮：删除第4个5，n=3，p不动 第4轮：p->Next==NULL → 退出 最终：L ──→ 5 ──→ NULL 返回 3。✅ 只保留第一个。

情况5：重复出现在链表末尾

初始：L ──→ 3 ──→ 8 ──→ 8 ──→ NULL ↑p 第1轮：删除第2个8，n=1，p不动 第2轮：p->Next==NULL → 退出 最终：L ──→ 3 ──→ 8 ──→ NULL 返回 1。✅
💡 总结：所有边界情况都无需特殊处理——代码逻辑已自然覆盖。这是算法设计良好的标志。

七、常见错误 & 考点总结

🔴 致命错误1：删除后p立即后移！
如果删除p->Next后写 p = p->Next（或把p=p->Next放在if外面），会导致连续多个重复时只删除了第一个重复。
例如 [2,2,2]：删除第二个2后p移到了第三个2的位置，但第三个2之前是第二个2的后继——它现在是p（值=2），而p前面是第一个2（值=2），算法应该继续删除p（第三个2），但p已经移走了，第三个2被保留。
正确做法：只有else分支才能p=p->Next。
⚠️ 错误2：while条件只写 p != NULL。
如果只检查p != NULL，当p指向最后一个结点时，p->Next == NULL，访问p->Next->Data会空指针崩溃。
必须两个条件：p != NULL && p->Next != NULL。
⚠️ 错误3：忘了free(q)。
只修改指针（p->Next = q->Next）但不free(q)，被删除的结点仍在堆内存中——造成内存泄漏。
删除链表结点三步不可少：保存(q) → 跳过(p->Next=q->Next) → 释放(free(q))。
⚠️ 错误4：忘记先q=p->Next再修改p->Next。
如果直接写 p->Next = p->Next->Next 然后 free(p->Next)，此时p->Next已经变了，free的是错误结点。
必须先用q暂存待删结点地址。
⚠️ 错误5：没理解"非递减"和"严格递增"的区别。
非递减允许相等 → 有重复。严格递增不允许相等 → 不需要去重。
本题考的就是"非递减"这个条件——利用它来简化算法。
💡 考试心法：
这道题的核心考点只有三个——
① 非递减 = 重复相邻（为什么？用传递性解释）
② 链表删除的标准三步（保存→跳过→释放）
③ 删除后p不后移（处理连续重复的关键！）
理解这三点，代码自然就写出来了。
💡 记忆口诀："非递减来重复连，相邻比较一遍完。相等就删p不动，不等才把p后搬。先存q来再跳链，最后free才算完。"

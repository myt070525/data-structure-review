🌳 双亲表示法求树的高度

用双亲表示法存储树，数组为 [7, 0, -1, 2, 3, 0, 0, 3]，
则该树的高度为（　）。

A. 3    B. 4    C. 5    D. 6
正确答案
C — 高度为 5
根结点下标2（值-1），最深路径长度为5

一、什么是双亲表示法？

双亲表示法：用一个数组存储树中每个结点的父结点下标。根结点的父结点记为 -1。

给定数组 parent[8] = [7, 0, -1, 2, 3, 0, 0, 3]：

结点下标 i01234567
parent[i] (父结点)70-123003

解读：

  • 结点 2 的 parent = -1 → 结点2是根结点（没有父亲）
  • 结点 3 的 parent = 2 → 结点3的父亲是结点2
  • 结点 0 的 parent = 7 → 结点0的父亲是结点7
  • 结点 7 的 parent = 3 → 结点7的父亲是结点3
  • 等等...
🔑 核心思路：从每个结点出发，沿着 parent 指针向上追溯，直到遇到 -1（根），途经的结点数就是该结点的深度。树的高度 = 所有结点深度的最大值。

方法：按父子关系逐层构建

1

2
列出父子关系：
结点3→父2，结点4→父3，结点7→父3，
结点0→父7，结点1→父0，结点5→父0，结点6→父0

三、计算每个结点的深度（追溯父结点法）

结点 i追溯路径（i→parent→...→根）深度
0 0 → 7 → 3 → 2 → -1 4
1 1 → 0 → 7 → 3 → 2 → -1 5
2 2 → -1 1
3 3 → 2 → -1 2
4 4 → 3 → 2 → -1 3
5 5 → 0 → 7 → 3 → 2 → -1 5
6 6 → 0 → 7 → 3 → 2 → -1 5
7 7 → 3 → 2 → -1 3

最大深度 = 5 → 树的高度 = 5。

💡 注意：树的高度 = 从根到最远叶子的结点数（不是边数）。结点1、5、6到根分别经过5个结点，所以树高为5。

四、最深路径可视化

三条最深路径（长度均为5）：

路径1：2 → 3 → 7 → 0 → 1 [2] → [3] → [7] → [0] → [1] （5个结点） 路径2：2 → 3 → 7 → 0 → 5 [2] → [3] → [7] → [0] → [5] （5个结点） 路径3：2 → 3 → 7 → 0 → 6 [2] → [3] → [7] → [0] → [6] （5个结点） 最短路径（路径1）：2 → 3 → 4 [2] → [3] → [4] （3个结点）

五、求树高的通用算法

对于双亲表示法，求每个结点深度的方法：

/* 计算结点 i 的深度 */ int depth(int parent[], int i) { int d = 0; while (i != -1) { d++; i = parent[i]; // 向上追溯到父结点 } return d; } /* 树的高度 = 所有结点深度的最大值 */ int treeHeight(int parent[], int n) { int maxDepth = 0; for (int i = 0; i < n; i++) { int d = depth(parent, i); if (d > maxDepth) maxDepth = d; } return maxDepth; }
1
遍历每个结点，从该结点出发，沿着 parent 数组向上追溯直到 -1
2
计数追溯过程中经过的结点数（包括自身和根）
3
取最大值作为树的高度

六、逐项排查

选项高度判断说明
A3❌只数了2→3→4这条短路径，忽略了更深的分支
B4❌可能误以为结点0是最深的（深度4），遗漏了结点1/5/6

📖 双亲表示法是第5章树的存储结构之一。理解"向上追溯"求深度的方法即可应对此类题目。

🔄 循环队列的判空条件

关于循环队列为空时的描述，正确的是（　）。

A. front 和 rear 必须都为 0
B. front == rear，可在任意下标位置
C. front 必须比 rear 大 1
D. rear 必须比 front 大 1
正确答案
B — front == rear，可在任意下标
队空条件是 front == rear，不限制具体位置

一、什么是循环队列？

循环队列是把数组的首尾逻辑上连接起来形成环形。就像操场跑道——跑到终点（数组末尾）后，绕回到起点（数组开头）继续跑。这通过取模运算来实现。

循环队列 = 环形数组（大小 M = 8） [0] [1] [2] [3] ╔═════╤═════╤═════╤═════╗ ║ │ │ │ ║ ║ │ │ │ ║ ╟─────┼─────┼─────┼─────╢ ← 数组，但首尾逻辑相连 ║ │ │ │ ║ ║ │ │ │ ║ ╚═════╧═════╧═════╧═════╝ [7] [6] [5] [4] rear：指向队尾的下一个空位（入队位置） front：指向队头元素（出队位置）

二、队空时 front == rear 可以在任意位置

场景演示：队空时 front 和 rear 的不同位置

情形1：初始状态，front=rear=0（最常见） M=8, 数组大小8 ┌───┬───┬───┬───┬───┬───┬───┬───┐ 下标: │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ └───┴───┴───┴───┴───┴───┴───┴───┘ F/R front=0, rear=0 → front==rear → 队空 ✅ 情形2：经过几次入队出队后，front=rear=3（也是队空！） ┌───┬───┬───┬───┬───┬───┬───┬───┐ 下标: │ │ │ │ 3 │ │ │ │ │ └───┴───┴───┴───┴───┴───┴───┴───┘ F/R front=3, rear=3 → front==rear → 队空 ✅ 情形3：绕了一圈后，front=rear=6（依然是队空！） ┌───┬───┬───┬───┬───┬───┬───┬───┐ 下标: │ │ │ │ │ │ │ 6 │ │ └───┴───┴───┴───┴───┴───┴───┴───┘ F/R front=6, rear=6 → front==rear → 队空 ✅

关键结论：只要 front == rear，无论它们指向数组的哪个位置，队列就是空的。所以选项B正确——可以在任意下标。

三、公式推导

队空条件

front == rear  →  队空

队满条件（牺牲一个空间）

(rear + 1) % M == front  →  队满

元素个数公式

元素个数 = (rear - front + M) % M
1
推导：当 rear ≥ front 时，元素数 = rear - front（正常情况）
2
推导：当 rear < front 时（绕了一圈），元素数 = rear + M - front
3
统一定义：统一公式 (rear - front + M) % M 涵盖了以上两种情况
💡 验证：当队列为空（front==rear）时，元素个数 = (rear - rear + M) % M = M % M = 0。公式自洽！

四、逐项排查

选项描述判断原因
A front和rear必须都为0 ❌ 错误 不限于0，任何位置都可以
B front==rear，可在任意下标 ✅ 正确 队空唯一条件是front==rear
C front必须比rear大1 ❌ 错误 这是另一种设计下的特殊情况
D rear必须比front大1 ❌ 错误 rear=front+1时队列只有1个元素

五、队空 vs 队满 —— 容易混淆的一对

状态条件元素个数特点
队空 front == rear 0 front和rear可在任意位置
队满 (rear+1)%M == front M-1 牺牲一个空间来区分空和满
正常 front ≠ rear (rear-front+M)%M 队列中有元素
圆圈视角：front 和 rear 在环形跑道上的关系 rear ↓ [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ] ↑ front front == rear → 队空（两者重合在任意位置） rear ↓ [x] [x] [ ] [x] [x] [x] [x] [x] [x] [x] [x] [x] [x] [x] ↑ front (rear+1)%M == front → 队满（rear紧跟在front后面）
🔑 为什么不直接用全部空间？如果 M 个空间全用，队空 (front==rear) 和队满 (front==rear) 就无法区分了——它们在数学上是同一个条件！所以约定牺牲一个空间，用 (rear+1)%M == front 表示队满。

六、实例演算

题目：循环队列大小 M=9，初始 front=rear=4。操作后 front=7, rear=2，求元素个数。

1
套公式：元素个数 = (rear - front + M) % M = (2 - 7 + 9) % 9
2
计算：(2 - 7 + 9) = 4，4 % 9 = 4
3
验证：front=7, rear=2，说明元素在 [7, 8, 0, 1]，共 4 个元素。✅
🎯 记忆技巧：循环队列有三宝——队空 front==rear，队满 (rear+1)%M==front，个数 (rear-front+M)%M。背熟这三条，循环队列题全秒。

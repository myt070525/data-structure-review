一、哈希查找的基本原理

哈希表（Hash Table）通过哈希函数 H(key) 将关键字映射到表中的某个地址。理想情况下，一次计算就能定位到目标元素 → O(1)时间复杂度。但实际上，不同的关键字可能映射到同一地址，产生冲突（Collision），冲突的处理效率决定了哈希表的实际性能。

二、三大影响因素详解

1 哈希函数（Hash Function）

哈希函数 H(key) 将关键字映射为地址。好的哈希函数应该：

  • 均匀分布：关键字均匀散布到地址空间，减少冲突
  • 计算简单：函数本身不能太复杂，否则抵消哈希的优势
  • 覆盖面广：充分利用整个地址空间

常见哈希函数：

方法公式特点
除留余数法H(key) = key mod p最常用，p一般取≤表长的最大素数
直接定址法H(key) = a·key + b无冲突但空间大
平方取中法取key²的中间几位分布较均匀
折叠法分段叠加适合长关键字

2 处理冲突的方法（Collision Resolution）

当两个不同的关键字映射到同一地址时，需要冲突处理策略。主要有两大类：

类别方法原理优缺点
开放定址法
（Open Addressing）		 线性探测 冲突后依次查找下一个空位 简单，但易产生"聚集"
二次探测 / 双哈希 用二次函数或第二个哈希函数决定步长 减少聚集，但计算复杂
链地址法
（Separate Chaining）		 拉链法 每个槽位维护一个链表，冲突元素挂在链表上 不受表长限制，但需要额外指针空间

3 装填因子（Load Factor）

装填因子 α = 表中记录数 / 哈希表长度 = n / m

装填因子是衡量哈希表"满"的程度的指标：

  • α 越大 → 表越满 → 冲突概率越高 → 查找效率越低
  • α 越小 → 表越空 → 冲突概率越低 → 查找效率越高（但空间浪费大）
  • 一般建议 α 控制在 0.6 ~ 0.9 之间

三、三个因素如何相互影响？

这三个因素不是孤立的。它们共同决定了哈希表的实际效率：

┌──────────────────┐ │ 哈希函数 H(key) │ ← 决定了冲突的"先天概率" └────────┬─────────┘ │ ┌───────────────┼───────────────┐ ▼ ▼ ▼ 均匀分布好 分布不均匀 计算太复杂 冲突少 ✅ 冲突多 ❌ 得不偿失 ❌ │ │ ▼ ▼ ┌────────────────────────────────┐ │ 处理冲突的方法 │ ← 决定了冲突后的"补救效率" │ 开放定址法 / 链地址法 │ └────────────────┬───────────────┘ │ ▼ ┌────────────────────────────────┐ │ 装填因子 α = n/m │ ← 决定了表整体的"拥挤程度" │ α大→冲突多→效率低 │ │ α小→冲突少→效率高但浪费空间 │ └────────────────────────────────┘

四、装填因子详解（重点）

📐 定义与计算

装填因子 α = n / m

其中 n = 已存入的记录数，m = 哈希表的总长度（槽位数）。

示例：哈希表长度为11，已存入8个记录，则 α = 8/11 ≈ 0.727。

🔑 为什么装填因子重要？

装填因子直接决定了冲突的概率。根据生日悖论（Birthday Paradox），当表中已有23个记录时，两个记录冲突的概率就超过50%——即使表长很大。装填因子越大，新插入的记录越难找到空位。

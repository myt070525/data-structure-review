📝 填空第7题 · 最小生成树算法

Prim算法 与 Kruskal算法 —— 两种MST经典算法的全面对比

Prim算法  和  Kruskal算法
两者都是求最小生成树（MST）的经典贪心算法

一、什么是最小生成树（MST）？

给定一个带权无向连通图 G=(V,E)，最小生成树是满足以下条件的生成树：

  • 包含图中所有 n 个顶点
  • 恰好有 n−1 条边（构成树）
  • 边的权值之和最小

通俗类比：要在 n 个城市之间铺设光纤，每两个城市之间铺设的成本不同（边的权值）。问怎样铺设才能使总成本最低、且所有城市连通？这就是MST问题。

二、两种算法全方位对比

对比维度Prim算法Kruskal算法
出发点 从顶点出发，逐步扩展生成树 从边出发，逐步合并连通分量
核心策略 每次从已选顶点集到未选顶点集的横切边中，选择一条权值最小的边加入MST 每次从所有未选边中，选择一条权值最小且不形成环的边加入MST
数据结构 邻接矩阵/邻接表 + 优先队列（最小堆） 边集数组 + 并查集（Union-Find）
时间复杂度 朴素：O(|V|²)
堆优化：O(|E|·log|V|) 		排序 + 并查集：O(|E|·log|E|)
实质 = O(|E|·log|V|)（因为E≤V²）
适用场景 稠密图（边多，|E|≈|V|²）
朴素Prim O(V²)更优		 稀疏图（边少，|E|≈|V|）
Kruskal对边排序主导
选边方式 始终维护一棵树，树逐渐长大 维护森林，逐步合并成树

三、ASCII图解：两种算法的区别

🔵 Prim算法思路（从顶点出发）

初始：任选顶点(1)作为起点，MST = {1}，候选边集 = {(1,2)=2, (1,3)=3, (1,4)=1, (1,5)=4} 第1步：候选边中最小为(1,4)=1 → 加入MST，顶点集={1,4} 新增候选边：(4,3)=8, (4,6)=3 候选边={(1,2)=2, (1,3)=3, (1,5)=4, (4,3)=8, (4,6)=3} 第2步：最小为(1,2)=2 → 加入MST，顶点集={1,4,2} 新增候选边：(2,3)=5, (2,5)=6, (2,6)=7 候选边={(1,3)=3, (1,5)=4, (4,3)=8, (4,6)=3, (2,3)=5, (2,5)=6, (2,6)=7} 第3步：最小(1,3)和(4,6)都是3，任选(1,3)=3 → 顶点集={1,4,2,3} 继续...直到所有6个顶点都加入 ★ 核心：Prim是在"已选顶点"和"未选顶点"之间的横切边中找最小值。

🟡 Kruskal算法思路（从边出发）

初始：每个顶点自成一个连通分量，所有边按权值排序： (1,4)=1, (1,2)=2, (1,3)=3, (4,6)=3, (1,5)=4, (5,6)=4, (2,3)=5, (3,5)=5, (2,5)=6, (2,6)=7, (4,3)=8, (5,6)=9... (有重复) 第1步：取最小边(1,4)=1，1和4不在同一分量 → 加入，合并{1}和{4} 第2步：取(1,2)=2，1和2不在同一分量 → 加入，合并{1,4}和{2} 第3步：取(1,3)=3，1和3不在同一分量 → 加入，合并{1,2,4}和{3} 第4步：取(4,6)=3，4和6不在同一分量 → 加入，合并 第5步：取(1,5)=4，1和5不在同一分量 → 加入。现在6个顶点全部连通！ MST已有5条边(n−1=5)，完成 ✅ ★ 核心：Kruskal每次都取全局最小边，用并查集判断是否形成环。

四、算法伪代码对比

Prim算法伪代码

// 朴素Prim O(V²)
Prim(G) {
    lowcost[1] = 0;                 // 顶点1加入MST
    for i = 2 to n:
        lowcost[i] = G[1][i];      // 初始化到MST的最短距离
    for i = 1 to n-1:              // 还需选n-1个顶点
        找lowcost中最小值对应的顶点k
        将k加入MST
        for j = 1 to n:             // 更新lowcost
            if G[k][j] < lowcost[j]:
                lowcost[j] = G[k][j];
}

Kruskal算法伪代码

// Kruskal O(E·logE)
Kruskal(G) {
    将所有边按权值升序排序;        // O(E·logE)
    初始化并查集，每个顶点一个集合;
    count = 0;                     // 已选边数
    for 每条边(u,v) in 排序后的边集:
        if Find(u) != Find(v):   // 不形成环
            将边加入MST;
            Union(u, v);            // 合并分量
            count++;
            if count == n-1: break; // MST完成
}

五、选择建议与记忆口诀

图类型推荐算法原因
稠密图（边多）Prim（朴素O(V²)）Kruskal排序O(E·logE)太大
稀疏图（边少）Kruskal O(E·logE)E≈V，排序开销可接受
需要堆优化Prim + 堆 O(E·logV)适合稀疏图的Prim变种
🧠 记忆口诀：
• Prim = "Pick vertex"（从顶点出发） —— 一棵树从小到大
• Kruskal = "Kruskal picks edges"（从边出发） —— 一片森林合并成树
• 两者都是贪心算法，MST结果是唯一的（当边权都不相等时）
⚠️ 常考易错：
• 两种算法得到的MST可能不同（当存在相等权值边时），但总权值一定相同（MST的权值是唯一的）。
• Prim不需要并查集，Kruskal必须用并查集判断环。
• 两者的前提都是无向连通图，如果图不连通则不存在生成树。

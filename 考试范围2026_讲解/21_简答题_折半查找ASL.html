📐 简答第3题 · 折半查找ASL

11个有序元素 → 判定树(4层) → ASL = 33/11 = 3

ASL成功 = 33 / 11 = 3
有序表：3, 8, 15, 17, 22, 31, 35, 40, 43, 59, 62

一、构建判定树（Decision Tree）

11个元素，下标1-11。折半查找每次取 mid = ⌊(low+high)/2⌋。

┌─────────────┐ Layer1 │ 22 │ mid=⌊(1+11)/2⌋=6, 值=22 └──────┬──────┘ ┌─────────────────┴─────────────────┐ │ │ ┌──────┴──────┐ ┌──────┴──────┐ Layer2 │ 8 │ │ 43 │ left_mid=⌊(1+5)/2⌋=3 └──────┬──────┘ └──────┬──────┘ right_mid=⌊(7+11)/2⌋=9 ┌─────────┴─────────┐ ┌─────────────┴─────────┐ │ │ │ │ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐ │ 3 │ │ 15 │ │ 31 │ │ 59 │ Layer3 └───┬───┘ └───┬───┘ └───┬───┘ └───┬───┘ │ │ │ │ ┌──┴──┐ ┌──┴──┐ ┌──┴──┐ ┌──┴──┐ │ × │ │ 17 │ │ 35 │ │ 62 │ Layer4 └─────┘ └──┬──┘ └──┬──┘ └──┬──┘ ┌──┴──┐ ┌──┴──┐ ┌──┴──┐ │ × │ │ 40 │ │ × │ └─────┘ └──┬──┘ └─────┘ ┌──┴──┐ │ × │ └─────┘ 内部结点（实际元素，共11个）的层数即查找次数： Layer 1: 22 (1个) → 比较1次 Layer 2: 8, 43 (2个) → 比较2次 Layer 3: 3, 15, 31, 59 (4个) → 比较3次 Layer 4: 17, 35, 40, 62 (4个) → 比较4次

二、计算ASL

层数（比较次数）该层元素元素个数层数×个数
12211×1 = 1
28, 4322×2 = 4
33, 15, 31, 5943×4 = 12
417, 35, 40, 6244×4 = 16
合计1133
ASL成功 = Σ(层数 × 该层结点数) / n
= (1×1 + 2×2 + 3×4 + 4×4) / 11
= (1 + 4 + 12 + 16) / 11
= 33 / 11 = 3
💡 直觉：平均查找长度=3，意味着平均只需比较3次就能找到目标。而顺序查找的平均比较次数为 (11+1)/2 = 6 次。折半查找效率提高了一倍。

四、判定树构造规则

mid = ⌊(low+high)/2⌋（向下取整），决定了判定树的形状：

  • 当区间长度为奇数时，mid恰好居中，左右子树对称
  • 当区间长度为偶数时，mid偏左，左子树比右子树多一个或少一个结点
  • n=11时：根22左侧5个，右侧5个 → 对称
⚠️ 注意：失败结点的比较次数 = 其父结点所在层数，但ASL成功不包含失败结点。若计算ASL失败，需要用失败结点的人数加权。

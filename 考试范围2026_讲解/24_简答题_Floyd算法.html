📐 简答第6题 · Floyd算法求最短路径

3顶点v₀,v₁,v₂ | 有向图 | D⁽⁻¹⁾ → D⁽⁰⁾ → D⁽¹⁾ → D⁽²⁾ 四矩阵变化

Floyd算法：从 D(−1) 到 D(2) 逐步引入中间顶点
最终得到所有顶点对之间的最短路径

一、Floyd算法思想

Floyd算法（Floyd-Warshall）求每一对顶点之间的最短路径。核心递推公式：

D^(k)[i][j] = min( D^(k−1)[i][j] , D^(k−1)[i][k] + D^(k−1)[k][j] ) └─ 不经过k ─┘ └────── 经过k ──────┘

即：第k次迭代时，考虑以顶点k作为中间顶点是否能使路径更短。

二、有向图结构（3顶点）

3个顶点: v₀, v₁, v₂ 有向边及权值： v₀ ──4──→ v₁ v₀ ──11─→ v₂ v₁ ──3──→ v₀ v₁ ──2──→ v₂ v₂ ──6──→ v₀ 邻接矩阵表示（∞表示无边，0表示自身）： v₀ v₁ v₂ v₀ [ 0 4 11 ] v₁ [ 3 0 2 ] v₂ [ 6 ∞ 0 ]

三、四个矩阵的变化过程

D(−1) —— 初始邻接矩阵（不经过任何中间顶点）

v₀ v₁ v₂ v₀ [ 0 4 11 ] v₁ [ 3 0 2 ] v₂ [ 6 ∞ 0 ] ∞表示两点之间没有直接边，距离为无穷大。

D(0) —— 允许经过顶点v₀

检查是否可以通过v₀缩短任意两点间的距离：

D^(0)[i][j] = min( D^(-1)[i][j] , D^(-1)[i][0] + D^(-1)[0][j] ) 检查 v₁→v₂: 原=2, 经v₀: v₁→v₀(3)+v₀→v₂(11)=14, 2<14 → 不变 检查 v₂→v₁: 原=∞, 经v₀: v₂→v₀(6)+v₀→v₁(4)=10 < ∞ → 更新为10 v₀ v₁ v₂ v₀ [ 0 4 11 ] v₁ [ 3 0 2 ] v₂ [ 6 10 0 ] ← v₂→v₁从∞变为10

D(1) —— 允许经过顶点v₀和v₁

D^(1)[i][j] = min( D^(0)[i][j] , D^(0)[i][1] + D^(0)[1][j] ) 检查 v₀→v₂: 原=11, 经v₁: v₀→v₁(4)+v₁→v₂(2)=6 < 11 → 更新为6 检查 v₂→v₀: 原=6, 经v₁: v₂→v₁(10)+v₁→v₀(3)=13, 6<13 → 不变 v₀ v₁ v₂ v₀ [ 0 4 6 ] ← v₀→v₂从11变为6 v₁ [ 3 0 2 ] v₂ [ 6 10 0 ]

D(2) —— 允许经过所有顶点（最终结果）

D^(2)[i][j] = min( D^(1)[i][j] , D^(1)[i][2] + D^(1)[2][j] ) 检查 v₀→v₁: 原=4, 经v₂: v₀→v₂(6)+v₂→v₁(10)=16, 4<16 → 不变 检查 v₁→v₀: 原=3, 经v₂: v₁→v₂(2)+v₂→v₀(6)=8, 3<8 → 不变 v₀ v₁ v₂ v₀ [ 0 4 6 ] ← 最终最短路径矩阵 v₁ [ 3 0 2 ] v₂ [ 6 10 0 ]

四、变化总结表

迭代允许的中间顶点发生的变化原因
D(−1) 无（直接边） — 初始邻接矩阵，∞表示无边
D(0) {v₀} v₂→v₁: ∞→10 路径 v₂→v₀→v₁ = 6+4 = 10
D(1) {v₀, v₁} v₀→v₂: 11→6 路径 v₀→v₁→v₂ = 4+2 = 6
D(2) {v₀, v₁, v₂} 无变化 经过v₂没有更短路径
🧠 Floyd算法核心记忆：
• 三重循环：for k=0 to n−1（中间顶点）
   for i=0 to n−1（起点）
     for j=0 to n−1（终点）
       D[i][j] = min(D[i][j], D[i][k] + D[k][j])
• 时间复杂度：O(n³)，空间复杂度：O(n²)
• 允许负权边，但不允许负权环
⚠️ 注意：k循环必须在最外层！这是Floyd算法的关键。如果把k放在内侧，算法将不正确。

📝 填空第2题 · 顺序表删除操作

删除第 i 个元素需要移动 n−i 个元素 —— 数组删除前后对比

移动元素个数 = n − i
长度为n的顺序表中，删除第i个（1≤i≤n）元素，需要将后面 n−i 个元素依次前移一位

一、顺序表删除的基本原理

顺序表（数组存储的线性表）的存储特点是：所有元素连续存放在一片内存中。这使得：

  • 优点：可以随机存取（O(1)访问任意位置）
  • 缺点：插入/删除需要移动大量元素（O(n)）

删除第 i 个元素后，原来在第 i+1 到第 n 位置的元素必须全部前移一位，填补被删除的空位。所以需要移动的元素个数是：从第 i+1 到第 n，共 n − (i+1) + 1 = n − i 个。

💡 规律总结：
• 删除位置越靠前 → 移动元素越多（最多 n−1 个）
• 删除位置越靠后 → 移动元素越少（最少 0 个）
• 平均移动次数 = (n−1)/2（等概率删除情况下）

四、伪代码实现

顺序表删除算法

/* 删除顺序表L中第i个元素（1≤i≤L.length） */
bool ListDelete(SeqList &L, int i, ElemType &e) {
    if (i < 1 || i > L.length)     // 位置非法
        return false;
    e = L.data[i-1];               // 保存被删元素
    for (int j = i; j < L.length; j++) // 从第i+1个开始前移
        L.data[j-1] = L.data[j];    // 每个元素前移一位
    L.length--;                        // 表长减1
    return true;
}

📊 循环次数分析

j 从 i 到 L.length-1（即 n−1），共循环 (n−1) − i + 1 = n − i 次。每次循环移动一个元素。这与公式完全一致。

五、与插入操作的对比

操作移动元素个数最好情况最坏情况平均
插入第i个位置 n − i + 1 0（插末尾） n（插开头） n/2
删除第i个元素 n − i 0（删末尾） n−1（删开头） (n−1)/2
⚠️ 常考易错：插入移动 n−i+1 个，删除移动 n−i 个。两者差1！因为插入时空位后面的所有元素（包括原先在第i位的）都要后移；而删除时只有第i个后面的元素需要前移，第i个本身被删除了。
🎯 记忆口诀：
"插入移 n−i+1，删除移 n−i"
"插入多一个，因为原位置也要让路"

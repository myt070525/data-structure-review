⏱️ 时间复杂度分析：O(log n)

下面代码的时间复杂度为（　）。

int x = 1;
while (n >= x * x)   x *= 2;
正确答案
O(log n)
对数时间复杂度 —— x 每次翻倍增长

一、直觉理解：x 翻倍的速度有多快？

想象你有一张纸，厚度为 1 毫米。你每次对折，厚度翻倍：

次数 k01234567... k
x 的值1248163264128... 2k

循环停止条件是 n >= x²，即 x² ≤ n，也就是 x ≤ √n。

而 x = 2k（k 是循环次数），所以条件变为：

2k ≤ √n   →   2k ≤ n1/2

取对数：k ≤ log₂(√n) = (1/2)·log₂ n

所以循环执行大约 (1/2)·log₂ n 次，时间复杂度为 O(log n)。

二、逐步推导（手算全过程）

1
初始化：x = 1，循环次数 k = 0
2
每次迭代：x *= 2 → x 变成原来的 2 倍。
第 1 次后 x = 2，第 2 次后 x = 4，第 3 次后 x = 8，…，第 k 次后 x = 2k
3
退出条件：n < x²，即 x² > n，也即 x > √n
4
代换：2k > √n → 取对数 → k > log₂(√n) = ½·log₂ n
5
结论：循环大约执行 ½·log₂ n 次 → O(log n)

三、代码逐行分析

/* 初始化：x = 1，即 2^0 */ int x = 1; /* 循环条件：n >= x*x，即 x² ≤ n */ /* 每次迭代：x *= 2，即 x 翻倍 */ while (n >= x * x) x *= 2; /* 循环次数 k 满足：2^k ≤ √n */ /* 即：k = O(log n) */
💡 关键洞察：循环变量 x 以 指数速度（2k）增长，而退出条件用的是 x²（也是指数，只是指数是 2k）。指数增长 → 对数时间。这是识别 O(log n) 的核心模式。

O(log n) 几乎是一条水平线 —— 数据量翻倍，它只多走一步。这就是对数时间的美妙之处。

七、考试易错点

⚠️ 陷阱1：看到 x * x 就以为是 O(n²)？
→ 错！x*x 是循环条件中的表达式，不是嵌套循环。循环体内 x *= 2 才决定时间复杂度。
⚠️ 陷阱2：把 O(log n) 和 O(√n) 搞混？
→ 条件是 x² ≤ n，但 x 本身是 2k。所以 22k ≤ n → k ≤ ½·log₂ n。
→ 虽然出现了 √n，但最终复杂度是 O(log n)，因为 x 的增长是指数级别。
⚠️ 陷阱3：log 的底数不重要？
→ 对！O(log₂ n)、O(log₁₀ n)、O(ln n) 都是等价的，因为换底公式只差一个常数因子：loga n = loga b · logb n。所以一律写成 O(log n)。

八、举一反三：快速判断练习

代码片段时间复杂度原因
for(i=1; i<n; i*=2) O(log n) i 每次翻倍
for(i=n; i>0; i/=2) O(log n) i 每次折半
while(n>1) n/=3; O(log₃ n) = O(log n) n 每次除以3
for(i=1; i<n; i++) O(n) i 每次加1 → 线性
for(i=1; i<n; i*=2)
  for(j=0; j<n; j++)		 O(n log n) 外层 log n × 内层 n
🎯 一句话总结：循环变量以乘法/除法方式变化 → O(log n)；以加法/减法方式变化 → O(n)。本题中 x *= 2 是乘法 → O(log n)。
📖 时间复杂度分析是数据结构第一章的核心内容，也是每年必考的基础题型。掌握"乘法→对数、加法→线性"的直觉即可快速判断。

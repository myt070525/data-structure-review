选择题 · 时间复杂度
分析下述代码的时间复杂度
💡 生活类比：细菌繁殖
想象培养皿里有一个细菌，每小时数量翻倍：1 → 2 → 4 → 8 → 16 → …
问：细菌数量什么时候超过 n 的平方根？
—— 这就是这道题的核心。x 就是"细菌数量"，每次翻倍增长，我们要数一数翻了多少次。
🔍 拆解题意
int x = 1;
while (n >= x * x)   // 条件：x² ≤ n，即 x ≤ √n
    x *= 2;           // x 每次翻倍
循环什么时候停？当 x² > n 时，也就是 x > √n 时。
我们要数的是：x 从 1 翻倍多少次会超过 √n。
📝 逐步推导
x 的演变轨迹：1, 2, 4, 8, 16, … , 2ᵏ
循环继续的条件：2ᵏ ≤ √n（因为 x = 2ᵏ，x² = 2²ᵏ ≤ n，即 2ᵏ ≤ √n）

两边取对数：k ≤ log₂(√n) = ½·log₂n
所以循环次数 k ≈ ½·log₂n
大O表示法只关心增长趋势，常数 ½ 被忽略 → O(log n)
✅ 最终答案
O(log n)
🧠 一句话总结：看到"变量翻倍增长"就是对数时间 O(log n)，翻倍越多越慢，是最优秀的复杂度之一。

选择题 · 逻辑结构
下列结构中，不属于基本逻辑结构的是？
💡 生活类比：人际关系的类型 vs 你住什么房子
「逻辑结构」描述的是 数据之间是什么关系——就像人和人之间可以是朋友（图）、家人（树）、排队的陌生人（线性）、各管各的（集合）。
「存储结构」描述的是 数据在电脑里怎么存放——就像你住的是平房、楼房还是帐篷。一个是关系，一个是住所，不能混为一谈。
哈希（散列）是一种存储方式，不是逻辑关系！
🔍 拆解题意
四类基本逻辑结构：
类型关系类比
集合元素同属一组，无其他关系一个班级的学生名单
线性一对一（前驱→后继）排队：每个人只有一个前面的人和一个后面的人
树形一对多（父→子）公司组织架构：一个老板多个员工
图形多对多社交网络：每个人可以认识多个人
哈希结构（散列表）是数据的 存储方式，不是逻辑结构。
✅ 最终答案
D. 哈希结构
🧠 一句话总结：逻辑结构 = 数据之间的关系（集合/线性/树/图）；存储结构 = 数据在内存中的摆放方式（顺序/链式/哈希/索引）。

选择题 · 线性表
以下哪种结构最适合频繁的插入和删除操作？
💡 生活类比：电影院连排座位 vs 小朋友手拉手
数组 = 电影院连排座位：第5个座位的人走了，后面所有人都要往前挪一个位置。中间加一个人？后面所有人往后挪。
链表 = 小朋友手拉手围圈：小明和小红之间要插入小刚？只需要小明松开小红的手去牵小刚，小刚再牵小红。谁都不用动！
🔍 拆解题意 · 时间复杂度对比
操作顺序表（数组）链表
按位置查找O(1) 一步到位O(n) 要数过去
在头部插入/删除O(n) 全员挪位O(1) 只改指针
在尾部插入/删除O(1)O(1)（有尾指针）
在中间插入/删除O(n) 平均挪一半O(1)（找到位置后）
结论：频繁插入删除 → 链表是王者。
✅ 最终答案
B. 链表
🧠 一句话总结：数组靠下标"飞过去"读，链表靠指针"改牵手"写——插入删除选链表，随机访问选数组。

选择题 · 栈
入栈序列为ABCD，以下哪个出栈序列不可能？
💡 生活类比：装薯片的筒
栈 = 品客薯片筒。先放进去的薯片在最底下，最后放进去的在最上面。
你只能从最上面拿！想拿最底下的？必须先把上面的全拿出来。
入栈 ABCD = 依次把 A、B、C、D 四片薯片塞进筒里（但中间可以随时取！）
🔍 拆解题意
规则：按 A→B→C→D 顺序依次入栈，但 任何时刻都可以选择出栈。
四个选项：A. DCBA   B. CDBA   C. BDAC   D. BADC
用栈一步一步模拟，看哪个做不到。
📝 逐步推导 · 模拟每个选项
A. DCBA ✓ 可以
push A → push B → push C → push D → pop D → pop C → pop B → pop A
顺序：D, C, B, A ✓

B. CDBA ✓ 可以
push A → push B → push C → pop C → push D → pop D → pop B → pop A
顺序：C, D, B, A ✓

C. BDAC ✗ 不可能！
push A → push B → pop B ✓（B出）
栈中剩：[A]
push C → push D → pop D ✓（D出）
栈中剩：[A, C]
现在需要出 A——但栈顶是 C，C 挡着 A 的路！A 在 C 下面。
必须先把 C 出了才能出 A，但序列下一个要的是 A 不是 C。
死局！A 被 C 压在下面出不来。

D. BADC ✓ 可以
push A → push B → pop B → pop A → push C → push D → pop D → pop C
顺序：B, A, D, C ✓
✅ 最终答案
C. BDAC（不可能）
🧠 一句话总结：栈出序列判据——某个元素出栈时，比它早入栈且还没出的元素必须按入栈的逆序依次出。

选择题 · 循环队列
循环队列为空时，front == rear，此时两个指针可以在？
💡 生活类比：旋转餐桌
想象一个圆形的旋转餐桌，上面摆了一圈盘子。有一个"取菜口"（front）和一个"放菜口"（rear）。
盘子全被取光了（队空），取菜口和放菜口自然重合在一起。
但！它们重合在哪里？取决于你之前转了桌子多少次——可能在任何一个位置！
🔍 拆解题意
循环队列用数组实现，front指向队头，rear指向队尾的下一个位置。
队空条件：front == rear（无论它们在数组的哪个下标）
因为队列的操作会让两个指针在数组中"转圈"，所以它们重合时可以在 任意合法下标。
✅ 最终答案
B. 任意下标
🧠 一句话总结：循环队列空时 front==rear，但具体在哪取决于历史操作——就像旋转餐桌的"起点"可以停在任意角度。

选择题 · 压缩存储
以下哪种矩阵可以压缩存储？
💡 生活类比：存照片时只存有用的像素
一张自拍照，背景是纯色——你需要把背景的每一个像素都存下来吗？当然不，只存人脸部分就够了。
矩阵压缩也是这个道理：只存有用的数据，重复的、必然为0的统统不存。
🔍 拆解题意
三种特殊矩阵都可以压缩：
矩阵类型特点可压缩？
对称矩阵a[i][j] == a[j][i]，一半是镜像✓ 只存上三角或下三角
三角矩阵上三角（或下三角）全为0或同一常数✓ 只存非零的那一半
对角矩阵只有主对角线附近几条斜线有非零值✓ 只存那几条对角线
✅ 最终答案
D. 以上全部都可以
🧠 一句话总结：只要矩阵里有规律可循的"冗余"（镜像、空白、带状），就可以压缩存储来省空间。

选择题 · 树
双亲表示法数组 [7,0,-1,2,3,0,0,3]，树高为？
💡 生活类比：族谱卡片
每个孩子身上带一张卡片，上面写着"我爸是谁"。把所有卡片摊在桌上：
小明卡片写"我爸是老王"，老王卡片写"我爸是老张"，老张卡片写"我没爸"……
顺着卡片一路追溯，就能画出整棵家族树！
🔍 拆解题意
数组下标 = 结点编号（0~7），数组值 = 该结点父结点的编号，-1 代表根。
结点编号01234567
父结点70-123003
📝 逐步推导 · 画树
第1步：找根。结点2的父是-1 → 结点2是根
第2步：根据父子关系连线。
　· 结点3的父是2 → 3是2的孩子
　· 结点4的父是3 → 4是3的孩子
　· 结点7的父是3 → 7是3的孩子
　· 结点0的父是7 → 0是7的孩子
　· 结点1,5,6的父都是0 → 1,5,6是0的孩子
📝 计算树高
从根（第1层）到最深叶子（1,5,6）： 根(2)→3→7→0→1/5/6，共 5层。
树高 = 5。
✅ 最终答案
C. 5
🧠 一句话总结：双亲表示法 = 每个结点存"我爸是谁"，从任一结点追溯到-1就是根，路径最长的层数就是树高。

选择题 · 图
下列哪种算法不能判断无向图的连通性？
💡 生活类比：修路的顺序 vs 找路的范围
DFS = 一个人钻进迷宫，一条路走到黑，看能走到哪些房间。
BFS = 从起点一层一层往外探索，像水波扩散。
拓扑排序 = 给任务排先后顺序（比如"先修完高数才能修线代"），它关心的不是"通不通"，而是"谁先谁后"——必须是有向图才谈得上先后。
无向图连方向都没有，拓扑排序根本用不了！
🔍 拆解题意
算法适用场景能判连通性？
DFS（深度优先搜索）有向图 + 无向图✓ 遍历看能否访问所有结点
BFS（广度优先搜索）有向图 + 无向图✓ 同样可以
拓扑排序只适用于有向无环图(DAG)✗ 无向图根本无法拓扑排
✅ 最终答案
B. 拓扑排序
🧠 一句话总结：拓扑排序的前提是"有向"+"无环"，无向图连方向都没有，何谈先后顺序？DFS/BFS才是判断连通性的通用工具。

选择题 · 快速排序
下面哪个序列不可能是一趟快速排序之后的结果？
📋 题目
下面哪个序列不可能是一趟快速排序之后的结果（　）。
A. 18, 10, 2, 88, 71　　B. 3, 14, 8, 27, 31
C. 8, 20, 15, 1, 23　　D. 23, 19, 28, 37, 50
💡 生活类比：军训按身高排队——找基准
教官说："中间这位同学别动！比他矮的站左边，比他高的站右边。"
一趟之后，这位"基准同学"就永远站在正确的位置上了，左边全比他矮，右边全比他高。
快排一趟后也是这个效果：基准归位，左边全≤它，右边全≥它。
🔍 拆解题意
判断方法很简单：逐个位置假设它是基准（pivot），检查是否满足「左边所有元素 ≤ 基准 ≤ 右边所有元素」。
如果五个位置全都无法成为合法基准 → 这个序列不可能是一趟partition的结果！
注意：一趟partition之后，基准必然被放到最终位置，且左右两侧的相对顺序可能任意。
📝 逐个选项推导
A. 18, 10, 2, 88, 71
　· 假设18是基准（位置0）：右边10,2,88,71。10<18✓, 2<18✓, 但88>18✗！右边既有小于18的又有大于18的→18不在正确位置 ✗
　· 假设10是基准（位置1）：左边18 > 10 ✗
　· 假设2是基准（位置2）：左边18,10 全部 > 2 ✗
　· 假设88是基准（位置3）：左边18,10,2全部 < 88 ✓，但右边71 < 88 ✗！✗
　· 假设71是基准（位置4）：左边18,10,2,88中88 > 71 ✗
　→ 五个位置全不满足！不可能！✅ 这就是答案。

B. 3, 14, 8, 27, 31
　· 假设3是基准（位置0）：右边14,8,27,31全部 > 3 ✓。3是全局最小值，作为基准在第一位完全合理。
　→ 可能 ✓

C. 8, 20, 15, 1, 23
　· 假设23是基准（位置4）：左边8,20,15,1全部 ≤ 23 ✓。23是全局最大值，作为基准在最后一位完全合理。
　→ 可能 ✓

D. 23, 19, 28, 37, 50
　· 假设23是基准（位置0）：左边为空✓，右边19<23但28,37,50>23→混在一起，23不在正确位置 ✗
　· 假设19是基准（位置1）：左边23 > 19 ✗
　· 假设28是基准（位置2）：左边23,19全部 < 28 ✓，右边37,50全部 > 28 ✓ → 可能！
　· 假设37是基准（位置3）：左边23,19,28全部 < 37 ✓，右边50 > 37 ✓ → 也可能
　· 假设50是基准（位置4）：左边全部 < 50 ✓ → 也可能
　→ 28、37、50都可以是合法基准 → 可能 ✓
✅ 最终答案
A. 18, 10, 2, 88, 71（不可能！）
🧠 一句话总结：快排一趟的"身份证"——存在一个元素，左边全≤它，右边全≥它。选项A中每个元素都通不过这个检验：18右边有比它大的88也有比它小的10和2；88右边有比它小的71；71左边有比它大的88……五个位置全军覆没，所以A不可能。

选择题 · 排序稳定性
下列排序算法中，不稳定的是？
💡 生活类比：排队时身高相同的人别插队
小明和小红都是170cm。按身高排序前，小明在小红前面。
稳定排序：排完后小明还在小红前面——尊重原始顺序。
不稳定排序：排完后小红可能跑到小明前面——原始顺序被打乱了。
堆排序在调整堆时，元素会"跳来跳去"跨很远交换，所以不稳定。
🔍 拆解题意
常见排序算法稳定性一览：
算法稳定性原因
冒泡排序稳定 ✓相邻比较，相等不交换
插入排序稳定 ✓逐个插入，相等保持顺序
归并排序稳定 ✓合并时相等取左边先
选择排序不稳定 ✗选最小交换时可能打乱顺序
快速排序不稳定 ✗partition交换会跨很远
堆排序不稳定 ✗建堆和调整堆时跳跃交换
希尔排序不稳定 ✗分组跳跃插入
✅ 最终答案
D. 堆排序
🧠 一句话总结：稳定的排序尊重"先来后到"（冒泡/插入/归并），不稳定的排序会"插队"（选择/快排/堆排/希尔）。

填空题 · 算法特性
算法的五大特性：有穷性、确定性、可行性、输入、输出。缺少哪一个？
💡 生活类比：菜谱 vs 无限循环
算法 = 做菜的菜谱。菜谱必须有"最后一步"——盛盘上桌。
如果一个菜谱写"搅拌 → 搅拌 → 搅拌 → ……"永远不停，你永远吃不到菜。
有穷性就是保证算法执行有限步后一定会停下来。
✅ 最终答案
有穷性
🧠 一句话总结：算法必须在有限步骤内结束——无限循环不是算法，是bug。

填空题 · 顺序表
在长度为n的顺序表中删除第i个元素，需要移动多少个元素？
💡 生活类比：电影院中间的人走了
一排10个人坐着看电影。第4个人起身走了——第5到第10这6个人都要往前挪一个座位。
删第i个 → 后面的 (n - i) 个人都要动。
📝 推导
数组下标：0, 1, …, i-2, i-1, i, …, n-1
删除位置i-1（即第i个元素）。需要移动的元素：从位置i到位置n-1。
数量 = (n-1) - i + 1 = n - i
✅ 最终答案
n - i
🧠 一句话总结：删除第i个元素，后面n-i个元素依次前移，删除越靠前代价越大。

填空题 · 后缀表达式
后缀表达式 2 3 * 7 - 6 4 + * 的求值结果
💡 生活类比：计算器怎么算 2×3-7× (6+4)？
计算机不喜欢中缀表达式（有括号和优先级），它喜欢 后缀表达式——运算符放在两个操作数后面。
就像你先说"2和3给你"，然后说"乘"；再说"7给你"，然后"减"……
📝 逐步推导 · 栈的变化
表达式：2 3 * 7 - 6 4 + *

① 遇到2：push → 栈：[2]
② 遇到3：push → 栈：[2, 3]
③ 遇到*：pop 3, pop 2 → 2×3=6 → push 6 → 栈：[6]

简答题 · Floyd算法
Floyd算法求所有顶点对最短路径的D矩阵
💡 生活类比：允许中转的航线规划
从北京直飞纽约很贵。但如果从北京→东京→纽约，可能更便宜！
Floyd算法就是反复问自己："如果允许经过城市0中转，有没有更短的路线？允许经过城市0和1呢？……"
逐步放开中转站，最终找到任意两城市间的最短路径。
🔍 核心公式
D(k)[i][j] = 只允许经过 0~k 号顶点中转时，i到j的最短距离。
递推：D(k)[i][j] = min( D(k-1)[i][j] , D(k-1)[i][k] + D(k-1)[k][j] )
即：要么不走k，要么经过k中转。
初始 D(-1) = 邻接矩阵（无边=∞，自身=0）。
图：左侧为3顶点有向带权图，右侧为Floyd算法D(-1)到D(2)的4个矩阵变化（高亮为新发现的最短路径）
📝 三重循环演示
for k = 0 to n-1:
　for i = 0 to n-1:
　　for j = 0 to n-1:
　　　D[i][j] = min(D[i][j], D[i][k] + D[k][j])
时间复杂度 O(n³)，但能一次性求出所有点对的最短路径。
✅ 答题要点
D(-1)邻接矩阵 → D(0)允许经0中转 → D(1)允许经0,1 → ... → D(n-1)即为最终最短路径矩阵
🧠 一句话总结：Floyd = 逐步开放中转点，三重循环不断"如果绕道k会不会更短"，最终得到所有点对最短路径。

关键：利用有序性——重复元素都挤在一起，只需比较相邻结点。
📝 思路推导
双指针法：
　· p 指向当前已确认"不重复"的最后一个结点
　· q = p→next，向前探索
　· 如果 q→data == p→data：q是重复的，删掉q（p→next = q→next, free(q)）
　· 如果 q→data != p→data：q是新值，p后移（p = q）
　· 继续直到链表末尾

时间复杂度 O(n)，空间复杂度 O(1)。
✅ 参考代码
void Slim(LinkList &L) {
    if (L == NULL || L->next == NULL) return;  // 空表或只有头结点
    LNode *p = L->next;   // p指向第一个数据结点
    while (p != NULL && p->next != NULL) {
        LNode *q = p->next;
        if (q->data == p->data) {
            // 重复：删除q
            p->next = q->next;
            free(q);
        } else {
            // 不重复：p后移
            p = p->next;
        }
    }
}
🧠 一句话总结：有序链表的重复元素必然相邻，用两个指针一前一后比大小，相同就删后面的，不同就一起前进。

编程题 · 顺序表
编写 Merge 函数：合并两个有序顺序表
💡 生活类比：合并两叠按身高排好的学生名单
左手一叠名单（按身高递增），右手一叠名单（也按身高递增）。要合并成一叠。
每次都看最上面的两张，谁矮就先拿出来。直到一叠空了，把另一叠剩下的全倒进去。
🔍 拆解题意
输入：两个有序顺序表 A 和 B
输出：一个新的有序顺序表 C，包含A和B的所有元素
核心：归并思想——双指针分别遍历A和B，每次取较小的放入C。
📝 思路推导
① 分配C的空间（长度 = A.length + B.length）
② 三个指针：i→A, j→B, k→C
③ while (i < A.length && j < B.length)：
　　if A[i] ≤ B[j] → C[k++] = A[i++]
　　else → C[k++] = B[j++]
④ 把A或B剩余的直接复制到C
时间复杂度 O(n+m)，空间复杂度 O(n+m)。
✅ 参考代码
typedef struct {
    int *data;
    int length;
} SeqList;

bool Merge(SeqList A, SeqList B, SeqList &C) {
    C.length = A.length + B.length;
    C.data = (int*)malloc(C.length * sizeof(int));
    if (C.data == NULL) return false;

    int i = 0, j = 0, k = 0;
    while (i < A.length && j < B.length) {
        if (A.data[i] <= B.data[j])
            C.data[k++] = A.data[i++];
        else
            C.data[k++] = B.data[j++];
    }
    // 复制剩余
    while (i < A.length) C.data[k++] = A.data[i++];
    while (j < B.length) C.data[k++] = B.data[j++];
    return true;
}
🧠 一句话总结：归并 = 双指针比大小，小的先走，走完把剩下的全接上——这就是归并排序的核心操作。

编程题 · 二叉树
编写 DestroyTree 函数：后序遍历销毁二叉树
💡 生活类比：拆房子必须先拆房顶再拆墙
你要拆一栋房子。能先拆地基吗？不行——房子会塌下来砸到你。
必须先拆屋顶（叶子），再拆墙壁（中间结点），最后拆地基（根）。
后序遍历 = 左→右→根 —— 正好是"先拆孩子的房子，再拆爸妈的房子"。
🔍 拆解题意
销毁二叉树 = 释放所有结点占用的内存。
为什么必须后序？如果先释放根结点，你就找不到它的孩子了——内存泄漏！
必须先释放左右子树，最后释放根。
核心：递归后序：左子树销毁 → 右子树销毁 → 释放根。
📝 思路推导
① 如果树为空（root == NULL）：直接返回
② 递归销毁左子树：DestroyTree(root→left)
③ 递归销毁右子树：DestroyTree(root→right)
④ 释放根结点：free(root)
⑤ 将根指针置空（通过传引用或返回）：root = NULL

时间复杂度 O(n)，访问每个结点恰好一次。
✅ 参考代码
typedef struct BiTNode {
    int data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;

void DestroyTree(BiTree &T) {
    if (T == NULL) return;           // 空树直接返回
    DestroyTree(T->lchild);         // ① 递归销毁左子树
    DestroyTree(T->rchild);         // ② 递归销毁右子树
    free(T);                         // ③ 释放根结点
    T = NULL;                        // ④ 指针置空，防止野指针
}
🧠 一句话总结：销毁二叉树 = 后序遍历 + free。必须先删孩子再删根——就像拆房子要从上往下，不能先拆地基。

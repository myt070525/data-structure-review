线性表中插入元素的位置范围

在长度为 n 的线性表的第 i 个位置插入元素，i 的合法值应为？
正确答案
C — 1 ≤ i ≤ n+1

一、通俗理解：排队插队

你面前有 5 个人 在排队：A B C D E。你想插队，你能站哪些位置？

i=1↓
A
i=2↓
B
i=3↓
C
i=4↓
D
i=5↓
E
i=6↓

你可以站在最前面（i=1，在A前面），也可以站在最后面（i=6，在E后面），还可以站在任意两人之间（i=2~5）。

5 个人 → 6 个"缝隙" → n 个人 → n+1 个位置

二、插入 vs 删除的区别记忆

📥 插入
1 ≤ i ≤ n+1
可以追加在末尾 → 多出1个位置
📤 删除
1 ≤ i ≤ n
只能删已有的 → 正好 n 个目标
💡 生活类比：插入 = 你排队插队（可以站最前面也可以站最后面）。删除 = 队伍里有人被赶出队伍（只能赶已经在队伍里的人）。

三、逐选项分析

A1 ≤ i ≤ n❌ 缺了 n+1，不能插到末尾
B0 ≤ i ≤ n❌ 多出 i=0（编号从1开始），又缺 n+1
C1 ≤ i ≤ n+1✅ 完美覆盖所有可插入位置
D1 ≤ i ≤ n-1❌ 缺得最多

四、代码验证

典型的顺序表插入代码中，合法性检查这样写：

int ListInsert(SeqList *L, int i, int x) {
if (i < 1 || i > L->length + 1) // i>n+1 才非法
return 0;
// ... 后移元素，放入x，length++
}

i>n+1 非法 → 合法的是 i ≤ n+1。代码是最诚实的。

🎯 通俗记忆："插比删多一，插入 n+1，删除只有 n。" 只要记住这句话，这题永远不会错。

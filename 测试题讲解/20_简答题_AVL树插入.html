AVL树插入操作与平衡旋转

📋 题目

已知如下AVL树T，依次插入关键字 20 和 21，画出每次插入后的树形，并回答：

（1）插入完成后，结点20的左孩子是____，右孩子是____；

（2）最终的AVL树中，叶结点个数是____。

初始AVL树 T： 30 / \ 16 50 \ / \ 27 45 60

✅ 参考答案

20的左孩子 = 16，20的右孩子 = 27，叶结点个数 = 4（叶结点为：16、21、45、60）。

一、什么是AVL树？

AVL树（Adelson-Velsky and Landis Tree）是最早被发明的自平衡二叉搜索树（Self-Balancing BST）。在普通的二叉搜索树（BST）中，如果插入序列恰好是有序的（例如1,2,3,4,5），树会退化成一条链表，查找效率从O(log n)暴跌到O(n)。AVL树通过一个巧妙的约束避免了这种退化。

🔑 核心定义

AVL树是一棵二叉搜索树，且满足：任意结点的左子树与右子树的高度差（平衡因子）的绝对值不超过1。

即，对任意结点，|平衡因子| ≤ 1，其中 平衡因子 BF = 左子树高度 − 右子树高度。

💡 记忆口诀："左高减右高，正负不超一"。BF ∈ {−1, 0, 1} 是平衡的；BF = ±2 则需要旋转修复。

1.1 平衡因子详解

我们为每个结点计算平衡因子（Balance Factor, BF）：

  • BF = 0：左右子树一样高 —— 完美平衡。
  • BF = +1：左子树比右子树高1层 —— 左倾，但仍平衡。
  • BF = −1：右子树比左子树高1层 —— 右倾，但仍平衡。
  • BF = +2：左子树太高了！需要右旋或左右双旋。
  • BF = −2：右子树太高了！需要左旋或右左双旋。

1.2 树高与平衡因子的计算

树高的递推定义：叶子结点高度为1（有些教材定义为0，本文统一使用叶子高度=1）；非叶子结点的高度 = max(左子树高度, 右子树高度) + 1。计算从叶子向上逐层进行。

例 对初始树T计算各结点的高度和BF：
结点16：左高=0（空），右高=1（27是叶子），高度=max(0,1)+1=2，BF=0−1=−1 ✓ 结点27：叶子，高度=1，BF=0 ✓ 结点45：叶子，高度=1，BF=0 ✓ 结点60：叶子，高度=1，BF=0 ✓ 结点50：左高=1，右高=1，高度=2，BF=0 ✓ 结点30：左高=2（16子树），右高=2（50子树），高度=3，BF=0 ✓ 全部|BF|≤1，初始树是AVL树 ✓

二、四大旋转操作详解

当插入导致某个结点的|BF|=2时，必须通过旋转来恢复平衡。旋转的本质是：在不破坏BST性质的前提下，提升中间高度子树的根结点，使树重新变"矮"变"扁"。四种旋转对应四种失衡形态：

① LL → 右单旋 (R)

A(+2) B(0) / → / \ B(+1) C A / C

左子树的左子树插入导致A左倾+2。对A做一次右旋。

② RR → 左单旋 (L)

A(-2) B(0) \ → / \ B(-1) A C \ C

右子树的右子树插入导致A右倾−2。对A做一次左旋。

③ LR → 左右双旋 (LR)

A(+2) C(0) / → / \ B(-1) B A \ C

左子树的右子树插入。先对B左旋，再对A右旋。

④ RL → 右左双旋 (RL)

A(-2) C(0) \ → / \ B(+1) A B / C

右子树的左子树插入。先对B右旋，再对A左旋。

🧠 如何判断旋转类型？
从被破坏平衡的最低结点A出发，沿插入路径向下走两步：
  • 第一步方向 = 插入在A的左还是右？
  • 第二步方向 = 插入在下一步结点的左还是右？
两步方向组合即为旋转类型：左-左→LL，左-右→LR，右-左→RL，右-右→RR。
简记：LL和RR是单旋（"顺拐"），LR和RL是双旋（"拐弯"）。

2.1 旋转的"物理直觉"

想象用一根绳子提着AVL树的根结点。如果某侧太重，树就会向那边"倒"。旋转相当于换一个提绳子的位置——把中间那个结点提起来当新的局部根，两边自然下垂成为它的子树。这正是为什么旋转后BST性质不破：新根的值恰好在旧根和旧子之间，左小右大的顺序保持不变。

六、常见错误与避坑指南

❌ 错误1：混淆"高度"与"深度"

高度是从该结点到最远叶子的边数（或结点数，取决于教材约定）。深度是从根到该结点的边数。计算BF用的是子树高度，不是深度。

❌ 错误2：旋转后忘记更新高度

每次旋转后，涉及旋转的结点（通常2~3个）高度都会变化。必须自底向上重新计算：先更新孩子，再更新父结点。

❌ 错误3：判断旋转类型只看局部

必须从最低失衡结点开始，沿插入路径走两步。只看失衡结点自身的BF（+2还是−2）只能判断第一步方向，第二步必须看它那个"高"孩子的BF。

❌ 错误4：忘记检查更上层的祖先

旋转修复了最低失衡结点后，更上层的祖先可能仍然失衡（虽然单次插入最多一个失衡点，但旋转可能改变上层高度）。养成好习惯：旋转后一路向上重新检查BF到根。

七、旋转类型判断流程图

  找到最低失衡结点A（|BF|=2）
       │
       ▼
  A的BF = +2（左高）？
     │ YES
     │   │
     │   ▼
     │ A的左孩子B的BF？
     │   ├── B.BF ≥ 0  →  LL型 → 对A做右单旋
     │   └── B.BF = −1 →  LR型 → 先对B左旋，再对A右旋
     │
     │ NO（即A.BF = −2，右高）
     │   │
     │   ▼
     │ A的右孩子B的BF？
     │   ├── B.BF ≤ 0  →  RR型 → 对A做左单旋
     │   └── B.BF = +1 →  RL型 → 先对B右旋，再对A左旋
🧠 记忆技巧：
• LL/RR是"同号"（A和B的BF符号相同）→ 单旋
• LR/RL是"异号"（A和B的BF符号相反）→ 双旋
• 单旋的旋转方向与A的BF符号相反（A=+2→右旋，A=−2→左旋）
• 双旋：先旋B（方向与B的BF符号相反），再旋A（方向与A的BF符号相反）

八、为什么AVL树值得学习？

AVL树虽然在工业界不如红黑树流行（红黑树的旋转次数更少，插入删除的常数因子更低），但它是理解自平衡数据结构的最佳入门：

  • 严格的平衡条件使分析变得清晰：高度严格控制在1.44 log₂(n+2) − 0.328 以内。
  • 四种旋转覆盖了所有可能的不平衡情况，形成了一个完备的操作集合。
  • 理解了AVL的旋转，再学红黑树的"变色+旋转"就如履平地。
  • 在查找密集型场景（如数据库索引、字典），AVL树仍有实际应用。

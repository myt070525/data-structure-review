一、什么是堆（Heap）？

1.1 堆的定义

堆是一种完全二叉树，同时满足堆的性质：

  • 大根堆（Max-Heap）：每个父节点的值 ≥ 所有子节点的值。根节点是全局最大值。
  • 小根堆（Min-Heap）：每个父节点的值 ≤ 所有子节点的值。根节点是全局最小值。

堆排序使用大根堆来实现升序排序。

1.2 完全二叉树的数组表示

堆通常用数组存储，父子关系通过下标计算（0-based索引）：

  • 节点 i 的左孩子：2i + 1
  • 节点 i 的右孩子：2i + 2
  • 节点 i 的父节点：⌊(i-1)/2⌋
  • 最后一个非叶节点：⌊n/2⌋ - 1
💡 记忆口诀："左孩2i+1，右孩2i+2，找爹(i-1)/2向下取整。"

五、堆排序伪代码

// ========== 向下调整（siftDown / heapify）========== function siftDown(arr, i, heapSize): largest = i left = 2 * i + 1 right = 2 * i + 2 if left < heapSize and arr[left] > arr[largest]: largest = left if right < heapSize and arr[right] > arr[largest]: largest = right if largest != i: swap(arr[i], arr[largest]) siftDown(arr, largest, heapSize) // 递归向下 // ========== 堆排序主函数 ========== function heapSort(arr): n = arr.length // 阶段一：建堆（从最后一个非叶节点向前） for i = floor(n/2) - 1 downto 0: siftDown(arr, i, n) // 阶段二：排序 for i = n - 1 downto 1: swap(arr[0], arr[i]) // 堆顶→末尾 siftDown(arr, 0, i) // 恢复堆

六、复杂度分析 & 记忆技巧

指标值说明
时间复杂度（建堆）O(n)虽然每个节点可能下沉log n层，但越下层节点数越多、下沉次数越少，级数求和为O(n)
时间复杂度（排序）O(n log n)n-1次交换，每次siftDown O(log n)
总时间复杂度O(n log n)最好/最坏/平均都是O(n log n)
空间复杂度O(1)原地排序，仅用常数辅助空间
稳定性不稳定交换可能破坏相等元素的相对顺序
⚠️ 常见错误1：建堆方向搞反。必须从最后一个非叶节点向前依次siftDown，不能从根开始。从根开始无法保证子树先满足堆性质。
⚠️ 常见错误2：siftDown时忘记检查heapSize。排序阶段堆在不断缩小，必须用heapSize限制范围，否则会把已"沉底"的元素重新拉回来。
💡 记忆口诀："建堆从下往上调，排序每次把顶抛。交换之后往下沉，n-1趟就排好。"
💡 考试快速推演法：① 画成完全二叉树；② 从⌊n/2⌋-1向前逐一siftDown建堆；③ 每趟：交换(0, heapSize-1)，heapSize--，siftDown(0)。记录每趟结果即可。

🃏 直接插入排序（Insertion Sort）

✅ 第1趟: [3, 6, 1, 4, 5, 2]  |  第2趟: [1, 3, 6, 4, 5, 2]

一、核心思想：打牌时"摸牌—插牌"

1.1 经典比喻

想象你在打扑克牌，摸到一张新牌后，你会：

  1. 右手拿起新牌（当前待插入元素）。
  2. 在左手已排好的牌中从右向左扫描。
  3. 找到第一个比新牌小的位置，把新牌插入它后面。
  4. 插入时，比新牌大的牌都要向右挪一位"腾位置"。
左手（已排序）： 3 6 ← 右手（新摸的牌）： 1

把1插入到3前面 → [1, 3, 6]

1.2 算法框架

将数组分为已排序区间（左侧）和未排序区间（右侧）。初始时第一个元素自成已排序区间。每趟从未排序区间取第一个元素，插入到已排序区间的正确位置。

五、插入排序伪代码

function insertionSort(arr, n): // 外层循环：从未排序区间逐个取元素 for i = 1 to n-1: key = arr[i] // "右手"摸到的新牌 j = i - 1 // 从已排序区间的末尾开始比较 // 内层循环：从右向左找插入位置，一边找一边右移 while j >= 0 and arr[j] > key: arr[j+1] = arr[j] // 比key大的元素向右挪一位 j = j - 1 arr[j+1] = key // 插入到正确位置

六、复杂度分析 & 特性

指标值说明
最好时间复杂度O(n)数组已经有序时，内层while一次都不执行
最坏时间复杂度O(n²)数组完全逆序时，每趟需移动所有已排序元素
平均时间复杂度O(n²)平均每趟需移动i/2个元素
空间复杂度O(1)原地排序，仅用key和j两个辅助变量
稳定性✅ 稳定while条件用 >（不用≥），相等元素相对顺序不变

为什么插入排序在小规模数据中很受欢迎？

  • 常数因子极小——没有递归开销，没有额外内存分配。
  • 对于"基本有序"的数组（如每个元素离最终位置不超过k），时间复杂度接近O(kn)。
  • TimSort和Introsort（C++ std::sort）在子问题规模小时会切换到插入排序。

七、常见错误 & 记忆技巧

⚠️ 错误1：while条件写成 arr[j] ≥ key。
如果使用 ≥，相等元素也会被移动，破坏稳定性。正确写法是 arr[j] > key。
⚠️ 错误2：忘记保存key，覆盖后丢失原始值。
arr[i]在移动过程中会被覆盖，必须先用 key = arr[i] 保存。
⚠️ 错误3：数组越界。j从i-1减到-1时必须停止。
while条件必须同时检查 j >= 0 和 arr[j] > key（先检查边界！）。
💡 记忆口诀："外层i从1到n-1，key存当前值别忘记。内层j从i-1往左移，大的右移小的停，最后key放j+1。"
💡 考试快速推演法：① 画一条竖线，线左边是已排序，线右边是未排序。
② 每趟把线右边第一个元素拿到左手，在左手从右向左找位置插入。
③ 找位置的过程中把比它大的元素都右移一格。
④ 插入后，竖线向右挪一位。重复n-1次。

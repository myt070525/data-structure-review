📝 填空第2题 · 后缀表达式

8*(5-2)/(7+3) → 后缀 = ? 扫描到+时栈中元素数 = ?

后缀：8 5 2 - * 7 3 + /
扫描到+时栈中元素数：3
（分别是 24, 7, 3）

一、三种表达式

中缀（infix）：操作符在中间 → 3 + 5。人来读写很自然，但计算机处理需要括号和优先级规则，很麻烦。

后缀（postfix / 逆波兰式）：操作符在后面 → 3 5 +。不需要括号！计算机只需一个栈就能轻松计算。

通俗类比：中缀是"3加5"（人话），后缀是"3, 5, 加起来"（机器指令的顺序）。

二、操作符优先级 — 核心规则

中缀转后缀时，操作符不能直接输出——要先压入一个"操作符栈"。什么时候压？什么时候弹？看优先级。

优先级操作符含义结合性
3（最高）^乘方右结合
2*   /   %乘、除、取余左结合
1（最低）+   -加、减左结合
括号 ( ) 不参与优先级比较——它们是"特权通道"

什么叫"左结合"？同等优先级的操作符从左往右算。比如 8/4*2 = (8/4)*2 = 4，不是 8/(4*2)=1。

"右结合"呢？从右往左算。比如 2^3^2 = 2^(3^2) = 2^9 = 512，不是 (2^3)^2 = 64。考试一般不考^。

三、中缀转后缀 — 三条铁律

📌 规则一：遇到操作数（数字）

直接输出。数字永远是"先来先走"，不参与任何优先级争斗。

📌 规则二：遇到操作符（+ - * /）

先弹后压：把操作符栈中所有"优先级 ≥ 当前操作符"的操作符弹出来输出，
然后再把当前操作符压入栈。

为什么"≥"呢？因为相同优先级时，先来的要先算（左结合）。所以栈顶那个*如果和我同级或更高，它应该比我先输出。

📌 规则三：遇到括号

( 左括号：无条件压栈，当作"优先级最低"。
) 右括号：把栈中操作符逐个弹出输出，直到遇到 ( 为止。然后丢弃这对括号。

括号的妙用：左括号把栈"封"住了，里面的操作符出不来。右括号一来，就把它们全放出来。括号本身不输出。

📌 收尾：表达式扫描完后

把操作符栈中剩余的符号全部弹出输出。

五、最容易搞混的两步 — 慢镜头回放

🔍 第8步：为什么 / 要把 * 弹出来？

* 和 / 的优先级都是 2。同优先级的判断口诀：先来的先走。

* 比 / 先入栈 → 当 / 来时，* 先被弹出 → 后缀中 * 出现在 / 前面 → 计算时先算乘后算除。

这就是"左结合"的体现：从左到右，先遇到的先算。如果允许 / 压在 * 上面，那后缀里 / 就会在 * 前面 → 变成先除后乘，结果就错了。

🔍 第5步：为什么 - 不弹 ( ？

栈顶是 (。它不是一个真正的操作符——它是"封印"。任何操作符遇到 ( 都不能弹出它，只能乖乖排在它后面。

直到对应的 ) 出现，才会把这对括号之间积压的操作符一次性释放。

六、优先级规则一句话总结

栈顶 ≥ 当前
→ 弹出栈顶
（先来的、高级的，先走）
栈顶 < 当前
→ 当前压栈
（后来的、高级的，等一等）
遇到 (
→ 无条件压栈
（封印，阻止弹出）
遇到 )
→ 弹到 ( 为止
（解除封印，释放操作符）
⚠️ 常见错误：很多人把"栈顶≥当前→弹出"记反了！不是"当前优先级高就压栈"，是栈顶的优先级≥当前时弹出栈顶。想象栈顶是"先来的人"，当前是"新来的"——如果新来的不比先来的更高级，先来的就先出去办事。
🎯 练习方法：找10个中缀表达式，手工转换后缀，再用后缀求值验证——做对3个，这题就彻底会了。

七、后缀求值（第二空）

扫描 8 → push → 栈=[8] 扫描 5 → push → 栈=[8,5] 扫描 2 → push → 栈=[8,5,2] 扫描 - → pop 2, pop 5 → 5-2=3 → push → 栈=[8,3] 扫描 * → pop 3, pop 8 → 8*3=24 → push → 栈=[24] 扫描 7 → push → 栈=[24,7] 扫描 3 → push → 栈=[24,7,3] ← 3个元素！ 扫描 + → pop 3, pop 7 → 7+3=10 → push → 栈=[24,10] 扫描 / → pop 10, pop 24 → 24/10=2.4 → push → 栈=[2.4]

一、哈希表的基本概念

哈希表（Hash Table），又称散列表，是一种通过关键字直接计算存储位置的数据结构。它的核心思想是：设计一个函数 H(key)，将任意关键字映射到固定范围内的地址，理想情况下插入、查找、删除都是O(1)时间。

🔑 核心三要素

  1. 哈希函数 H(key)：将关键字映射为地址（0 ∼ m−1）。
  2. 哈希表：长度为m的数组，存储实际元素。
  3. 冲突解决方法：当两个不同关键字映射到同一地址时（即冲突/collision），如何处理。

1.1 什么是冲突？

由于关键字空间通常远大于地址空间（例如所有可能的整数 vs 11个槽位），冲突是必然的。好的哈希函数可以减少冲突，但无法消除冲突。因此，冲突解决方法的质量直接决定了哈希表的性能。

1.2 线性探测法（Linear Probing）

线性探测是开放地址法（Open Addressing）中最简单的一种。当插入关键字key时：

  1. 计算初始位置 d₀ = H(key)。
  2. 若该位置为空 → 插入，结束。
  3. 若该位置已被占用 → 线性地往后找：d₁ = (d₀+1) % m，d₂ = (d₀+2) % m，……直到找到空位。
💡 形象比喻：就像去电影院找座位。你的票上写着第5排，但到了发现那个座位有人了。线性探测就是：看看第6排、第7排……一直往后找，找到第一个空位就坐下。如果走到最后一排还没找到，就绕回第0排继续找（因为表是循环的）。

四、计算ASL（平均查找长度）

4.1 每个关键字的查找长度

查找长度定义为：在查找成功的前提下，比较关键字的次数。对于线性探测法，查找某个key的过程与插入时探测的路径完全一致——从H(key)开始，线性探测直到找到该key。

4.2 求和并计算ASL

Σ 查找长度 = 1 + 1 + 1 + 2 + 1 + 2 + 2 = 10

📐 解读：平均每个关键字的查找只需约1.43次比较。相比顺序查找的5.5次和折半查找的2.9次（上一题数据），哈希表的查找效率非常高。而且随着表长增大、装填因子降低，ASL可以无限接近1。

五、查找失败时的ASL（拓展）

虽然题目只要求ASL成功，但理解ASL失败对掌握哈希表很重要。查找失败发生在：要查的key不在表中，算法沿探测路径一直找到空位才能确认"不存在"。

例如，查找key=6：H(6)=6。检查位置6（38，不匹配），位置7（17，不匹配），位置8（空！）。比较了3次才确认不存在。对每个可能的初始地址（0~10）计算这样的探测长度，取平均即得ASL失败。

六、线性探测的问题：聚集现象

⚠️ 一次聚集（Primary Clustering）

线性探测最严重的问题是聚集：一旦某个区域出现了连续被占用的槽位，后续任何哈希到该区域附近的关键字都会被"吸"进去，导致该区域越来越拥挤，查找长度不断增长。

从我们的最终表中可以看到：地址5-6-7形成了连续的占用块（5,38,17），地址10-0形成了另一个块（21,32）。这种聚集会随着装填因子α的增大而急剧恶化。

6.1 改进方法

  • 二次探测（Quadratic Probing）：探测步长变为 1², −1², 2², −2², …（或 1², 2², 3², …），减少一次聚集。
  • 双重哈希（Double Hashing）：使用第二个哈希函数决定步长，探测序列为 d₀, d₀+h₂(key), d₀+2·h₂(key), …。
  • 链地址法（Separate Chaining）：每个槽位存放一个链表，冲突的元素挂在同一个链表上。这是Java HashMap采用的方案。

七、常见错误与避坑指南

❌ 错误1：余数算错

key % 11 的余数范围是0~10。例如89÷11=8余1（不是7）。36÷11=3余3。建议列竖式验证：被除数 = 除数×商 + 余数。

❌ 错误2：忘记模运算导致数组越界

线性探测的每一步都要对m取模：(d + i) % m。尤其是当探测到表尾（m−1位置）后，下一个位置应为0（循环回去），而不是m（越界）。

❌ 错误3：查找长度不是"探测步数"

查找长度 = 比较关键字的次数，而不是探测的地址个数。如果初始位置就是目标，比较1次即命中。如果初始位置被占但值不同，需要再探测下一个位置再比较——每次比较都计数。

❌ 错误4：混淆"插入时比较次数"与"查找时比较次数"

对于线性探测，插入时和查找时的探测路径完全相同，因此比较次数也相同。但这是开放地址法的特性，链地址法则不同（插入总是在链表头，查找可能需要遍历链表）。

八、ASL公式速查（线性探测）

设装填因子 α = n/m（n为元素个数，m为表长），则线性探测的近似ASL为：

ASL成功 ≈ ½ · (1 + 1/(1−α))
ASL失败 ≈ ½ · (1 + 1/(1−α)²)

本题α=7/11≈0.636，代入得：ASL成功 ≈ ½(1+1/0.364) ≈ ½(1+2.75) ≈ 1.87。公式值为近似值，与精确计算（10/7≈1.43）有偏差是因为n较小，公式在大n下更准确。

🧠 记忆技巧：
• 线性探测 = "一条路走到黑"，顺序往后找空位
• 查找长度 = 比较次数，不是步数
• 模运算保证循环：d = (d+1) % m
• α越大 → 聚集越严重 → ASL增长越快

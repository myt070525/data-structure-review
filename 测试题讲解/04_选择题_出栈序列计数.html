栈的出栈序列计数

若入栈序列为 ABCD，在所有可能的出栈序列中，
以 C 开头的出栈序列个数为？
正确答案
C — 3 种
CDBA   CBDA   CBAD

一、栈的"游戏规则"

栈（Stack）是后进先出（LIFO）的数据结构。就像一叠盘子：

  • 入栈（push）：把盘子放到最上面
  • 出栈（pop）：只能拿最上面那个盘子
  • 中间不能抽盘子：不能越过上面的取下面的

这意味着：如果 A 比 B 先进栈，那么 B 一定比 A 先出栈（除非 A 在 B 入栈前就出栈了）。

🚫 出栈序列的充要条件：在出栈序列中，对于任意三个元素 i < j < k（入栈序），
不能出现 k 先出，i 其次，j 最后 的模式（即"大小中"模式）。

二、以 C 开头，逐一手动枚举

前提分析

C 第一个出栈，说明什么？说明入栈一直进行到 C 入栈后立刻把 C 弹出。此时：

已出栈：C
栈中：AB
尚未入栈：D

关键约束：A 在 B 下面，所以 B 必须先于 A 出栈（栈顶在B这边）。D 任何时候入栈都行。

枚举全部可能

剩余元素：A、B、D，其中 B 必须比 A 先出。所有排列：

序列B比A先出？栈合法？结果
C D B A✅ B在A前✅ D入→出, B出, A出✅ 合法
C B D A✅ B在A前✅ B出, D入→出, A出✅ 合法
C B A D✅ B在A前✅ B出, A出, D入→出✅ 合法
C D A B❌ A在B前—❌
C A D B❌ A在B前—❌
C A B D❌ A在B前—❌

总共 3 种合法序列。

三、用决策树思考（更系统的方法）

C出栈后，每一步有两个选择：出栈（pop栈顶）或入栈（push下一个未入栈元素，如果有的话）。

C出栈后（栈=[A,B]，待入=[D]） ├─ 选 pop → B出栈 → （栈=[A]，待=[D]） │ ├─ 选 pop → A出栈 → （栈=[]，待=[D]） │ │ └─ 选 push → D入→出 → CBAD ✅ │ └─ 选 push → D入→出 → （栈=[A]） │ └─ 选 pop → A出栈 → CBDA ✅ │ └─ 选 push → D入栈 → （栈=[A,B,D]，待=[]） ├─ 选 pop → D出栈 → （栈=[A,B]） │ └─ 选 pop → B出→A出 → CDBA ✅ │ └─ 选 pop → B出栈 → ... → 同样得到上面某个序列

四、进阶：卡特兰数 —— n个元素有多少合法出栈序列？

n 个元素的合法出栈序列总数 = 第 n 个卡特兰数：

Cₙ = C(2n, n) / (n+1) = (2n)! / (n!(n+1)!)

五、考试解题技巧

  1. 先确定约束：以 X 开头 → X 之前入栈的元素都还在栈里，顺序已知
  2. 列出余下元素：栈中元素（自底向上）+ 待入栈元素
  3. 用决策树或排列法：保证栈中元素的相对顺序（底部的后出）
  4. 逐一检验：每个候选序列模拟一遍栈操作，验证是否合法
💡 考试抢分法：如果时间紧，直接把所有排列写出来（最多 n! 种），然后逐个模拟栈检验。4个元素只有 24 种排列，1-2分钟就能筛完。
📖 栈是第3章核心内容，出栈序列合法性是必考题型。

怎么判断循环队列为空？

可用于判断循环队列是否为空的代码为？
A、Front == Rear　　B、Front+1 == Rear　　C、Front == 0　　D、Rear == 0
正确答案
A — Front == Rear

一、循环队列的"轮回"设计

循环队列用数组实现，不是直筒子——队尾到了数组末端就绕回到数组开头（模运算）。就像操场跑圈，跑完一圈又回到起点。

问题来了：队空和队满时，Front 和 Rear 都在同一个位置。怎么区分？

常见方案：牺牲一个存储空间，规定当 (Rear+1) % M == Front 时为队满，Front == Rear 时为队空。

通俗理解：就像停车场留一个空位方便倒车——留一个空格来区分空和满。

二、队空 vs 队满

🟢 队空
Front == Rear
队头追上了队尾 → 没东西了
🔴 队满
(Rear+1)%M == Front
队尾快追上队头 → 满了
💡 通俗类比：Front 像顾客，Rear 像厨师。顾客追上厨师 = 没菜了（空）。厨师追上一圈快撞到顾客 = 菜满了要等（满）。

三、逐项排查

AFront == Rear✅ 队空标准条件
BFront+1 == Rear❌ 这是另一种设计风格下的判满或是一种特殊情况，不是标准判空
CFront == 0❌ Front=0 只是队列物理上在数组开头而已，跟空不空没关系
DRear == 0❌ 同理，Rear=0 不代表队空

四、考题关联——测验卷也考了

测验卷第4题：循环队列大小9，初始front=rear=4（空）。操作后front=7，rear=2，求队列元素个数。

公式：元素个数 = (rear - front + M) % M

计算：(2 - 7 + 9) % 9 = 4 % 9 = 4个元素。

🎯 记忆：Front 追 Rear = 空。Rear 追 Front = 满。永远记住这两个条件！

🔗 编程题：有序链表插入（Insert_sort）

第28题 · 向递增有序单链表中插入新元素
✅ 填空答案：
(1) p->Next != NULL   (2) p = p->Next   (3) q->Data = X
(4) q->Next = p->Next   (5) p->Next = q

一、题目理解：我们要做什么？

1.1 问题描述

已知一个带头结点的递增有序单链表，要求编写函数 Insert_sort，将值为 X 的新结点插入到链表的正确位置，保持链表递增有序。（假设链表中没有与X相等的元素）

1.2 核心任务

  1. 定位：找到插入位置的前驱结点p（p之后就是X应该插入的位置）。
  2. 创建：为新结点q分配内存，存入数据X。
  3. 插入：将q插入到p之后，即"挂"在p和p->Next之间。

1.3 链表结构定义

typedef struct LNode { int Data; struct LNode *Next; } LNode, *LinkList;

二、链表插入的"四步法"——核心模式

2.1 通用链表插入范式

在结点p之后插入新结点q，需要执行两步指针操作：

插入前： p ──→ [p->Next 结点] ↑ 要在这里插入 q 插入后： p ──→ q ──→ [原 p->Next 结点]

2.2 关键两步（顺序不可颠倒！）

步骤代码含义为什么不能先做⑤？
第④步 q->Next = p->Next 让新结点q的Next指向p的后继 ⚠️ 如果先执行 p->Next = q，那么 p->Next 已经被改写成q，原来的后继结点地址就丢失了！q->Next 将无法指向正确位置。

规则：永远先让新结点指向后继，再让前驱指向新结点。
第⑤步 p->Next = q 让前驱p的Next指向新结点q

2.3 形象类比：插队

想象你在排队买奶茶。小明（q）想插到小红（p）后面、小刚（原p的后一个人）前面。

  1. 先做④：小明先把手搭在小刚的肩膀上（q->Next = 小刚），这样就不会跟丢小刚。
  2. 再做⑤：小红转身把手搭在小明肩膀上（p->Next = q），完成插入。

如果反过来——小红先搭小明（⑤先做），那小刚在哪儿就彻底不知道了！这就是"断链"错误。

三、完整代码 & 逐行解析

void Insert_sort(LinkList L, int X) { // L: 带头结点的递增有序单链表 // X: 待插入的元素值 LNode *p, *q; // ─── 阶段一：定位插入位置的前驱结点p ─── p = L; // 从头结点开始 while (p->Next != NULL) { // 填空(1)：只要还有后继结点 if (p->Next->Data > X) break; // 找到了！p的后继比X大，X应插在p之后 p = p->Next; // 填空(2)：没找到，p向后移动 } // ─── 阶段二：创建新结点 ─── q = (LNode*)malloc(sizeof(LNode)); q->Data = X; // 填空(3)：存入数据 // ─── 阶段三：执行插入（关键！④必须在⑤之前）─── q->Next = p->Next; // 填空(4)：新结点先指向后继 p->Next = q; // 填空(5)：前驱再指向新结点 }

逐行详细解释

填空代码解释
(1) p->Next != NULL while循环的继续条件：只要p后面还有结点，就继续比较。
如果写成 p != NULL，p最终会走到NULL，之后p->Next就会空指针异常。
如果写成 p->Next->Data < X，等于把循环条件和break条件合并了（也可以，但题目显然分开写）。
(2) p = p->Next p向后移动一步，相当于"指针滑动"——这是遍历链表的最基本操作。
注意：这里p始终指向"待比较结点的前驱"，所以是p先移再比较p->Next。
(3) q->Data = X 为新分配的结点q填入数据X。这是malloc之后的常规操作。
(4) q->Next = p->Next 必须先执行！让新结点q接管p的后继链。此时p->Next还是原来的后继结点。
(5) p->Next = q 修改p的后继指针指向q，完成插入。此时原p->Next的地址已经安全地保存在q->Next中了。

四、可视化演示：向 [2, 5, 8] 中插入 X=6

初始链表

L(head)Next→ → 2Next→ → 5Next→ → 8NULL

阶段一：定位——p指针的移动过程

迭代p指向p->Next指向p->Next->Data与X=6比较动作
初始L (head)结点222 ≤ 6p后移→指向2
第1轮结点2结点555 ≤ 6p后移→指向5
第2轮结点5结点888 > 6 ✓break！p停在5

p最终指向值为5的结点，新结点应插在5之后、8之前。

阶段二：创建新结点q（Data=6）

6? ← 新结点q，malloc后Data=6，Next未初始化

阶段三：执行插入

④ 先执行：q->Next = p->Next（q->Next指向结点8）

5Next→ → 8NULL 6Next→ ↗ q->Next指向8

⑤ 再执行：p->Next = q（结点5的Next指向q）

L → 2 → 5 → 6 → 8

🎉 插入完成！链表变为 [2, 5, 6, 8]，依然递增有序！

五、边界情况分析

情况1：插入到链表头部（X比所有元素都小）

例如链表[3,5,8]，插入X=1。

while循环：p->Next->Data=3 > 1 → 直接break。p停在头结点L。

插入操作：q插在L之后、3之前 → 链表变为[1,3,5,8]。✅ 正确！

情况2：插入到链表尾部（X比所有元素都大）

例如链表[3,5,8]，插入X=10。

while循环：依次比较3≤10、5≤10、8≤10，都满足，p最终指向8。下一轮p->Next==NULL，循环条件不满足退出。

插入操作：q->Next = p->Next = NULL，p->Next = q。链表变为[3,5,8,10]。✅ 正确！

情况3：空链表（只有头结点）

L->Next == NULL → while条件直接不满足 → p停在L。

q->Next = NULL，L->Next = q。链表变为[X]。✅ 正确！

💡 三个边界情况都无需特殊处理——代码已经自然覆盖。这是带头结点链表设计的优雅之处。

六、常见错误 & 考点总结

🔴 致命错误1：④和⑤顺序颠倒！
如果先写 p->Next = q 再写 q->Next = p->Next，第二句的p->Next已经被改成q了，q->Next指向自己，链表断裂。
记忆口诀："先接后，再接前；先指后继，再改前驱。"
⚠️ 错误2：while循环条件写成 p != NULL。
p最终会走到链表尾结点的Next（NULL），但我们需要p停在尾结点而不是NULL。正确条件是 p->Next != NULL。
⚠️ 错误3：p初始化为 L->Next 而非 L。
如果p从第一个数据结点开始，插入到头部的情况就无法处理（找不到头结点作为前驱）。必须从L开始。
⚠️ 错误4：忘记malloc就直接用q。
必须先分配内存，才能使用q->Data和q->Next。
💡 考试心法：这道题的5个空分别考察——
① while循环边界条件
② 链表指针后移
③ 结构体成员赋值
④⑤ 链表插入的双指针操作（顺序是关键！）
理解"为什么④在⑤之前"比记住答案更重要。

七、拓展：链表插入 vs 数组插入

维度有序链表插入有序数组插入
定位O(n) 从头遍历O(log n) 二分查找（但移动仍是O(n)）
插入动作O(1) 改两个指针即可O(n) 需要批量右移元素
总时间O(n)O(n)
空间O(1) 额外O(1) 额外（原地）
优势场景频繁插入/删除的动态数据频繁随机访问的静态数据

⏱️ 填空题第1题 · 时间复杂度

下面程序段的时间复杂度是？

for (i = 0; i < 1000; i ++)
for (j = 0; j < i; j ++)
sum += i + j;
正确答案
O(1)

一、为什么是 O(1) 而不是 O(n²)？

看到两重 for 循环，很多人脱口而出"O(n²)"——这是一个经典的陷阱。

仔细看：外层循环上限是 1000，不是 n！

1000 是一个常量。无论输入规模 n 多大，这个循环最多执行 1000×999/2 ≈ 500,000 次。这是一个固定的数字，不随 n 变化。

时间复杂度衡量的是"随着输入规模 n 增大，执行时间怎么增长"。如果执行时间和 n 无关，那就是常数阶 O(1)。

二、通俗理解

假设你有一个函数 f(n)：

  • 如果 n=10，执行 500,000 次
  • 如果 n=100万，还是执行 500,000 次
  • 如果 n=100亿，依然是 500,000 次

执行次数不变 = 常数时间 = O(1)。

⚠️ 关键判断：循环上限是常量还是变量 n？常量 → O(1)；变量 n → O(n)或更高。
🎯 一句话：O(1) 不代表快，只代表"不管你输入多大，我都是一个速度"。500,000 次也可以算 O(1)，因为它是常数。

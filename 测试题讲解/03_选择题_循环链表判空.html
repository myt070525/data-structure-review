循环链表怎么判断为空？

L 是带头结点的循环链表，判断 L 是否为空的代码为？
正确答案
B — L->Next == L

一、先搞懂什么叫"带头结点的循环链表"

拆开三个概念：

  • 带头结点：链表有一个"虚拟"的头结点（不存实际数据），只是方便操作。头结点用 L 指向。
  • 循环：最后一个结点的 Next 指针不是 NULL，而是指向头结点，形成一个环。
  • 空表：头结点后面没有数据结点了。也就是说，头结点的 Next 指向自己。

通俗地说：头结点是"哨兵"，站在队伍最前面。当队伍里只有哨兵一个人的时候，哨兵后面还是哨兵自己——表是空的。

二、图解：空表 vs 非空表

空表：头结点指向自己 L->Next==L ｜ 非空表：头结点→数据结点→...→头结点

三、逐项辨析

AL->Next == NULL❌ 这是不带头结点的单链表的判空条件。循环链表没有 NULL！
BL->Next == L✅ 带头结点的循环链表，头结点指向自己 = 空表。
CL == NULL❌ L 是头结点的地址，只要链表存在，L 就不会是 NULL（因为头结点已经分配了）
DL[0] == NULL❌ 数组写法，链表根本没有下标操作。这是凑数的干扰项。

四、对比：各种链表的判空条件

链表类型判空条件说明
不带头结点的单链表L == NULL连头结点都没有
带头结点的单链表L->Next == NULL头结点后面没东西
不带头结点的循环链表L == NULL啥都没有
带头结点的循环链表L->Next == L✅ 本题答案
🎯 通俗记忆：循环链表的空，就是"头结点自己在转圈圈"。L->Next == L，自己指自己，环里只有一个人。

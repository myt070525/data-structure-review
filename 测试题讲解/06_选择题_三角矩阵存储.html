📐 三角矩阵的压缩存储

用一维数组存储 n 阶三角矩阵，需要多大空间？

正确答案
C — n(n+1)/2

一、什么是三角矩阵？为什么能"压缩"？

三角矩阵中，对角线一侧的所有元素都是同一个常数（通常是0或∞）。既然都是同一个值，就不用每个都存——只存有用的那些，常数只存1次。

紫色 = 需要存储的元素（下三角+对角线）。灰色 = 同一个常数c，存1次即可。

二、推导：到底需要多少个空间？

下三角（含对角线）的元素个数：

1 + 2 + 3 + ... + n = n(n+1)/2

第1行有1个（a₁₁），第2行有2个（a₂₁,a₂₂），...，第n行有n个。这是等差数列求和。

加上存储那个常数 c 的 1 个空间，总共 n(n+1)/2 + 1。但选项里没有这个"+1"，说明题目考的是有效数据元素的数量本身。

三、逐项辨析

An/2❌ 太小了。就算只有对角线也要 n 个
Bn·n/2❌ n²/2 = 全矩阵的一半。没算对角线被减半
Cn(n+1)/2✅ 等差数列求和，正确
Dn(n-1)/2❌ 少了对角线上的 n 个元素
🎯 同一个公式扩展：
下三角矩阵 → n(n+1)/2
对称矩阵 → n(n+1)/2（因为只存一半+对角线）
三对角矩阵 → 3n-2（只有三条对角线有数据）
💡 通俗理解：一个楼梯，第1级1个砖，第2级2个砖...第n级n个砖。总砖数 = n(n+1)/2。三角矩阵就是这样一个"数据楼梯"。

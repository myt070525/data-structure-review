🌿 编程题：叶结点链表化（BuildLeafList）

第27题 · 将二叉树的所有叶结点串成双向链表
✅ 填空答案：(1) T->Left!=NULL || T->Right!=NULL   (2) BuildLeafList(T->Left)   (3) BuildLeafList(T->Right)   (4) PreLeaf->Right=T   (5) T->Left=PreLeaf

一、题目理解：我们要做什么？

1.1 问题描述

给定一棵二叉树，要求编写递归函数 BuildLeafList，将树中所有叶结点（没有孩子的结点）串成一个带头结点的双向循环链表。叶结点原本的 Left 和 Right 指针（原本是NULL）被复用为双向链表的前驱和后继指针。

1.2 关键概念

  • 叶结点：左右孩子均为NULL的结点。它是树的"末端"。
  • 非叶结点：至少有一个孩子不为NULL的结点（包括只有一个孩子的分支结点）。
  • 头结点H：一个哑结点（dummy node），不存储实际数据，仅作为链表的起点和终点。
  • 全局指针PreLeaf：始终指向最近访问到的叶结点，用于链表的双向链接。

1.3 遍历策略：中序遍历（左→根→右）

函数采用中序遍历框架：先递归处理左子树，然后处理当前结点，最后递归处理右子树。叶结点按中序顺序被依次访问，链表中的顺序 = 叶结点的中序遍历顺序。

二、可视化示例：一棵二叉树的变化

2.1 原始二叉树

A / \ B C / \ / \ D E F G / \ \ H I J 其中叶结点（中序顺序）：H → I → E → F → J 非叶结点：A, B, C, D, G

2.2 操作完成后的双向链表

┌─────────────────────────────────────────────────────┐ │ (带头结点的双向循环链表) │ │ │ │ head(H) ⇄ H ⇄ I ⇄ E ⇄ F ⇄ J ⇄ head(H) │ │ ← Left 指针指向前驱 │ │ → Right 指针指向后继 │ └─────────────────────────────────────────────────────┘ 关键变化： · H.Left 原先为NULL → 现在指向J（前驱） · H.Right原先为NULL → 现在指向I（后继） 等等 · H.Right 原先为NULL → 现在指向链表下一个叶结点 · H.Left 原先为NULL → 现在指向链表上一个叶结点

三、完整代码 & 逐行解析

3.1 数据结构定义

typedef struct BiTNode { int Data; struct BiTNode *Left; // 左孩子 / 链表前驱 struct BiTNode *Right; // 右孩子 / 链表后继 } BiTNode, *BiTree; BiTNode *PreLeaf = NULL; // 全局指针，指向最近访问到的叶结点

3.2 主函数：BuildLeafList

void BuildLeafList(BiTree T, BiTree H) { // T: 当前遍历到的树结点 // H: 链表头结点 if (T == NULL) return; // 空树/空子树，直接返回 // ─── 填空(1)：判断T是否为非叶结点 ─── if (T->Left != NULL || T->Right != NULL) { // 情形一：T不是叶结点 → 继续递归遍历左右子树 // ─── 填空(2)：中序遍历左子树 ─── BuildLeafList(T->Left); // ─── 填空(3)：中序遍历右子树 ─── BuildLeafList(T->Right); } else { // 情形二：T是叶结点（Left和Right均为NULL） // 将其链入双向链表 if (PreLeaf == NULL) { // 这是第一个被访问到的叶结点 // 建立头结点与第一个叶结点的双向连接 H->Right = T; T->Left = H; } else { // 这是第2、3、4...个叶结点 // 与前一个叶结点PreLeaf建立双向连接 // ─── 填空(4)：前一个叶结点的Right指向当前叶结点 ─── PreLeaf->Right = T; // ─── 填空(5)：当前叶结点的Left指向前一个叶结点 ─── T->Left = PreLeaf; } // 更新PreLeaf，当前叶结点变为"前一个" PreLeaf = T; // 将当前叶结点的Right置为NULL（暂时） // 如果是最后一个叶结点，后续会与头结点形成循环 T->Right = NULL; } }

3.3 调用入口（考试可能给的框架）

void Solve(BiTree root) { BiTNode *H = (BiTNode*)malloc(sizeof(BiTNode)); H->Left = H->Right = NULL; PreLeaf = NULL; // 初始化全局指针 BuildLeafList(root, H); // 形成循环链表：最后一个叶结点 ⇄ 头结点 if (PreLeaf != NULL) { PreLeaf->Right = H; // 最后叶结点的后继指向头 H->Left = PreLeaf; // 头结点的前驱指向最后叶结点 } }

四、填空推理：为什么这样填？

填空(1)：T->Left != NULL || T->Right != NULL

作用：判断T是否为非叶结点。
如果一个结点的Left和Right均为NULL，它才是叶结点。所以"不是叶结点"的条件是：至少有一个孩子不为NULL，即 Left != NULL || Right != NULL。
易错点：不要写成 &&（且）。如果写成&&，则只有"两个孩子都不为NULL"才进入递归，会漏掉只有一个孩子的分支结点。

填空(2)和(3)：递归遍历左右子树

这是标准的中序遍历框架。先处理左子树（②），再处理右子树（③）。对于非叶结点本身不执行任何链表操作，它只负责"派发"递归。

填空(4)和(5)：双向链表插入操作

这是双向链表尾部追加结点的标准操作：

  1. PreLeaf->Right = T —— 前驱的"后继"指针指向新结点。
  2. T->Left = PreLeaf —— 新结点的"前驱"指针指向前驱。

注意顺序：④和⑤的顺序可以互换，因为它们操作的是不同字段。但如果涉及链表中间插入（断开原链），则必须先连后继再连前驱，否则会丢失信息。本题是尾部追加，顺序无影响。

💡 核心观察：非叶结点（A, B, C, D, G）在遍历中只起到"路由"作用——它们把递归派发到子树，自身不参与链表。只有叶结点被实际链接。

六、常见错误 & 考点总结

⚠️ 错误1：混淆叶结点判断条件。叶结点 = Left和Right同时为NULL。非叶结点 = 至少一个不为NULL（用 || 而非 &&）。
⚠️ 错误2：遍历顺序写错。题目要求中序（左→根→右）。如果写成先序（根→左→右）或后序（左→右→根），链表顺序会不同。
⚠️ 错误3：忘记更新PreLeaf。每处理完一个叶结点，必须 PreLeaf = T，否则下一个叶结点无法正确链接。
⚠️ 错误4：忘记形成循环。遍历结束后，最后一个叶结点的Right和头结点的Left需要额外处理才能形成完整的双向循环链表。
💡 记忆技巧："非叶递归左右跑，叶结点来把链搞。首个叶连头结点，后续叶连前面叶。PreLeaf追着叶结点跑，最后闭环不能少。"
💡 考点本质：这道题融合了三个知识点——①二叉树的遍历（中序）、②叶结点的判定条件、③双向链表的尾部插入。三者结合考察综合能力。

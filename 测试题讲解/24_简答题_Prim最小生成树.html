一、什么是最小生成树（MST）？

1.1 场景类比：修建村村通公路

假设有6个村庄（A, B, C, D, E, F），现在要铺设光缆把它们全部连接起来。任意两个村庄之间修路的成本不同（由地形决定）。目标：用最少的成本让所有村庄连通。

这个问题的数学模型就是最小生成树（Minimum Spanning Tree, MST）：

  • 生成树（Spanning Tree）：包含图中所有顶点，且没有回路的连通子图。有n个顶点则恰好n-1条边。
  • 最小（Minimum）：在所有可能的生成树中，边权之和最小的那棵。
💡 记忆口诀："全部连通无回路，n个顶点n-1边，边权和最小就是MST。"

二、Prim算法核心思想

2.1 一句话概括

Prim算法是一种贪心算法：从任意一个起点开始，每一步选择一条连接"已选顶点集合"与"未选顶点集合"的最小权重边，把边另一端的顶点拉入集合。重复n-1次后得到MST。

2.2 形象类比：病毒传播

想象一种"智能病毒"从城市C开始感染：它总是去感染未被感染且离已感染区域最近的城市。每次只感染一个新城，并记录传播路径。n-1轮后，所有城市都被感染，记录下的路径就是MST。

2.3 数据结构支撑

辅助数组含义初始化
visited[ ]顶点是否已加入生成树全false，起点C置true
lowcost[ ]每个未选顶点到当前生成树的最小边权起点到各邻居的边权，无边则为∞
parent[ ]记录每个顶点通过哪条边连入树（用于回溯路径）起点为-1

2 选择C-B (权重3) —— 当前最小边。B加入生成树。

S = {C, D, B}。用B更新lowcost：B-A=6（比当前7小→更新A为6），B-F=8（比9小→更新F为8），B-E不与E直连。B-D=4但D已在树中忽略。
更新后lowcost: A:6, E:4★, F:8。最小是E的4（E通过D连入）。

3 选择D-E (权重4) —— 通过D连接E。E加入生成树。（注意：不是B-D=4因为D已在树中）

S = {C, D, B, E}。用E更新：E-A=∞（无边），E-F=7（比当前8小→更新F为7）。
更新后lowcost: A:6★, F:7。最小是A的6（A通过B连入）。

4 选择B-A (权重6) —— A加入生成树。

S = {C, D, B, E, A}。用A更新：A-F=5（比当前7小→更新F为5）。
更新后lowcost: F:5★。最小是F的5。

5 选择A-F (权重5) —— F加入生成树。全部顶点加入完毕！

S = {C, D, B, E, A, F}。6个顶点全部在树中，算法结束。

五、Prim vs Kruskal 对比

维度Prim算法Kruskal算法
基本策略选点：从一棵树向外生长选边：对所有边排序，逐条取最短且不构成回路的边
数据结构优先队列（最小堆）+ visited数组并查集（Union-Find）
时间复杂度O(V²) 或 O((V+E)logV)（堆优化）O(E log E)
适用场景稠密图（边多）稀疏图（边少）
起始点需要指定起点不需要起点，全局考虑
中间状态始终是一棵连通的树可能形成多片"森林"再合并

六、Prim算法伪代码

// 输入：带权连通图G=(V,E)，起始顶点start // 输出：最小生成树的边集 function Prim(G, start): for each v in V: visited[v] = false lowcost[v] = ∞ parent[v] = -1 lowcost[start] = 0 for i = 0 to n-1: // 1. 从未选顶点中找到lowcost最小的顶点u u = findMin(lowcost, visited) visited[u] = true // 2. 输出边(parent[u], u) if parent[u] != -1: print(parent[u], u, lowcost[u]) // 3. 用u去更新所有未选邻居的lowcost for each neighbor v of u: if not visited[v] and weight(u,v) < lowcost[v]: lowcost[v] = weight(u,v) parent[v] = u

七、常见错误 & 记忆技巧

⚠️ 错误1：混淆Prim与Dijkstra。
Prim的lowcost[i]是"顶点i到当前生成树的最小距离"，
Dijkstra的dist[i]是"顶点i到源点的最短路径长度"。两者更新逻辑不同！
⚠️ 错误2：更新lowcost时忘记检查visited。
已经加入生成树的顶点不能再被更新，否则会形成回路或覆盖正确值。
⚠️ 错误3：以为每步选全局最小边。
Prim选的是"一端在树内、一端在树外"的最小边，不是所有未用边中的最小边。Kruskal才是全局选边。
💡 记忆技巧：Prim = "向外扩张的树"
想象一棵树从种子发芽，每次长出一根最短的新枝，最终覆盖所有节点。起始顶点就是种子，lowcost就是每个节点到树边界的"最短距离"。
💡 考试快速答题法：
① 画出图，标好所有边权。
② 从起点C出发，圈出C。看C的所有边：C-D(2), C-B(3), C-A(7), C-F(9)。选最小的2，圈出D。
③ 现在{ C,D }看所有一端在圈内一端在圈外的边：C-B(3), C-A(7), C-F(9), D-B(4), D-E(4), D-F(10)。最小是C-B(3)。
④ 重复直到所有顶点被圈出。圈出的顺序就是答案。

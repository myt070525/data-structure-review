🔄 循环队列 — 元素个数计算

循环队列的容量 MAXSIZE = 9，
当前 front = 7，rear = 2，
问队列中有几个元素？
正确答案
4 个元素
元素在位置：7, 8, 0, 1（共4个）

一、循环队列的结构理解

循环队列把数组的头和尾"接"起来，形成一个逻辑上的环。当 rear 或 front 走到数组末尾时，会"绕回"到数组开头。

循环队列 MAXSIZE = 9（下标 0~8） ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐ │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │ └───┴───┴───┴───┴───┴───┴───┴───┴───┘ ↑ ↑ ↑ 元素区 rear=2 front=7 (有数据) (下一个入队位置) (队头位置) 更直观的圆圈图： ┌─────────────────────────┐ ↓ │ ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐ │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │ └───┴───┴───┴───┴───┴───┴───┴───┴───┘ ↑ ↑ ↑ │ │ │ └─ 数据区 ──┘ │ (rear→front │ 顺时针方向) │ │ 数据从 front(7) 开始， │ 沿顺时针走到 rear(2) 之前： │ 7 → 8 → 0 → 1 │ ↑ │ └── 4个元素 ────┘

二、核心公式推导

循环队列元素个数通用公式：

count = (rear − front + MAXSIZE) % MAXSIZE
①
rear - front = 2 - 7 = -5
直接相减为负数，说明 rear 绕回了一圈
②
-5 + MAXSIZE = -5 + 9 = 4
加上容量后转为正数
③
4 % 9 = 4
取模（这里4<9，结果不变）
🎯 结果：4 个元素

三、公式为什么这样设计？

情况1：rear ≥ front（正常情况，没有绕回）

[ _ | _ | A | B | C | _ | _ | _ ] 0 1 2 3 4 5 6 7 ↑front ↑rear count = rear - front = 4 - 2 = 2 个元素（位置2和3）

情况2：rear < front（绕回情况，本题就是）

[ A | B | _ | _ | _ | _ | C | D ] 0 1 2 3 4 5 6 7 ↑rear ↑front count = (rear - front + MAXSIZE) % MAXSIZE = (1 - 6 + 8) % 8 = 3 % 8 = 3 元素在位置 6,7,0 → 共3个

加上 MAXSIZE 再取模的本质：处理 rear 绕回导致相减为负的情况。取模确保结果在 [0, MAXSIZE) 范围内。

四、手动画圆圈法（不用背公式！）

如果你记不住公式，可以画一个圆圈，从 front 数到 rear：

MAXSIZE = 9，下标 0 到 8 围成一个圈： ┌───┐ │ 0 │ ← 数据区（顺时针从7数到2之前） ┌───┐├───┤├───┐ │ 8 ││ ││ 1 │ ← 数据区 ├───┤│ │├───┤ │ 7 ││ ││ 2 │ ← rear（下一个入队位置，不算在元素中） ├───┤│ │├───┤ │ 6 ││ ││ 3 │ ├───┤│ │├───┤ │ 5 ││ ││ 4 │ └───┘└───┘└───┘ ↑ front（队头，算第一个元素） 从 front(7) 开始顺时针数： 位置7 → 1个 位置8 → 2个 位置0 → 3个 位置1 → 4个 位置2 → 是rear，不计数！停止！ → 共 4 个元素 ✅
💡 口诀："从 front 开始数，数到 rear 就停（不含 rear）"，顺时针绕圈数。如果 front==rear，队列为空（count=0）。

五、循环队列关键知识点对比

情况条件公式/结论
队列为空 front == rear count = 0
队列为满 (rear+1) % MAXSIZE == front 牺牲一个位置作为"哨兵"
元素个数 任意情况 (rear - front + MAXSIZE) % MAXSIZE
入队操作 — rear = (rear + 1) % MAXSIZE
出队操作 — front = (front + 1) % MAXSIZE
最大元素数 — MAXSIZE - 1（因为要区分空和满）

六、为什么"牺牲一个位置"？

循环队列面临的一个经典问题：如何区分"队空"和"队满"？

两者都会出现 front == rear：

  • 队空时：front 和 rear 指向同一个位置（没有元素）
  • 队满时：rear 绕了一圈追上了 front（所有位置都满了）

为了区分，常见方案是牺牲一个存储位置，使得"队满"的条件变为 (rear+1) % MAXSIZE == front（rear 的下一个就是 front，但那个位置不存数据）。

MAXSIZE=9，最大存储 8 个元素（牺牲1个位置） 本题：已有 4 个元素，还剩 4 个空位可入队 （位置 2,3,4,5 可入队，位置6是front-1="哨兵"）
⚠️ 注意：不同教材对循环队列的实现略有差异。有的教材用一个单独的计数器 count 来区分空/满，此时不需要牺牲位置，最大存储 = MAXSIZE。请以你的教材为准！

七、同类变式练习

变式①：front = 3, rear = 8, MAXSIZE = 10

count = (8 - 3 + 10) % 10 = 15 % 10 = 5

变式②：front = 0, rear = 0, MAXSIZE = 6

count = (0 - 0 + 6) % 6 = 0 → 队列为空

变式③：front = 4, rear = 3, MAXSIZE = 7（满的情况）

count = (3 - 4 + 7) % 7 = 6 % 7 = 6 = MAXSIZE - 1 → 队列满！

🎯 考试快速检查法：计算 (rear - front + MAXSIZE) % MAXSIZE。如果结果为 0 → 队空；结果为 MAXSIZE-1 → 队满（牺牲一个位置方案）。

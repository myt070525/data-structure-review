📐 时间复杂度 — 嵌套 vs 并列循环

代码如下，求时间复杂度：
for (i = 0; i < m; i++)
  for (j = 0; j < n; j++)
    a[i][j] = 0;
正确答案
T(n) = O(m × n)
若 m ≈ n，可简写为 O(n²)

一、什么是时间复杂度？

时间复杂度衡量的是算法执行时间随输入规模增长的趋势，而不是精确的秒数。我们用大O记号来表示：

  • O(1)：常数时间 — 不管输入多大，执行时间不变
  • O(n)：线性时间 — 输入翻倍，时间也翻倍
  • O(n²)：平方时间 — 输入翻倍，时间翻四倍
  • O(log n)：对数时间 — 非常快，二分查找就是

通俗类比：O(1)是"看一眼"，O(n)是"翻一遍"，O(n²)是"每个人都和其他人握一次手"。

二、嵌套循环 vs 并列循环 — 关键区别

🔗 嵌套循环（乘法关系）

// 嵌套：外层每执行1次，内层执行n次 for (i = 0; i < m; i++) // m 次 for (j = 0; j < n; j++) // 每次 n 次 a[i][j] = 0; // 总共 m×n 次

总执行次数 = m × n → O(m × n)

📏 并列循环（加法关系）

// 并列：两个循环依次执行 for (i = 0; i < m; i++) // m 次 a[i] = 0; for (j = 0; j < n; j++) // n 次 b[j] = 0;

总执行次数 = m + n → O(m + n)（不是 O(m×n)！）

❌ 嵌套
O(m×n)
内层重复执行
→ 乘法
✅ 并列
O(m+n)
各自执行一遍
→ 加法

四、逐行分析法

代码行语句执行次数说明
1 for (i=0; i<m; i++) m + 1 条件判断执行 m+1 次（最后一次判断为假）
2 for (j=0; j<n; j++) m × (n+1) 外层每循环一次，内层判断执行 n+1 次
3 a[i][j] = 0; m × n 核心赋值语句，主导时间复杂度

核心语句执行 m × n 次，因此 T(n) = O(m × n)。

五、常见循环模式对比一览

模式代码结构时间复杂度例子
单层循环 for(i=0;i<n;i++) O(n) 遍历数组
双层嵌套 for(i) for(j) O(n²) 矩阵赋值、冒泡排序
三层嵌套 for(i) for(j) for(k) O(n³) 矩阵乘法（朴素）
独立并列 for(i) ... for(j) ... O(n+m) 两个独立遍历
指数缩减 for(i=1;i<=n;i*=2) O(log n) 二分查找
嵌套缩减 for(i) while(j/=2) O(n log n) 归并排序、堆排序

六、三步判断法（考试技巧）

1
找最内层核心语句 — 通常是执行次数最多的那句。忽略声明、忽略条件判断。本题核心语句是 a[i][j]=0。
2
数嵌套层数 — 每层循环给核心语句的执行次数乘以该层变量范围。i 循环 m 次，j 循环 n 次 → m×n。
3
取最高阶，去系数 — 如果 m≈n，O(m×n) 就是 O(n²)。常数系数如 2n² → O(n²)。
⚠️ 常见错误：把两个并列的循环当成嵌套来算！
并列循环：for(i) ... 完了再 for(j) ... → 结果是 O(m+n)，不是 O(m×n)！
记住口诀："套着走→乘法，顺着走→加法"
🎯 考试抓分技巧：看到两个 for 关键字，先看大括号的位置——如果第二个 for 在第一个的 大括号里面，那就是嵌套（乘法）；如果在第一个的 大括号外面，那就是并列（加法）。

七、典型变式练习

变式①：内层次数依赖外层

for (i = 0; i < n; i++) for (j = 0; j < i; j++) // 注意：j < i，不是 j < n！ sum++;

此时总次数 = 0 + 1 + 2 + ... + (n-1) = n(n-1)/2 → O(n²)

变式②：两层并列

for (i = 0; i < n; i++) a[i] = 0; // n 次 for (j = 0; j < m; j++) b[j] = 0; // m 次

总次数 = n + m → O(n + m)，不是 O(n×m)！

变式③：循环内调用函数

for (i = 0; i < n; i++) binary_search(arr, target); // 每次 O(log n)

总时间 = n × O(log n) → O(n log n)

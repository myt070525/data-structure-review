二、带头双向链表的结构

2.1 什么是"带头结点的双向链表"？

头结点（Header / Dummy Node）是一个不存储实际数据的结点，它的存在使得链表的边界操作（如在头部插入/删除）与中间操作统一，无需特殊处理。

带头结点的双向链表（删除前）： ┌─ L（头结点/哑结点）────────────────────────────────────┐ │ │ ▼ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ HEADER │◄───│ Data: │◄───│ Data: │◄───│ Data: │ │ (L) │ │ Node₁ │ │ Node₂ │ │ Node₃ │ │ │───▶│ (第1个) │───▶│ (第2个) │───▶│ (第3个) │───▶ NULL └──────────┘ └──────────┘ └──────────┘ └──────────┘ ▲ ▲ ▲ ▲ │ │ │ │ L->Next q->Next Node₂.Next Node₃.Next (指向Node₁) (指向Node₂) 指针约定： · 每个结点有两个指针：Next（→后继）和 Previous（←前驱） · 头结点L的Previous通常为NULL（或指向尾结点形成循环链表，本题为NULL） · 最后一个结点的Next为NULL · 空表：L->Next == NULL（头结点后面没有实际结点）

2.2 关键指针关系

表达式含义删除前示例值
L头结点指针（函数参数）指向Header
L->Next第一个实际结点（或NULL）指向Node₁
q待删除结点指针（填空②设置）L->Next = Node₁
q->Next待删结点的后继指向Node₂
q->Next->Previous后继结点的前驱指针原指向Node₁(q)，需改为L

3.2 逐步操作详解

步骤代码操作说明示意图
① if (L->Next == NULL) return; 判空：如果头结点后面没有结点，直接返回 L.Next → NULL（空表）
② q = L->Next; 定位：q指向第一个实际结点（待删除） q → Node₁
④ L->Next = q->Next; 前向 bypass：头结点的Next跳过q，直接指向q的后继 L.Next → Node₂
⑤ if (q->Next != NULL)
    q->Next->Previous = L;		 后向 bypass：后继结点的Previous回指头结点L
需先判断q->Next是否为NULL（q可能是最后一个结点）		 Node₂.Prev → L
⑥ free(q); 释放：释放q指向的结点内存 Node₁ 被回收
🔴 关键：④⑤顺序为什么不能反？

如果先执行⑤ q->Next->Previous = L，再执行④ L->Next = q->Next：
· 步骤⑤中，q->Next还是有效的（因为q还没被free，q->Next仍指向Node₂）——可以正常工作。
· 步骤④中，L->Next被改为q->Next——也正常。

那为什么强调顺序？ 因为这是"先改前向指针，再改后向指针"的标准范式。在一般的双向链表删除中（不仅限于头结点），如果操作涉及 q->Previous->Next 和 q->Next->Previous，一旦先修改了 q->Next 或 q->Previous 中的一个，就可能影响另一个操作所需的原始值。养成"④→⑤"的习惯能避免在复杂场景中犯错。

更准确地说：本题中④和⑤操作的是不同字段（L->Next 和 q->Next->Previous），顺序在本题中不影响正确性。但考试中如果问"顺序能否互换"，建议回答"建议④在⑤之前，以保证代码可读性和通用性"。

四、逐空推理：为什么这样填？

填空①：L->Next == NULL

考察点：空表的判定条件。
带头结点的双向链表中，头结点L始终存在。如果链表为空（没有任何实际数据结点），则头结点的Next指针为NULL。因此 L->Next == NULL 是判空条件。
易错：不要写成 L == NULL——头结点L本身一定存在（调用者传入），NULL的只可能是L->Next。也不要写成 q == NULL——此时q还没有被赋值。

填空②：q = L->Next

考察点：定位待删除结点。
要删除的是第一个实际结点，它就在头结点L的后面。因此 q = L->Next 即可让q指向它。
易错：不要写成 q = L（会删头结点本身！），也不要写成 q = L->Next->Next（会跳过第一个结点）。

填空③：q->Next->Previous = L

考察点：双向链表的"后向重连"。
q被删除后，q的后继结点（q->Next）的前驱指针不再指向q，而应指向L。
易错：①忘记判空——如果q是最后一个结点（q->Next == NULL），则q->Next->Previous会引发空指针异常。题目已给出if保护。②写成 q->Previous->Next = q->Next——这是删除"中间结点"的写法，但本题q的前驱就是L，所以q->Previous->Next就是L->Next，与④重复但也可行。

/* 完整示例：带头双向链表 + Delete_head */ #include <stdio.h> #include <stdlib.h> typedef struct DNode { int Data; struct DNode *Next, *Previous; } DNode; /* 删除第一个实际结点 */ void Delete_head(DNode *L) { DNode *q; if (L->Next == NULL) // ① 判空 return; q = L->Next; // ② q指向第一个结点 L->Next = q->Next; // ④ 头结点跳过q if (q->Next != NULL) // ⑤ 如果q后面还有结点 q->Next->Previous = L; // 后继结点的前驱回指L free(q); // ⑥ 释放q } /* 测试：创建 头⇄10⇄20⇄30 并依次删除 */ int main() { DNode *L = (DNode*)malloc(sizeof(DNode)); L->Previous = NULL; // 插入30, 20, 10（头插法） for (int v = 30; v >= 10; v -= 10) { DNode *n = (DNode*)malloc(sizeof(DNode)); n->Data = v; n->Next = L->Next; n->Previous = L; if (L->Next) L->Next->Previous = n; L->Next = n; } // 链表：L ⇄ 10 ⇄ 20 ⇄ 30 Delete_head(L); // 删除10 → L ⇄ 20 ⇄ 30 Delete_head(L); // 删除20 → L ⇄ 30 Delete_head(L); // 删除30 → L（空表） Delete_head(L); // 空表，直接return return 0; }

七、常见错误 & 考点总结

⚠️ 错误1：判空条件写成 L==NULL。L是头结点指针，由调用者保证非空。真正的空表条件是头结点后面没有结点，即L->Next==NULL。
⚠️ 错误2：填空③忘记判空保护。如果q是最后一个结点（q->Next==NULL），直接写q->Next->Previous会段错误！题目已给出if保护，填空③只需写操作本身。
⚠️ 错误3：填空②写成 q=L。这样会删除头结点本身，导致整个链表结构被破坏。头结点不应该被删除。
⚠️ 错误4：混淆Next和Previous的方向。双向链表中，"Next=后继"、"Previous=前驱"。删除时要同时修复前驱的Next（跳过q）和后继的Previous（回指L）。
💡 考点本质：三道填空分别考察——①带头链表空表判定；②定位待删结点；③双向链表删除的后向指针修复。核心是理解"头结点是哨兵，不存数据，简化边界"这一设计思想。
💡 记忆口诀："一判空、二定位、三跳前、四回指、五释放"。一二三（Next）是正向链，四（Previous）是反向链——双向链表删除 = 正向反向都绕过被删结点。

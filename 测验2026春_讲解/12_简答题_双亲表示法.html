1.2 解读方式

每个结点的parent值告诉我们"谁是我的双亲"。例如：

  • A的parent=1 → A的双亲是下标1的结点，即B
  • B的parent=−1 → B没有双亲，B是根结点
  • C的parent=4 → C的双亲是下标4的结点，即E
  • D的parent=8 → D的双亲是下标8的结点，即I
  • E的parent=0 → E的双亲是下标0的结点，即A
  • F的parent=0 → F的双亲是A
  • G的parent=8 → G的双亲是I
  • H的parent=8 → H的双亲是I
  • I的parent=0 → I的双亲是A

三、孩子兄弟表示法转换

3.1 什么是孩子兄弟表示法？

孩子兄弟表示法（又叫二叉链表表示法或左孩子右兄弟表示法）是树的另一种存储方式。每个结点有两个指针：

  • Left（FirstChild）：指向该结点的第一个孩子
  • Right（NextSibling）：指向该结点的下一个兄弟

这样任意一棵树都可以转化为一棵二叉树！转化规则：左指针连孩子，右指针连兄弟。

3.2 转换过程

结点孩子列表FirstChild →兄弟链（NextSibling）
B{A}AA.NextSibling = NULL（独生子）
A{E, F, I}EE→F→I→NULL
E{C}CC.NextSibling = NULL
F{}（叶子）NULL—
I{D, G, H}DD→G→H→NULL
C{}（叶子）NULL—
D{}（叶子）NULL—
G{}（叶子）NULL—
H{}（叶子）NULL—

3.3 孩子兄弟表示法的二叉树形态

原树（左孩子右兄弟表示法转化后的二叉树）： ┌───┐ │ B │ └─┬─┘ │ (FirstChild) ┌─┴─┐ │ A │ ───→ NULL (NextSibling: 独生子) └─┬─┘ │ (FirstChild) ┌─┴─┐ │ E │ ───→ F ───→ I ───→ NULL └─┬─┘ (无孩) └─┬─┘ │ │ (FirstChild) ┌─┴─┐ ┌──┴──┐ │ C │ │ D │ ───→ G ───→ H ───→ NULL └───┘ └─────┘ 说明： · 实线（↓）= FirstChild 指针 · 虚线（→）= NextSibling 指针 · 原树中"兄弟关系"变成了二叉树中的"右链" · 原树中"父子关系"变成了二叉树中的"左链"
💡 关键对应：原树中，A有三个孩子E、F、I。在孩子兄弟表示法中，A的FirstChild=E，然后E→F→I通过NextSibling串联。原树中I有三个孩子D、G、H，同理：I的FirstChild=D，D→G→H通过NextSibling串联。

四、三种表示法对比

表示法存储结构找双亲找孩子找兄弟空间适用场景
双亲表示法 一维数组，每个元素存data+parent下标 O(1) ✓ O(n) ✗ O(n) ✗ O(n) 并查集、频繁找双亲
孩子链表法 数组+链表，每个结点挂一个孩子链表 O(n) ✗ O(1) ✓ O(k) O(n+e) 频繁找孩子
孩子兄弟法 二叉链表（left=孩子, right=兄弟） O(n) ✗ O(1) ✓ O(1) ✓ O(n) 通用、可转二叉树

4.1 双亲表示法的结构定义

/* 双亲表示法 —— 顺序存储 */ #define MAX_SIZE 100 typedef struct { char Data; // 结点数据 int Parent; // 双亲在数组中的下标，-1表示根 } PTNode; typedef struct { PTNode nodes[MAX_SIZE]; int n; // 结点总数 } PTree;

4.2 孩子兄弟表示法的结构定义

/* 孩子兄弟表示法 —— 链式存储（二叉链表） */ typedef struct CSNode { char Data; struct CSNode *FirstChild; // 指向第一个孩子 struct CSNode *NextSibling; // 指向下一个兄弟 } CSNode, *CSTree;

六、常见错误 & 考点提醒

⚠️ 错误1：双亲表示法的下标搞混。parent存储的是数组下标，不是数据本身。A的parent=1表示双亲是nodes[1]=B；而E的parent=0表示双亲是nodes[0]=A。
⚠️ 错误2：还原树时漏掉多孩子。一个结点可能是多个结点的双亲（如A是E、F、I的双亲）。还原时必须找出所有parent等于该结点下标的结点。
⚠️ 错误3：孩子兄弟法中兄弟链顺序错。兄弟链必须按某种顺序（如从左到右）。本题中A的三个孩子按E→F→I的顺序串联，不要写成F→E→I。
⚠️ 错误4：混淆"结点的度"和"树的度"。结点A的度=3（3个孩子），结点E的度=1，结点I的度=3。树的度=max(各结点的度)=3。
💡 考点本质：双亲表示法是并查集的基础存储结构；孩子兄弟表示法是将树转化为二叉树的关键桥梁。掌握二者的相互转换是考试重点。

🌳 简答题：二叉树遍历

第10题 · 给定一棵7结点二叉树，写出三种遍历序列
✅ 答案：先序 abdgcef · 中序 dgbaecf · 后序 gdbefca

一、题目回顾

1.1 二叉树结构

已知一棵二叉树的结构如下：

  • a 为根结点
  • b 是 a 的左孩子，c 是 a 的右孩子
  • d 是 b 的左孩子
  • g 是 d 的右孩子
  • e 是 c 的左孩子，f 是 c 的右孩子

要求：写出该二叉树的先序遍历、中序遍历、后序遍历序列。

1.2 树形图示

┌───┐ │ a │ ← 根结点 (root) └─┬─┘ / \ / \ ┌───┐ ┌───┐ │ b │ │ c │ └─┬─┘ └─┬─┘ / / \ / / \ ┌───┐ ┌───┐ ┌───┐ │ d │ │ e │ │ f │ ← e和f是c的左右孩子 └─┬─┘ └───┘ └───┘ \ \ ┌───┐ │ g │ ← g是d的右孩子 └───┘ 共7个结点：a, b, c, d, e, f, g

二、三种遍历方式详解

2.1 什么是二叉树的遍历？

遍历是指按某种搜索路径访问树中每个结点恰好一次。根据访问"根"的时机不同，分为三种基本遍历：

遍历方式访问顺序规则简称记忆口诀
先序 根 → 左 → 右 NLR "先访问根"
中序 左 → 根 → 右 LNR "根在中间"
后序 左 → 右 → 根 LRN "根最后访问"
💡 核心记忆：三种遍历的左永远在右之前。区别仅在于"根"的位置——先（根最先）、中（根中间）、后（根最后）。

三、先序遍历（NLR）—— 根 → 左 → 右

3.1 遍历过程

先序遍历路径（①~⑦为访问顺序）： ┌───┐ ①→ │ a │ ─────────────────────┐ └─┬─┘ │ / \ │ / \ │ ┌───┐ ┌───┐ │ ②→ │ b │ │ c │ ←⑤ │ └─┬─┘ └─┬─┘ │ / / \ │ / / \ │ ┌───┐ ┌───┐ ┌───┐ │ ③→ │ d │ │ e │ │ f │ │ └─┬─┘ └───┘ └───┘ │ \ ↑⑥ ↑⑦ │ \ │ ┌───┐ │ ④→ │ g │ │ └───┘ │ │ 访问顺序：①a → ②b → ③d → ④g → ⑤c → ⑥e → ⑦f

3.2 逐步推导

步骤当前结点动作已访问序列
1a访问根a，递归左子树a
2b访问根b，递归左子树ab
3d访问根d，递归左子树（空），递归右子树abd
4g访问根g，左右均空，返回abdg
5ca的右子树：访问根c，递归左子树abdg c
6e访问根e，左右均空，返回abdgce
7f访问根f，左右均空，返回abdgcef
✅ 先序遍历结果：a b d g c e f

四、中序遍历（LNR）—— 左 → 根 → 右

4.1 遍历过程

中序遍历路径（①~⑦为访问顺序）： ┌───┐ ④→ │ a │ ─────────────────────┐ └─┬─┘ │ / \ │ / \ │ ┌───┐ ┌───┐ │ ③→ │ b │ │ c │ ←⑥ │ └─┬─┘ └─┬─┘ │ / / \ │ / / \ │ ┌───┐ ┌───┐ ┌───┐ │ ②→ │ d │ │ e │ │ f │ │ └─┬─┘ └───┘ └───┘ │ \ ↑⑤ ↑⑦ │ \ │ ┌───┐ │ ①→ │ g │ │ └───┘ │ 访问顺序：①g → ②d → ③b → ④a → ⑤e → ⑥c → ⑦f 关键：d没有左孩子，但有右孩子g。中序先访问d的左（空）， 然后访问d本身，再访问d的右子树g。 等等——不对！d有右孩子g，按照"左→根→右"， 应先遍历d的左子树（空），访问d，再遍历右子树g。 但g没有左孩子，所以先访问g。 正确顺序：①g ②d ③b ...

4.2 逐步推导

步骤当前结点动作已访问序列
1a→b→d→g一路向左到底：a→b→d→g。g无左孩子，访问gg
2dg的父结点d：左子树已遍历完，访问d，再遍历d的右子树g（已访问）gd
3bd的父结点b：左子树已遍历完，访问bgdb
4ab的父结点a：左子树已遍历完，访问a，进入右子树gdba
5c→e一路向左：c→e。e无左孩子，访问egdbae
6ce的父结点c：左子树已遍历完，访问cgdbaec
7f遍历c的右子树：访问fgdbaecf
✅ 中序遍历结果：d g b a e c f
注意：d在g之前！因为d是g的父结点，中序规则"左→根→右"：先访问d的左（空），再访问d，最后访问d的右（g）。所以顺序是d→g，不是g→d。
⚠️ 常见错误：以为"一路向左到底"就是先访问最左下的g。实际上中序遍历的"访问"发生在从左子树返回之后！走到g时，g虽然没有左孩子，但g不是第一个被访问的——等等，g确实是最左下的结点，它没有左孩子，所以g确实是第一个被访问的。但题目中d在g之前，说明g是d的右孩子，不是左孩子！

五、后序遍历（LRN）—— 左 → 右 → 根

5.1 遍历过程

后序遍历路径（①~⑦为访问顺序）： ┌───┐ ⑦→ │ a │ ─────────────────────┐ └─┬─┘ │ / \ │ / \ │ ┌───┐ ┌───┐ │ ③→ │ b │ │ c │ ←⑥ │ └─┬─┘ └─┬─┘ │ / / \ │ / / \ │ ┌───┐ ┌───┐ ┌───┐ │ ②→ │ d │ │ e │ │ f │ │ └─┬─┘ └───┘ └───┘ │ \ ↑④ ↑⑤ │ \ │ ┌───┐ │ ①→ │ g │ │ └───┘ │ 访问顺序：①g → ②d → ③b → ④e → ⑤f → ⑥c → ⑦a

5.2 逐步推导

步骤当前结点动作已访问序列
1g左右均空，访问gg
2d左右子树均遍历完，访问dgd
3b左右子树均遍历完，访问bgdb
4e左右均空，访问egdbe
5f左右均空，访问fgdbef
6c左右子树均遍历完（e和f已访问），访问cgdbefc
7a左右子树均遍历完，访问agdbefca
✅ 后序遍历结果：g d b e f c a

六、三种遍历对比总结

遍历方式访问序列第一个访问最后一个访问应用场景
先序 NLR a b d g c e f 根 a 最右下 f 复制树、前缀表达式
中序 LNR d g b a e c f 最左下 d 最右下 f BST有序输出、中缀表达式
后序 LRN g d b e f c a 最左下 g 根 a 删除树、后缀表达式

6.1 树结点位置与遍历顺序的关系

每个结点在三种遍历中的位置编号： 结点 先序 中序 后序 ──── ──── ──── ──── a 1 4 7 / \ b c 2 3 3 / / \ d e f 3 2 2 \ 4 1 1 g 观察： · 先序：根总是第1个（NLR，N在最前） · 中序：根在中间，左子树的结点都在根之前，右子树的结点都在根之后 · 后序：根总是最后1个（LRN，N在最后） · 三种遍历中，左子树的结点始终在右子树的结点之前被访问
💡 应试技巧：给定一棵树的图形，快速写出遍历序列的方法——用笔沿着树的外轮廓画一圈（类似"包络线"），经过结点左侧时写先序，经过结点下方时写中序，经过结点右侧时写后序。这就是著名的"三色遍历法"。

七、递归代码实现（参考）

/* 二叉链表存储结构 */ typedef struct BiTNode { char Data; struct BiTNode *Left, *Right; } BiTNode, *BiTree; /* 先序遍历：根 → 左 → 右 */ void PreOrder(BiTree T) { if (T == NULL) return; printf("%c ", T->Data); // ① 访问根 PreOrder(T->Left); // ② 遍历左子树 PreOrder(T->Right); // ③ 遍历右子树 } /* 中序遍历：左 → 根 → 右 */ void InOrder(BiTree T) { if (T == NULL) return; InOrder(T->Left); // ① 遍历左子树 printf("%c ", T->Data); // ② 访问根 InOrder(T->Right); // ③ 遍历右子树 } /* 后序遍历：左 → 右 → 根 */ void PostOrder(BiTree T) { if (T == NULL) return; PostOrder(T->Left); // ① 遍历左子树 PostOrder(T->Right); // ② 遍历右子树 printf("%c ", T->Data); // ③ 访问根 }
💡 三种遍历的代码几乎一模一样，唯一区别是 printf 的位置！先序在递归之前，中序在两次递归之间，后序在递归之后。

八、常见错误 & 考点提醒

⚠️ 错误1：混淆遍历顺序。三种遍历的"左→右"顺序不变，变的是"根"的位置。不要把中序记成"右→根→左"。
⚠️ 错误2：跳过空子树。即使某子树为空，递归流程中仍然要"经过"它。例如d的左子树为空，中序仍要先遍历这个"空"，然后才访问d。
⚠️ 错误3：用手工"扫视"代替逐步推导。对于复杂树，凭直觉"扫一眼"很容易漏掉结点。建议用纸笔逐步写出递归调用栈。
⚠️ 错误4：忘记左永远先于右。无论哪种遍历，左子树的结点一定在右子树的结点之前被访问。如果某序列中右子树结点出现在左子树结点之前，那一定不是合法的遍历序列。
💡 考点本质：二叉树的三种遍历是数据结构最基础的操作，后续的BST查找、表达式树求值、树的序列化/反序列化都依赖它。务必滚瓜烂熟。

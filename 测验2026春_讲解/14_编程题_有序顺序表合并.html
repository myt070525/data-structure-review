📋 编程题：有序顺序表合并

第14题 · 编写Merge函数，将两个有序顺序表合并为一个有序顺序表
✅ 核心：双指针归并 · 时间复杂度 O(m+n) · 空间复杂度 O(m+n)

一、题目理解

1.1 问题描述

已知两个有序顺序表L1和L2（假设均为升序排列），编写函数 Merge(L1, L2) 将它们合并为一个新的有序顺序表L3。要求L3中的元素仍然保持升序，且算法的时间复杂度为O(m+n)。

1.2 输入输出示例

L1 = [ 2, 5, 8, 11, 15 ] (长度 m=5，升序) L2 = [ 1, 3, 6, 9, 12, 14 ] (长度 n=6，升序) ↓ Merge(L1, L2) ↓ L3 = [ 1, 2, 3, 5, 6, 8, 9, 11, 12, 14, 15 ] (长度 m+n=11)

1.3 核心思想：双指针归并

想象桌上有两叠从小到大排好序的扑克牌。要合成一叠仍然有序的牌，每次只需比较两叠牌最上面的那张，把较小的拿走放入结果叠。这就是归并排序中"归并（Merge）"操作的本质。

二、完整代码

2.1 顺序表结构定义

#define MAX_SIZE 100 typedef struct { int Data[MAX_SIZE]; // 存储元素的数组 int Length; // 当前长度 } SeqList;

2.2 Merge函数（核心）

/* 合并两个升序顺序表L1和L2，返回新的升序顺序表L3 */ SeqList Merge(SeqList L1, SeqList L2) { SeqList L3; L3.Length = L1.Length + L2.Length; // L3的长度 = 两者之和 int i = 0, j = 0, k = 0; // 三指针：i→L1, j→L2, k→L3 // ─── 阶段一：两表都有元素时，比较并取较小的 ─── while (i < L1.Length && j < L2.Length) { if (L1.Data[i] <= L2.Data[j]) { // L1的更小（或相等） L3.Data[k] = L1.Data[i]; // 取L1的 i++; // i后移 } else { // L2的更小 L3.Data[k] = L2.Data[j]; // 取L2的 j++; // j后移 } k++; // k总是后移 } // ─── 阶段二：L1还有剩余，全部倒入L3 ─── while (i < L1.Length) { L3.Data[k] = L1.Data[i]; i++; k++; } // ─── 阶段三：L2还有剩余，全部倒入L3 ─── while (j < L2.Length) { L3.Data[k] = L2.Data[j]; j++; k++; } return L3; // 返回合并后的新顺序表 }
💡 注意：阶段二和阶段三最多只有一个会执行（while(i<L1.Length)和while(j<L2.Length)是互斥的剩余处理）。当阶段一的while结束时，要么i已经走完L1，要么j已经走完L2。

四、复杂度分析

指标值说明
时间复杂度O(m + n)每个元素恰好被访问和复制一次。while循环的总迭代次数 = m + n
空间复杂度O(m + n)需要创建L3来存放m+n个元素。如果不算L3（视为输出），则为O(1)
稳定性稳定 ✓当L1[i]==L2[j]时取L1的（≤），相等元素的相对顺序保持不变
比较次数≤ m+n−1每次比较至少消耗一个元素（i++或j++），最多比较m+n−1次

4.1 为什么是O(m+n)而不是O(m×n)？

每次比较后，i或j至少有一个会+1。i最多移动到m，j最多移动到n，所以最多进行m+n次比较。这是一种典型的"双指针单向扫描"模式，两个指针都只前进不后退，因此是线性时间。

💡 变体：如果要求不创建新表，而是将L2合并到L1中（假设L1有足够空间），则需要从后往前归并（三指针逆向移动），以避免覆盖L1中尚未处理的元素。这是"合并两个有序数组"在LeetCode等平台上的经典变体。

六、变体：原地合并（从后往前）

/* 将L2合并到L1中，假设L1的空间足够容纳m+n个元素 */ /* L1的有效元素在 [0..m-1]，后面有n个空位 */ void MergeInPlace(int L1[], int m, int L2[], int n) { int i = m - 1; // L1的最后一个有效元素 int j = n - 1; // L2的最后一个元素 int k = m + n - 1; // 合并后的最后一个位置 while (i >= 0 && j >= 0) { if (L1[i] > L2[j]) L1[k--] = L1[i--]; // 大的放后面 else L1[k--] = L2[j--]; } while (j >= 0) L1[k--] = L2[j--]; // L2还有剩余（L1的剩余已经在自己位置上了） }
原地合并示意（从后往前）： L1 (m=3, 容量=7): [ 2, 5, 8, __, __, __, __ ] ↑i=2 ↑k=6 L2 (n=4): [ 1, 3, 6, 9 ] ↑j=3 每次比较 L1[i] 和 L2[j]，把较大的放到 L1[k]，然后对应指针--。 这样做不会覆盖L1中未处理的元素（因为k始终在i之后）。

七、常见错误 & 考点总结

⚠️ 错误1：忘记处理剩余元素。阶段一结束后，L1或L2可能还有剩余。必须用单独的while循环把剩余元素全部复制到L3。漏掉会导致L3不完整。
⚠️ 错误2：循环条件写错。循环条件是 i < L1.Length && j < L2.Length（两个都没走完才继续比较），而不是 ||。
⚠️ 错误3：忘记k++。每次给L3赋值后，k必须+1。建议把 k++ 写在while体末尾统一执行，或者合并到赋值语句中写成 L3.Data[k++] = ...。
⚠️ 错误4：稳定性处理不当。当L1[i]==L2[j]时，若使用<（而非<=）会取L2的元素，导致稳定性被破坏。使用<=保证L1的相等元素先被放入L3。
💡 考点本质：有序顺序表合并是归并排序（Merge Sort）的核心操作。理解双指针单向扫描模式后，可以轻松扩展到：合并多个有序链表、求两个有序数组的交集/并集/差集、以及外部排序中的多路归并。
💡 记忆口诀："i指L1，j指L2，谁小取谁往后挪；一表取完停比较，剩余全部倒进锅。"
